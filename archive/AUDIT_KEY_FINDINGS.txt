❌ ISSUE #2: Inline Styles Scattered Throughout Docs
   Location: index.html, docs/*.html
   Severity: BLOCKING (violates core philosophy)
   Count: 205+ instances
   
   Example (index.html, line ~400):
   ├─ <div style="background: linear-gradient(135deg, ...); border: none;">
   ├─ <div style="display: grid; grid-template-columns: repeat(auto-fit, ...);">
   ├─ <div style="display: flex; gap: var(--a-space-m); flex-wrap: wrap; ...">
   
   Why it's wrong:
   → Framework philosophy: CSS variables in CSS files, not inline
   → CSS scattered everywhere becomes unmaintainable
   → Responsive design broken (can't apply media queries to inline styles)
   → Contradicts "semantic-first" approach
   
   Fix: Move to semantic component classes in CSS files
   Effort: 3-4 hours

❌ ISSUE #3: Utility-Class Patterns in Docs
   Location: index.html, docs/animations.html, etc.
   Severity: BLOCKING (explicitly contradicts framework)
   Count: 17+ instances
   
   Examples:
   ├─ <div class="grid grid-cols-2">
   ├─ <div class="card glass animate delay-1" style="opacity: 0;">
   ├─ <div style="display: flex; gap: var(--a-space-m); flex-wrap: wrap;">

   Example:
   ├─ WRONG: <div class="grid grid-cols-2">
   ├─ RIGHT: <div class="feature-grid">

❌ ISSUE #4: Inline Onclick Handlers
   Location: docs/animations.html, docs/components/modal.html, etc.
   Severity: BLOCKING (violates component pattern)
   Count: 22 instances
   
   Example:
   ├─ WRONG: <button onclick="handleThemeChange('dark')">Dark</button>
   ├─ RIGHT: <button data-theme="dark" class="theme-toggle">Dark</button>
   ├─ Then: document.addEventListener('click', (e) => {
   │        if (e.target.matches('.theme-toggle')) { ... }
   │      });
   
   Why it's wrong:
   → Component lifecycle pattern (init/destroy) doesn't work with inline onclick
   → Memory leaks possible
   → Not declarative
   → Inconsistent with framework's design
   
   Fix: Use event delegation + data-* attributes
   Effort: 2-3 hours

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
