# Data Tables
Beautiful, scrollable, interactive data tables with a single CSS class.
## The Semantic Reality
Look at any utility-first framework's documentation for a basic striped data table. You will find over 50 classes spread across the <thead>, <tr>, <th>, and <td> elements. If you want to change the padding, you have to edit 20 different nodes.
Axiom01 asks: *Why are you styling the leaves of a tree instead of the trunk?*
### Axiom01 Approach
```html
<!-- HTML: Standard | Classes: 1 -->
<table class="table">
  <caption>Recent Transactions</caption>
  <thead>
    <tr>
      <th>Date</th>
      <th>Description</th>
      <th>Amount</th>
      <th>Status</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Oct 24, 2026</td>
      <td>Server Hosting</td>
      <td>$45.00</td>
      <td><span class="badge success">Paid</span></td>
    </tr>
    <tr>
      <td>Oct 21, 2026</td>
      <td>Domain Renewal</td>
      <td>$12.00</td>
      <td><span class="badge error">Failed</span></td>
    </tr>
  </tbody>
</table>

```
## Native Responsiveness
Tables are notoriously hard to make responsive. Our .table class automatically changes the display property of the <table> element to block and applies overflow-x: auto.
This means the table handles its own horizontal scrolling on mobile devices **without requiring a wrapper <div>**. Less HTML, fewer wrappers, mathematically perfect padding via fluid variables.
## Customizing The Axiom Way
Want a specific table to have a "compact" view for dense dashboards? Don't write 20 new utility classes. Adjust the spacing tokens for that specific table context:
```html
<table class="table" style="--a-space-md: 0.5rem; --a-space-sm: 0.25rem;">
  <!-- Your compact table content here -->
</table>

```
