📋 VIOLATION DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1️⃣  INLINE STYLES (50+ pages)
      Example: <div style="margin: var(--a-space-m) 0;">
      Problem: Breaks CSS theming, couples markup to presentation
      Impact:  Dark mode harder, hard to maintain, anti-pattern
      Fix:     Move to CSS classes or semantic elements

  2️⃣  CUSTOM CLASSES (83/83 pages)
      Examples: .form-demo, .github, .menu, .search, .button-group
      Problem: Not part of framework, violates stated philosophy
      Impact:  Confusing for developers, maintenance burden
      Fix:     Use only core framework classes

  3️⃣  DIVITIS (15 pages)
      Example: <div class="nav"><div class="link">...</div></div>
      Problem: Should use semantic <nav>, <ul>, <li>
      Impact:  Accessibility issue, harder for screen readers
      Fix:     Replace with semantic HTML elements

  4️⃣  DIV ROLES (10+ pages)
      Example: <div role="navigation">
      Problem: Should use semantic <nav> instead
      Impact:  ARIA is fallback, not replacement; A11y compromise
      Fix:     Replace div[role] with semantic elements

  5️⃣  FRAMEWORK CONFUSION (index.html)
      Examples: class="max-w-sm", class="delay-3", class="border"
      Problem: Mixing Tailwind/Bootstrap with Axiom01
      Impact:  Developers don't know what's framework
      Fix:     Use only Axiom01 classes or move to CSS
