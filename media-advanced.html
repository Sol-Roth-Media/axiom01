Optimizing Images for the Web

Images are crucial for engaging user experiences, but they can significantly impact performance. Axiom encourages best practices for image optimization and responsive delivery.

Responsive Images with srcset and sizes

Ensure your images look great and load fast on any device by providing multiple image sources for different screen sizes and resolutions.

<img
    src="img/axiom.jpeg"
    srcset="img/axiom-small.jpeg 480w, img/axiom.jpeg 800w"
    sizes="(max-width: 600px) 480px, 800px"
    alt="Axiom Logo Responsive"
    class="responsive-image"
>

Lazy Loading Images

Improve initial page load times by deferring the loading of off-screen images until they are needed.

<img src="img/axiom.jpeg" alt="Axiom Logo" loading="lazy">

Simply add the loading="lazy" attribute to your <img> tags.

Modern Image Formats (WebP, AVIF)

Utilize modern image formats like WebP and AVIF for superior compression and quality. Use the <picture> element for graceful degradation.

<picture>
    <source srcset="img/axiom.webp" type="image/webp">
    <img src="img/axiom.jpeg" alt="Axiom Logo WebP">
</picture>

Icons & SVGs: Scalable and Accessible

Icons are essential for visual communication. Axiom supports both icon fonts and inline SVGs, with a focus on scalability, performance, and accessibility.

Using Icon Fonts (e.g., Font Awesome)

Icon fonts are easy to use and style with CSS. Axiom integrates well with popular icon libraries.

<i class="fas fa-star" aria-hidden="true"></i> Star Icon
<span class="sr-only">Star Icon</span>

Remember to use aria-hidden="true" for decorative icons and provide a screen-reader-only text alternative for meaningful icons.

Inline SVGs

For ultimate control over styling and animation, inline SVGs are a powerful choice. They are resolution-independent and can be manipulated with CSS and JavaScript.

<svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">

    <line x1="8" y1="12" x2="16" y2="12"></line>

Accessibility for Icons

Always consider accessibility when using icons. For purely decorative icons, use aria-hidden="true". For interactive icons or icons conveying meaning, provide alternative text using <span class="sr-only"> or appropriate ARIA attributes.

Customizing Media with CSS Variables

Axiom provides CSS variables to control aspects of media, such as border-radius for images or default icon sizes.

:root {
    /* Image Styling */
    --a-image-border-radius: var(--a-border-radius-s); /* Default border-radius for images */
    --a-image-max-width: 100%; /* Default max-width for responsive images */

    /* Icon Styling */
    --a-icon-size-s: 1rem;
    --a-icon-size-m: 1.5rem;
    --a-icon-size-l: 2rem;
    --a-icon-color: var(--a-color-on-surface); /* Default icon color */
}

These variables allow for consistent styling of media elements across your application.

Media Accessibility

Ensuring media is accessible is vital for users with visual impairments, cognitive disabilities, or those using assistive technologies.

Alternative Text for Images (alt attribute)

Every meaningful image must have a concise and descriptive alt attribute. This text is read by screen readers and displayed if the image fails to load. For decorative images, use an empty alt="" to hide them from screen readers.

<img src="photo.jpg" alt="A golden retriever playing in a field of flowers.">
<img src="decorative-line.svg" alt="">

Captions and Transcripts for Video/Audio

For video content, provide captions (for dialogue and important sounds) and transcripts (full text of all spoken content and descriptions of visual information). For audio content, provide transcripts.

Contrast for Overlays and Text on Images

If you place text over an image, ensure there is sufficient color contrast between the text and the background image, especially as the image changes due to responsiveness or user interaction.

SVG Accessibility

For inline SVGs, use <title> and <desc> elements within the SVG for a descriptive name and longer description, respectively. Link them using aria-labelledby and aria-describedby. For decorative SVGs, use aria-hidden="true".

<svg role="img" aria-labelledby="svgTitle svgDesc"
    <title id="svgTitle">A smiling sun icon</title>
    <desc id="svgDesc">A yellow sun with rays, indicating good weather.</desc>
    <!-- SVG path data -->
</svg>
