# Axiom01 Developer Guide

This guide is for developers who want to contribute to the Axiom01 framework or understand its internal workings.

## Project Structure

-   `css/`: Contains the core Axiom01 CSS files (`axiom.css`) and site-specific styles (`site.css`).
-   `js/`: Contains the core Axiom01 JavaScript files (`axiom.js`) and site-specific scripts (`index-page-manager.js`).
-   `docs/`: Contains documentation files, including component examples and advanced guides.
-   `assets/images/`: Stores images used on the main page.

## Getting Started with Development

1.  **Clone the repository:**
    `git clone https://github.com/Sol-Roth-Media/axiom01.git`
2.  **Install dependencies:** (If any, currently none specified for front-end development beyond a browser)
3.  **Open `index.html`** in your browser to view the main page.
4.  **Make changes:** Modify HTML, CSS, or JavaScript files as needed.

## Building and Testing

Currently, Axiom01 is a pure front-end framework. Changes to HTML, CSS, and JavaScript can be tested directly in the browser.

## Future Development

-   **Build Process:** Implement a build process using tools like Webpack or Parcel for minification, transpilation, and asset optimization.
-   **Component Library:** Develop a more robust component library with dedicated JavaScript for interactive elements.
-   **Automated Testing:** Set up unit and integration tests for JavaScript components.
