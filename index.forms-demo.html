Accessibility Tips

  • Use aria-label for all form fields.
  • Group related fields with fieldset and legend.
  • Ensure all inputs have associated label elements.
  • Use required for mandatory fields.
  • Provide clear instructions and error messages.
  • Test forms with keyboard navigation and screen readers.
  • Use role="form" for complex forms to help assistive technologies.
  • Use autocomplete attributes for common fields (e.g., email, name).

Best Practices

  • Keep forms simple and focused.
  • Validate input on both client and server.
  • Use clear, concise labels and instructions.
  • Provide feedback for errors and successful submissions.
  • Support mobile and desktop layouts.

Media Component Demo

Accessibility Tips

  • Use alt text for all images.
  • Use aria-label for important media elements.
  • Ensure media content is keyboard and screen reader accessible.
  • Test with screen readers and keyboard navigation.

Best Practices

  • Use figure and figcaption for complex media blocks.
  • Combine images, icons, and text for richer content.
  • Use badges for status or type indication.
  • Provide captions for images and media when possible.
  • Use tabindex for custom interactive media elements.
  • Use tabindex="0" on custom interactive media elements to ensure keyboard accessibility.
  • Use role attributes for non-standard elements (e.g., role="img" for SVGs).
  • Consider aria-describedby for additional context or captions.
  • Use aria-live for dynamic media regions to announce updates to assistive technologies.
  • Test all interactive media with multiple browsers and assistive tech.
  • For video/audio, provide transcripts or captions for accessibility.
  • Use aria-hidden="true" for decorative images/icons.
  • For SVGs, use focusable="false" and aria-hidden="true" for decorative icons.
  • For carousels/slideshows, use aria-roledescription="carousel" and provide navigation controls with accessible labels.
  • For images/icons used as buttons or links, ensure aria-label or aria-labelledby describes their action.
  • For complex media, use aria-controls to associate controls with their media regions.
