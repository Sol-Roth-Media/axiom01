Accessibility Features

  • ARIA Live Regions: aria-live="polite" for most alerts, "assertive" for errors
  • Semantic Roles: role="alert" for immediate announcements
  • Icon Accessibility: Icons marked with aria-hidden="true"
  • Close Button Labels: Descriptive aria-label for screen readers
  • Keyboard Navigation: Close buttons accessible via Tab/Enter/Space
  • Focus Management: Focus returns to appropriate element after dismissal

Best Practices:

  • Use aria-live="assertive" for critical errors, "polite" for other alerts
  • Provide clear, actionable messages with specific next steps
  • Include close buttons for dismissible alerts with descriptive labels
  • Use semantic colors that align with message importance
  • Test alerts with screen readers to ensure proper announcements
  • Consider auto-dismiss timing for non-critical alerts
