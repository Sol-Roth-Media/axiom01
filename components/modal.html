Accessibility Features

  • Focus Management: Focus is trapped within the modal and returned to trigger element on close
  • Keyboard Navigation: Tab/Shift+Tab cycles through focusable elements, Escape closes modal
  • ARIA Attributes: role="dialog", aria-modal="true", aria-labelledby
  • Screen Reader Support: Proper heading structure, error announcements, live regions
  • Backdrop Dismiss: Click outside modal content to close (can be disabled for critical modals)
  • Visual Focus: Clear focus indicators within modal content

JavaScript Implementation

Modal functionality with comprehensive accessibility:

// Modal manager with focus trapping and accessibility
const AxiomModal = {
  activeModal: null,
  focusableSelectors: 'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])',

  open(modalId) {
    const modal = document.getElementById(modalId);
    if (!modal) return;

    this.activeModal = modal;
    this.previousFocus = document.activeElement;

    modal.classList.add('open');
    modal.setAttribute('aria-hidden', 'false');
    document.body.classList.add('modal-open');

    this.trapFocus(modal);
  },

  close() {
    if (!this.activeModal) return;

    this.activeModal.classList.remove('open');
    this.activeModal.setAttribute('aria-hidden', 'true');
    document.body.classList.remove('modal-open');

    if (this.previousFocus) {
      this.previousFocus.focus();
    }

    this.activeModal = null;
  }
};

Best Practices:

  • Always provide a clear, descriptive modal title
  • Include a visible close button with proper aria-label
  • Use appropriate button variants (primary, secondary, danger)
  • Test keyboard navigation and screen reader compatibility
  • Consider backdrop dismiss behavior for different modal types
  • Validate forms within modals with clear error messages
