Modal Component

Overview

The Modal component provides a dialog interface for user interactions, confirmations, and forms. It follows accessibility best practices and is styled with the axiom01 framework.

Example

Basic Modal: A modal dialog overlays the page and focuses user attention for important actions, confirmations, or forms. It is accessible, traps keyboard focus, and can be closed with Escape or by clicking outside. Use semantic markup and minimal classes for clarity and maintainability.

Example Modal

This is a modal dialog. It traps focus and can be closed with Escape or by clicking outside.

<button class="button" onclick="document.getElementById('demo-modal').classList.add('open')">Open Modal</button>
<div class="modal" id="demo-modal" role="dialog" aria-modal="true" aria-labelledby="modal-title" tabindex="-1">
  <div class="modal-content">
    <header>
      <h3 id="modal-title">Example Modal</h3>
      <button aria-label="Close" onclick="document.getElementById('demo-modal').classList.remove('open')">&times;</button>
    </header>
    <section>
      <p>This is a modal dialog. It traps focus and can be closed with Escape or by clicking outside.</p>
    </section>
    <footer>
      <button class="button" onclick="document.getElementById('demo-modal').classList.remove('open')">Close</button>
    </footer>
  </div>
</div>

Advanced Example

Advanced Modal: Modals can contain forms, multiple actions, and any semantic content. This example shows a contact form inside a modal, with Cancel and Send actions. All controls are accessible and keyboard-friendly.

Contact Form

<button class="button" onclick="document.getElementById('adv-modal').classList.add('open')">Open Advanced Modal</button>
<div class="modal" id="adv-modal" role="dialog" aria-modal="true" aria-labelledby="adv-modal-title" tabindex="-1">
  <div class="modal-content">
    <header>
      <h3 id="adv-modal-title">Contact Form</h3>
      <button aria-label="Close" onclick="document.getElementById('adv-modal').classList.remove('open')">&times;</button>
    </header>
    <section>
      <form>
        <label for="name">Name</label>
        <input id="name" name="name" type="text" required>
        <label for="email">Email</label>
        <input id="email" name="email" type="email" required>
        <label for="message">Message</label>
        <textarea id="message" name="message" required></textarea>
      </form>
    </section>
    <footer>
      <button class="button" onclick="document.getElementById('adv-modal').classList.remove('open')">Cancel</button>
      <button class="button" type="submit" form="contact-form">Send</button>
    </footer>
  </div>
</div>

Style Guide

  • Use .modal with role="dialog", aria-modal="true", and aria-labelledby for accessibility.
  • Trap keyboard focus within the modal when open.
  • Support closing via Escape key and clicking outside the modal.
  • Use semantic elements styled via .modal and element selectors, following Axiom01 philosophy.
  • Style with axiom01 classes for consistency.
