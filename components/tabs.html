Accessibility Features

  • ARIA Roles: Proper tablist, tab, and tabpanel roles
  • State Management: aria-selected indicates active tab
  • Keyboard Navigation: Arrow keys navigate tabs, Tab enters panel content
  • Focus Management: Focus remains on selected tab or moves into panel
  • Screen Reader Support: Proper label associations and panel announcements
  • Orientation Support: aria-orientation for vertical tab lists

Keyboard Interactions

  • Tab: Move focus into tab panel content
  • Shift + Tab: Move focus to previous tab in tab list
  • Left/Right Arrow (horizontal): Navigate between tabs
  • Up/Down Arrow (vertical): Navigate between tabs
  • Space or Enter: Activate focused tab
  • Home: Move focus to first tab
  • End: Move focus to last tab

CSS Implementation

Tabs styling using CSS variables and semantic selectors:

.tabs {
  display: flex;
  flex-direction: column;
}

.tabs.vertical {
  flex-direction: row;
}

.tab-list {
  display: flex;
  border-bottom: var(--a-border-width-base) solid var(--a-color-outline);
}

.tabs.vertical .tab-list {
  flex-direction: column;
  border-bottom: none;
  border-right: var(--a-border-width-base) solid var(--a-color-outline);
}

.tab {
  padding: var(--a-padding-small) var(--a-padding-medium);
  background: transparent;
  border: none;
  cursor: pointer;
  border-bottom: 2px solid transparent;
}

.tab[aria-selected="true"] {
  background: var(--a-color-primary-container);
  border-bottom-color: var(--a-color-primary);
}

.tab-panel {
  padding: var(--a-padding-medium);
  outline: none;
}

Best Practices:

  • Use semantic roles (tablist, tab, tabpanel)
  • Implement complete keyboard navigation patterns
  • Ensure only one tab can be selected at a time
  • Make tab panels focusable with tabindex="0"
  • Test navigation with keyboard-only users
  • Consider content organization and panel size
