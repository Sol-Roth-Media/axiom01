Card Component Demo

The Card component is used to group related content and actions. Cards are useful for dashboards, product previews, user profiles, and more. Use semantic elements for structure and simple classes for style.

Overview

The Card component is a flexible container for grouping related content and actions. Cards are ideal for dashboards, product previews, user profiles, and more. Use semantic elements (article, header, section, footer) for structure and simple classes for style. Always provide alt text for images and clear labels for actions to ensure accessibility.

Basic Card Example

A basic card uses semantic structure and a single action button.

Card Title

This is a simple card component. You can use it for content blocks, product previews, etc.

<article class="card">
  <header>
    <h3>Card Title</h3>
  </header>
  <section>
    <p>This is a simple card component.</p>
  </section>
  <footer>
    <button class="primary">Action</button>
  </footer>
</article>

Usage & Accessibility

  • Structure: Use <article class="card"> for the container, <header>, <section>, and <footer> for content.
  • Actions: Place buttons in the footer for card actions. Use clear, descriptive labels.
  • Images/Badges: Add images with alt text and badges for status or highlights.
  • Links/Icons: Add links or icons for richer content and navigation.
  • Accessibility: Always provide alt text for images and clear button labels.
