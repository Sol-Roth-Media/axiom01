Card Component

Cards group related content and actions using semantic HTML structure. Perfect for dashboards, product previews, user profiles, and content blocks.

Basic Card

Simple card structure using semantic elements with minimal classes.

Basic Card Title

This is the main content area of the card. Use semantic section elements to structure your content properly.

<article class="card">
  <header>
    <h3>Basic Card Title</h3>
  </header>
  <section>
    <p>This is the main content area of the card.</p>
  </section>
  <footer>
    <button class="primary">Primary Action</button>
  </footer>
</article>

Card with Image

Cards can include images with proper accessibility attributes.

Visual Content Card

Cards with images should always include descriptive alt text for accessibility. The image becomes part of the semantic structure.

<article class="card">
  <img src="image.jpg" alt="Descriptive alt text">
  <header>
    <h3>Visual Content Card</h3>
  </header>
  <section>
    <p>Content with proper image accessibility.</p>
  </section>
  <footer>
    <button class="secondary">View Details</button>
    <button class="primary">Take Action</button>
  </footer>
</article>

Accessibility Features

  • Semantic Structure: Uses <article> for the card container
  • Proper Headings: Clear heading hierarchy within cards
  • Image Accessibility: Descriptive alt text for all images
  • Action Labels: Clear button text and aria-labels when needed
  • Badge Context: ARIA labels for status indicators
  • Icon Handling: Decorative icons marked with aria-hidden="true"

CSS Implementation

Cards use element selectors within the card context:

.card {
  background: var(--a-color-surface);
  border-radius: var(--a-border-radius-base);
  box-shadow: var(--a-shadow-base);
}

.card header {
  padding: var(--a-padding-medium);
  border-bottom: var(--a-border-width-base) solid var(--a-color-outline);
}

.card section {
  padding: var(--a-padding-medium);
}

.card footer {
  padding: var(--a-padding-medium);
  display: flex;
  gap: var(--a-space-small);
  justify-content: flex-end;
}

Best Practices:

  • Use semantic <article> element for card containers
  • Structure content with <header>, <section>, <footer>
  • Provide descriptive alt text for all images
  • Use clear, actionable button text
  • Add ARIA labels for context when needed
  • Test keyboard navigation through card elements
