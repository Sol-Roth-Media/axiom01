Documentation Search

The Axiom01 documentation search lets you quickly find component demos, guides, and advanced usage examples. Type a keyword in the search bar (top navigation) and press Enter to see matching documentation pages.

How to Use

  1. Locate the Search docs... field in the top navigation bar.
  2. Type a keyword (e.g., "card", "forms", "theme", "navbar").
  3. Press Enter to view results in a floating box.
  4. Click a result to open the documentation page, or click Close to dismiss the results.

Feature Details

  • Client-side search: Fast, privacy-friendly, no server requests.
  • Extensible: Add more pages to docPages in js/scripts.js to expand coverage.
  • Accessibility: Keyboard and screen reader friendly, with ARIA labels and focus management.
  • Customizable: Style the search box and results via CSS for your theme.

Code Example

// Add more searchable pages in js/scripts.js
const docPages = [
  { title: 'Card Component', url: 'components/card.html' },
  { title: 'Forms Component', url: 'components/forms.html' },
  // ...more pages
];

// Search logic
form.addEventListener('submit', function(e) {
  e.preventDefault();
  const query = input.value.trim().toLowerCase();
  const results = docPages.filter(page => page.title.toLowerCase().includes(query));
  // ...show results
});

Advanced Usage

  • Implement fuzzy matching for more flexible search.
  • Integrate with external search APIs for larger docs.
  • Enhance accessibility with ARIA live regions and keyboard navigation.
  • Style results for dark/light themes using CSS variables.

Accessibility & Best Practices

  • Use role="search" and aria-label for forms.
  • Ensure keyboard navigation (Tab, Escape, Enter).
  • Use aria-modal and aria-labelledby for modals.
  • Provide clear focus states and error messages.
  • Style with CSS variables for theme consistency.

Advanced Usage

  • Implement fuzzy matching for flexible search (Fuse.js).
  • Integrate with external APIs for large documentation sets.
  • Live region updates for screen readers.
  • Custom result templates and highlighting.
