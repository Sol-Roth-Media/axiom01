Overview

The Navbar component provides site-wide navigation, supports dropdowns, jump menus, and adapts to mobile and desktop layouts. Use semantic <nav> and <ul> elements for structure, and minimal classes for styling. Always ensure links are clear and accessible.

Basic Navbar Example

A basic navbar uses a horizontal list of links for site navigation.

[Ax01]
  • Home
  • Features
  • Pricing
  • About
<nav class="navbar" aria-label="Main Navigation">
  <a href="index.html" class="logo" aria-label="Axiom Framework Home">[Ax<span style="color:var(--a-color-primary);">01</span>]</a>
  <ul class="nav-links">
    <li><a href="#home">Home</a></li>
    <li><a href="#features">Features</a></li>
    <li><a href="#pricing">Pricing</a></li>
    <li><a href="#about">About</a></li>
  </ul>
</nav>

Advanced Navbar Example

The advanced navbar supports dropdowns, jump menus, and a responsive toggle. Use <ul> and <li> for structure, .dropdown for dropdown menus, and .menu-toggle for mobile support.

[Ax01]
  • Home
  • Features
    • Feature 1
    • Feature 2
  • Pricing
  • About
<nav class="navbar" aria-label="Main Navigation">
  <a href="index.html" class="logo" aria-label="Axiom Framework Home">[Ax<span style="color:var(--a-color-primary);">01</span>]</a>
  <button class="menu-toggle" aria-label="Toggle Navigation" aria-expanded="false" aria-controls="nav-links-adv">
    <i class="fa-solid fa-bars" aria-hidden="true"></i>
  </button>
  <ul class="nav-links" id="nav-links-adv">
    <li><a href="#home">Home</a></li>
    <li class="dropdown">
      <a href="#features" aria-haspopup="true" aria-expanded="false">Features</a>
      <ul aria-label="submenu">
        <li><a href="#feature1">Feature 1</a></li>
        <li><a href="#feature2">Feature 2</a></li>
      </ul>
    </li>
    <li><a href="#pricing">Pricing</a></li>
    <li><a href="#about">About</a></li>
    <li>
      <select onchange="location = this.value;" aria-label="Jump Menu">
        <option value="#">Jump to...</option>
        <option value="#home">Home</option>
        <option value="#features">Features</option>
        <option value="#docs">Docs</option>
      </select>
    </li>
  </ul>
</nav>

Accessibility & Usage Tips

  • Use semantic <nav> and <ul> for navigation structure.
  • Ensure all links have clear, descriptive text.
  • Dropdowns should be keyboard accessible and use aria-haspopup and aria-expanded where appropriate.
  • Use minimal classes for styling and responsive layouts.

Usage

The Axiom01 Navbar is a responsive, accessible navigation component. It supports a mobile menu toggle, dropdowns, jump menus, and custom links. To use, include the following markup and ensure navbar.js is loaded:

<nav class="navbar" aria-label="Main Navigation">
  <a href="index.html" class="logo" aria-label="Axiom Framework Home">[Ax<span style="color:var(--a-color-primary);">01</span>]</a>
  <button class="menu-toggle" aria-label="Toggle Navigation" aria-expanded="false" aria-controls="nav-links">
    <i class="fa-solid fa-bars" aria-hidden="true"></i>
  </button>
  <ul class="nav-links" id="nav-links">
    <li><a href="index.html">Home</a></li>
    <li><a href="components/card.html">Card</a></li>
    <li><a href="components/forms.html">Forms</a></li>
    <li><a href="components/progress-bar.html">Progress Bar</a></li>
    <li><a href="components/navbar.html">Navbar</a></li>
    <li><a href="components/media.html">Media & File</a></li>
  </ul>
</nav>
<script src="js/navbar.js"></script>

Accessibility

  • Uses aria-label for navigation landmarks.
  • Menu toggle uses aria-expanded and aria-controls for screen readers.
  • Keyboard navigation is supported for all links and dropdowns.

Customization

  • Edit navbar.css to change colors, spacing, and layout.
  • Add or remove links in the nav-links list.
  • Use dropdown and jump menu patterns for advanced navigation.

Mobile & Responsive

  • The navbar automatically adapts to mobile screens. The .menu-toggle button appears on small screens and toggles the navigation menu.
  • Ensure navbar.js is loaded for toggle functionality.

Accessibility Best Practices

  • Use aria-label on <nav> for clear navigation landmarks.
  • Menu toggle button should have aria-expanded and aria-controls to indicate menu state.
  • Dropdown menus should use aria-haspopup and aria-expanded on the trigger link.
  • All interactive elements (links, buttons, dropdowns) must be reachable and operable via keyboard (Tab, Enter, Space).
  • Use aria-label or visible text for jump menus/selects.

Customizing Navbar

  • Edit navbar.css to change colors, spacing, and layout. Use CSS variables for easy color updates.
  • Modify the nav-links list to add, remove, or reorder links.
  • To enable dropdowns, use li.dropdown and nested <ul> elements. To disable, remove the .dropdown class and nested list.
  • Jump menus are enabled by adding a <select> element inside the navbar. Remove it to disable.
  • For layout changes, adjust flex/grid styles in navbar.css.

Enabling/Disabling Features

  • Dropdowns: Add li.dropdown and a nested <ul> for dropdown functionality. Remove these for a simple navbar.
  • Jump Menus: Add a <select> element for jump navigation. Remove to disable.
  • Mobile Toggle: The .menu-toggle button and navbar.js are required for responsive toggling.
