Why Semantic First?

For decades, web development has been dominated by class-based frameworks and BEM naming conventions. AXIOM01 v3 represents a rebellion against this complexity.

We believe that HTML should be meaningful, CSS should be simple, and frameworks should enable developers rather than constrain them.

The Three Principles

  1. Semantic HTML: Write meaningful markup where elements and class names describe WHAT they ARE, not HOW they look.
  2. Mathematical Design: Use OKLCH color space for perceptually uniform colors that automatically adapt to dark mode.
  3. Framework Responsibility: The framework handles CSS. You write HTML. No build tools required.
