Axiom01 is a semantic-first UI framework that prioritizes accessibility and clean markup. This section covers the basic principles and getting started guide.

  • Semantic HTML structure
  • Minimal, meaningful classes
  • CSS variables for theming
  • Built-in accessibility features

Each Axiom01 component follows a consistent pattern of semantic HTML, element-based CSS selectors, and accessibility-first design.

Components are built with:

  1. Semantic structure: Using appropriate HTML elements
  2. CSS variables: For consistent theming and customization
  3. ARIA attributes: For comprehensive screen reader support
  4. Keyboard navigation: Full keyboard accessibility

Axiom01 accordions include comprehensive accessibility features:

  • ARIA States: aria-expanded indicates panel state
  • Keyboard Navigation: Space/Enter to toggle, Arrow keys to navigate
  • Focus Management: Clear focus indicators and logical tab order
  • Screen Reader Support: Proper regions and label associations

Customize accordions using CSS variables and variant classes:

.accordion.compact {
  /* Reduced spacing */
}

.accordion.bordered {
  /* Visible borders between items */
}
<div class="accordion">
  <article class="accordion-item">
    <header>
      <button type="button" class="accordion-trigger"
              aria-expanded="false" aria-controls="panel-1" id="trigger-1">
        <span>Section Title</span>
        <span class="accordion-icon" aria-hidden="true">▼</span>
      </button>
    </header>
    <section id="panel-1" class="accordion-panel"
             role="region" aria-labelledby="trigger-1" hidden>
      <div class="accordion-content">
        Content goes here...
      </div>
    </section>
  </article>
</div>

Accessibility Features

  • ARIA States: aria-expanded indicates whether panels are open or closed
  • Keyboard Navigation: Space/Enter toggles panels, Up/Down arrows navigate between triggers
  • Focus Management: Clear focus indicators and logical tab order through accordion items
  • Screen Reader Support: Proper regions, headings, and label associations
  • Visual Indicators: Icons change to reflect panel state (open/closed)
  • Content Structure: Each panel has semantic content with proper heading hierarchy

Keyboard Interactions

  • Tab: Move focus to next accordion trigger
  • Shift + Tab: Move focus to previous accordion trigger
  • Space or Enter: Toggle the current panel (expand/collapse)
  • Down Arrow: Move focus to next accordion trigger
  • Up Arrow: Move focus to previous accordion trigger
  • Home: Move focus to first accordion trigger
  • End: Move focus to last accordion trigger

CSS Implementation

Accordion styling using CSS variables and semantic selectors:

.accordion {
  border-radius: var(--a-border-radius-base);
  overflow: hidden;
}

.accordion-item {
  border-bottom: var(--a-border-width-base) solid var(--a-color-outline);
}

.accordion-trigger {
  width: 100%;
  padding: var(--a-padding-medium);
  background: var(--a-color-surface);
  border: none;
  text-align: left;
  cursor: pointer;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.accordion-trigger:hover {
  background: var(--a-color-surface-variant);
}

.accordion-trigger:focus-visible {
  outline: var(--a-border-width-focus) solid var(--a-color-focus);
  outline-offset: calc(-1 * var(--a-border-width-focus));
}

.accordion-panel {
  overflow: hidden;
  transition: max-height 0.3s ease;
}

.accordion-content {
  padding: var(--a-padding-medium);
}

Best Practices:

  • Use semantic <article> elements for each accordion item
  • Provide clear, descriptive trigger text
  • Include visual indicators (icons) for panel state
  • Test keyboard navigation thoroughly
  • Ensure content within panels maintains proper heading hierarchy
  • Consider performance with many accordion items
