Button Component

Buttons trigger actions and navigation. Axiom01 buttons use semantic HTML with minimal, meaningful classes and comprehensive accessibility features.

Basic Buttons

Default button styles using semantic classes and proper focus states.

<button>Default Button</button>
<button class="primary">Primary Action</button>
<button class="secondary">Secondary Action</button>
<button class="danger">Destructive Action</button>

Button States

Buttons support disabled, loading, and active states with proper ARIA attributes.

<button disabled>Disabled Button</button>
<button class="primary" disabled>Disabled Primary</button>
<button class="primary" aria-pressed="true">Active Toggle</button>
<button class="secondary" aria-busy="true">Loading...</button>

Buttons with Icons

Use semantic icon markup with proper ARIA labels for accessibility.

<button class="primary">
  <span class="icon" aria-hidden="true">+</span>
  Add Item
</button>
<button aria-label="Settings">
  <span class="icon" aria-hidden="true">⚙</span>
</button>

Button Sizes

Size variants using CSS variables for consistent scaling.

<button class="small">Small Button</button>
<button>Default Size</button>
<button class="large">Large Button</button>

Accessibility Features

  • Keyboard Navigation: All buttons are focusable with Tab and activated with Space/Enter
  • Focus Indicators: Clear visual focus states using :focus-visible
  • ARIA Support: Proper labels for icon-only buttons and state indicators
  • Screen Readers: Meaningful text content and appropriate roles

CSS Implementation

Buttons use semantic element selectors and CSS variables:

button {
  /* Base button styles using CSS variables */
  padding: var(--a-padding-small) var(--a-padding-medium);
  font-size: var(--a-font-size-base);
  border-radius: var(--a-border-radius-base);
}

button.primary {
  background-color: var(--a-color-primary);
  color: var(--a-color-on-primary);
}

button:focus-visible {
  outline: var(--a-border-width-focus) solid var(--a-color-focus);
  outline-offset: var(--a-space-tiny);
}

Best Practices:

  • Use semantic button text that describes the action
  • Provide aria-label for icon-only buttons
  • Use aria-pressed for toggle buttons
  • Disable buttons during async operations with aria-busy="true"
  • Test keyboard navigation and screen reader compatibility
