Axiom01 Style Guide & Aesthetic Theme

This guide demonstrates how to create visually consistent, beautiful, and responsive layouts using Axiom01. Learn to use semantic markup, variable-driven spacing, scalable typography, and advanced theming for a modern, aesthetic UI.

Content Flow & Layout Principles

Use semantic HTML elements (<header>, <section>, <article>, <footer>) to structure your content. Axiom01's grid and flex utilities, combined with variable-driven gaps, ensure even flow and spacing.

Use .card header, .card section, and .card footer for clean, maintainable styles.

Grid layouts use display: grid and gap: var(--a-grid-gutter) for even spacing. Flex layouts use gap and align-items for vertical rhythm.

Typography & Scaled Font Sizes

Typography is managed with variables for font size, weight, and line height. Scale headings and text responsively using var(--a-font-size-h1) to var(--a-font-size-small).

Spacing: Padding, Margin, & Gaps

All spacing is managed with variables for consistency and easy theme switching. Use var(--a-padding-*), var(--a-margin-*), and var(--a-grid-gutter) for layout rhythm.

Large Padding
Small Padding
  • Use var(--a-padding-small), var(--a-padding-large) for inner spacing.
  • Use var(--a-margin-base), var(--a-margin-xlarge) for outer spacing.
  • Grid/flex gaps: var(--a-grid-gutter) for even columns/rows.

Component Composition & Patterns

Compose components with semantic elements and minimal classes. Use parent class + element selectors for styling. Example patterns:

Modal Example
Modal

Use .modal with ARIA roles, focus management, and keyboard navigation.

Notification Example
Toast

Use .notification.success, .notification.error for type-based styling and auto-dismiss.

  • Use ARIA roles/attributes for accessibility.
  • Combine classes for variants (e.g., .alert.success).
  • Enhance native elements (e.g., input[type='date'] for datepicker).
  • Modal: Use .modal with role="dialog", aria-modal="true", aria-labelledby. Ensure keyboard navigation (Escape to close, Tab trap), focus management, and click-outside-to-close. See js/axiom.js for details.
  • Notification: Use .notification or AxiomNotify with role="status" or role="alert". Support dismissible notifications, keyboard navigation, and clear visual indicators. Avoid auto-dismiss for important messages.
  • Datepicker: Enhance input[type='date'] with .axiom-datepicker for accessibility. Use ARIA labels, keyboard navigation, and a calendar grid. See js/axiom.js for details.

Advanced Theming: Beyond Colors

Axiom01 themes can change more than colors: update radii, shadows, gradients, surface effects, and spacing for a unique look. Example: the Aesthetic theme below.

Aesthetic Theme Demo

Gradients, large radii, soft shadows, and surface effects create a modern, aesthetic feel.

Surface Variant

Use var(--a-color-surface-variant) for cards, modals, and overlays.

  • Change border radius for softer or sharper corners.
  • Update shadows for depth and elevation.
  • Use gradients for backgrounds and accents.
  • Adjust surface colors for cards, modals, and overlays.
  • Scale spacing for a more open or compact feel.

Switch themes by changing data-theme="aesthetic" on <html>. All variables update automatically.

Advanced Theming & Customization

Axiom01 supports deep customization using CSS variables. You can create custom themes, gradients, and layout variations by overriding variables in your CSS or using the data-theme attribute.

Custom Theme Example

html[data-theme='my-custom-theme'], .my-custom-theme {
  --a-color-primary-hue: 340;
  --a-color-secondary-hue: 260;
  --a-saturation-primary: 75%;
  --a-brightness-primary: 50%;
  --a-color-surface: hsl(340, 15%, 10%);
  --a-color-on-surface: hsl(340, 10%, 95%);
  --a-space-base: 18px;
  --a-border-radius-base: 8px;
  --a-gradient-primary: linear-gradient(135deg, hsl(340, 75%, 50%), hsl(260, 70%, 50%));
}

Best Practices

  • Use semantic HTML for structure (<header>, <section>, <footer>, etc.)
  • Minimize class usage; prefer element selectors and CSS variables
  • Override variables for theming instead of writing new classes
  • Use gradients and spacing variables for visual interest and consistency
