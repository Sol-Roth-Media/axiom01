New Axiom01 Components

Examples of new components that extend the Axiom01 UI framework.

Modal Component

The Modal component provides a way to display content in a layer that sits on top of the main page content. It's useful for dialogs, lightboxes, and other content that should appear in front of the main content.

Basic Usage

To create a modal, you need:

  1. A trigger element with data-modal-target attribute pointing to the modal's ID
  2. A modal container with the corresponding ID
  3. Close buttons with data-modal-close attribute
<!-- Modal Trigger -->
<button data-modal-target="example-modal">Open Modal</button>

<!-- Modal Container -->
<div id="example-modal" class="modal" aria-hidden="true">
    <div>
        <header>
            <h3>Modal Title</h3>
            <button class="modal-close" data-modal-close aria-label="Close">×</button>
        </header>
        <section>
            <p>Modal content goes here...</p>
        </section>
        <footer>
            <button class="button" data-modal-close>Cancel</button>
            <button class="button primary">Confirm</button>
        </footer>
    </div>
</div>

Accessibility Features

The Modal component includes several accessibility features:

  • Proper ARIA attributes (role="dialog", aria-modal="true", etc.)
  • Focus is trapped within the modal when open
  • Focus returns to the trigger element when closed
  • Escape key closes the modal
  • Background scrolling is prevented when modal is open

Notification System

The Notification System provides a way to display toast notifications to users. It supports different types of notifications (success, error, info, warning) and can be configured with various options.

Basic Usage

The Notification System is available globally as AxiomNotify:

// Show a basic notification
AxiomNotify.show('This is a notification message');

// Show a success notification
AxiomNotify.success('Operation completed successfully!');

// Show an error notification
AxiomNotify.error('An error occurred!');

// Show a warning notification
AxiomNotify.warning('Warning: This action cannot be undone.');

// Show an info notification
AxiomNotify.info('Here is some information for you.');

// Show a notification with custom options
AxiomNotify.show('Custom notification', {
    type: 'success',           // info, success, error, warning
    duration: 3000,            // ms, 0 for no auto-dismiss
    dismissible: true,         // whether user can dismiss with a close button
    position: 'top-right',     // top-right, top-left, bottom-right, bottom-left
    icon: true                 // whether to show an icon
});

  • Visual indicators for different notification types

Infinite Scroll Component

The Infinite Scroll component automatically loads more content as the user scrolls near the bottom of a container. It is accessible, unobtrusive, and works with any semantic container (e.g., <div>, <main>, <section>, <article>).

Basic Usage

  1. Add the axiom-infinite-scroll class to any scrollable container.
  2. Optionally set data-infinite-scroll-callback="functionName" to call a global function when triggered.
  3. Or listen for the axiom:infinite-scroll event on the container.
  4. Adjust the trigger threshold with data-infinite-scroll-threshold="value" (pixels from bottom).
<div class="axiom-infinite-scroll" style="height: 300px; overflow-y: auto;">
    <!-- Content items here -->
</div>
// Listen for the event
const container = document.querySelector('.axiom-infinite-scroll');
container.addEventListener('axiom:infinite-scroll', function() {
    // Load more content here
});

// Or use a callback
function loadMoreContent(container) {
    // Load more content here
}
// In HTML: data-infinite-scroll-callback="loadMoreContent"
