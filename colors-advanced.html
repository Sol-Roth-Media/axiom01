Advanced Use of Colors & Gradients

Axiom01 provides a rich set of color variables, containers, gradients, and greys to help you build beautiful, high-contrast sites with minimal effort. This guide covers advanced usage patterns for color design tokens, including containers, on-container, and gradients.

Use container variables to create backgrounds and surfaces that work with your primary, secondary, and accent colors. The on-container variables ensure text and icons have optimal contrast on any container.

Semantic Color Swatches

Axiom provides semantic color tokens for common UI states and elements, such as success, error, warning, info, muted, outline, and surface. Use these for alerts, notifications, borders, backgrounds, and text to ensure consistent meaning and accessibility across your site.

Positive (Success) var(--a-color-positive)
Negative (Error) var(--a-color-negative)
Warning var(--a-color-warning)
Info var(--a-color-info)
Muted var(--a-color-muted)
Outline var(--a-color-outline)
Surface var(--a-color-surface)
  • Positive for success states, confirmations, and positive feedback.
  • Negative for errors, destructive actions, and negative feedback.
  • Warning for cautions, alerts, and attention areas.
  • Info for informational messages and highlights.
  • Muted for secondary text, icons, and subtle UI elements.
  • Outline for borders, dividers, and subtle separation.
  • Surface for backgrounds, cards, and containers.

Use these semantic colors in your CSS for instant, meaningful styling. For example:

.alert-success {
  background: var(--a-color-positive);
  color: var(--a-color-on-surface);
}
.alert-error {
  background: var(--a-color-negative);
  color: var(--a-color-on-surface);
}
.card-muted {
  background: var(--a-color-muted);
  color: var(--a-color-on-surface);
}

Practical Color Scheme Tips

Practical Color Scheme Tips: Use axiom01 color variables and utility classes to quickly create a variety of color schemes for your site. Below are practical tips and examples for each style. For more, see the Axiom01 Style Guide.

Black & White
  • Use var(--a-color-surface) and var(--a-color-on-surface) for high-contrast layouts.
  • Apply .bg-surface and .text-on-surface classes for instant B&W styling.
  • Accessibility: Ensure text contrast ratio is at least 4.5:1 for body text.
.bg-surface { background: var(--a-color-surface); }
.text-on-surface { color: var(--a-color-on-surface); }
Subtle
  • Use lighter greys (--a-color-gray-2 to --a-color-gray-8) for backgrounds.
  • Accent with muted colors: var(--a-color-muted), .bg-muted.
  • Accessibility: Use subtle backgrounds for cards and sections, but keep text dark for readability.
.bg-muted { background: var(--a-color-muted); }
Bold
  • Use var(--a-color-primary), var(--a-color-secondary) for strong accents.
  • Apply .bg-primary, .bg-secondary for bold backgrounds.
  • Accessibility: Use bold colors for call-to-action buttons and highlights.
.bg-primary { background: var(--a-color-primary); }
.bg-secondary { background: var(--a-color-secondary); }
Punchy
  • Use punchy accent tokens: var(--a-color-punchy-1), var(--a-color-punchy-2).
  • Try .bg-punchy-1, .bg-punchy-2 for vibrant sections.
  • Accessibility: Use punchy colors sparingly to avoid overwhelming users.
.bg-punchy-1 { background: var(--a-color-punchy-1); }
.bg-punchy-2 { background: var(--a-color-punchy-2); }
Pastel
  • Use pastel tokens: var(--a-color-pastel-1), var(--a-color-pastel-2).
  • Apply .bg-pastel-1, .bg-pastel-2 for soft, gentle backgrounds.
  • Accessibility: Pastel backgrounds work well for onboarding and info cards.
.bg-pastel-1 { background: var(--a-color-pastel-1); }
.bg-pastel-2 { background: var(--a-color-pastel-2); }
