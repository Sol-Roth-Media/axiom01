## ✨ Key Features

- **Semantic HTML Only** - No div hacks, proper elements
- **One Class Per Component** - `.card` not `.card-header` + `.card-body`
- **Design Tokens Only** - CSS variables for all values
- **100% Accessible** - WCAG 2.1 AA compliance built-in
- **No Utility Classes** - Semantic-first approach
- **Dark Mode Built-in** - Automatic theme support
- **Keyboard Navigable** - All components keyboard accessible
- **Responsive by Default** - Fluid, mobile-first design
- **Zero Dependencies** - Vanilla JavaScript only
- **Production Ready** - 77 tested components

## ❓ Frequently Asked Questions

### How do I get started?
See [Installation Guide](./INSTALLATION.md)

### Can I use AXIOM01 with a framework like React or Vue?
Yes! AXIOM01 is framework-agnostic CSS + vanilla JS. Mix and match with any framework.

### How do I customize colors?
See [Theming section](./INSTALLATION.md#theming)

### Is AXIOM01 accessible?
Yes, WCAG 2.1 Level AA compliance is built-in. See [Accessibility Guide](./INSTALLATION.md#accessibility-wcag-21-aa)

### Can I contribute components?
Yes! Read [Contributing Guide](./CONTRIBUTING.md)

### How do I report bugs?
Open an issue on [GitHub](https://github.com/Sol-Roth-Media/axiom01/issues)

### What browsers are supported?
Modern browsers with ES6+ support: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+

### Is there TypeScript support?
Not yet. Coming in a future release. Core JS uses JSDoc comments.

### Can I use AXIOM01 commercially?
Yes! MIT license allows commercial use.
