Code Snippet Conventions

This page defines the canonical way to add code snippets in Axiom01 documentation so snippets render consistently, stay readable in all themes, and optionally gain Prism syntax highlighting.

Recommended Markup

Use semantic HTML and keep snippets inside a figure with a figcaption when the language or purpose needs a label.

If no caption is needed, plain <pre><code> is valid and styled consistently by css/doc-styles.css.

Uniform Rules For Docs Pages

  • Always keep snippet text inside a code element nested in pre.
  • Prefer a Prism language class such as language-html, language-css, language-javascript, or language-bash.
  • Escape example markup as text (for example &lt;section&gt;) so it is displayed, not executed.
  • Do not manually add token spans; Prism generates token markup when loaded.
  • Avoid inline styles for snippets. Rely on shared snippet styles from css/doc-styles.css.

Prism Integration Strategy

Axiom01 should keep Prism as an optional progressive enhancement. Baseline readability must not depend on Prism, and syntax coloring should layer on top when Prism is present.

Use Prism when a page benefits from syntax coloring (API-heavy docs, long snippets, mixed languages). For short one-liners, plain snippets are usually enough.

Author Checklist

  • Add snippet with pre > code, optionally wrapped by figure.code-block.
  • Add a Prism language class when practical.
  • Verify readability in both light and dark themes.
  • Confirm horizontal scrolling works for long lines.
  • Run repository checks before handoff.
