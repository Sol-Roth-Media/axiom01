Typography

Axiom01's typography system creates hierarchy, readability, and visual rhythm across your interface.

Typography Philosophy

Typography is more than just choosing fonts—it's about creating a harmonious scale that guides users through your content. Axiom01 uses a mathematically scaled system with:

  • Perfect Vertical Rhythm - All line heights and margins align to a baseline grid
  • Optimal Reading Width - Lines automatically constrain to 65 characters for comfortable reading
  • Semantic Scaling - Heading sizes (h1-h6) scale proportionally
  • Font System - Three font families with sensible defaults and customization
  • Weight Hierarchy - Consistent font weights for emphasis and hierarchy

Font Families

Axiom01 includes three carefully selected font families. These can be switched dynamically using data attributes.

Default: Inter (Sans Serif)

Inter is a highly legible, modern sans-serif designed specifically for screen use. It's the default for both headings and body text. 

--ax-font-body: Inter, system-ui, -apple-system, "Segoe UI", sans-serif;
--ax-font-heading: Inter, system-ui, -apple-system, "Segoe UI", sans-serif;

Serif: Lora

Lora is a contemporary serif typeface optimized for screen and print. Use for editorial or formal contexts. 

--ax-font-body: "Lora", Georgia, serif;
--ax-font-heading: "Lora", Georgia, serif;

Monospace: JetBrains Mono

JetBrains Mono is a professional monospace font perfect for code, terminal text, and technical content. 

--a-font-family-mono: "JetBrains Mono", SFMono-Regular, Menlo, Monaco, monospace;

Switching Fonts

<!-- Default (Inter) -->
<html data-font-body="system" data-font-heading="system">

<!-- Serif (Lora) -->
<html data-font-body="serif" data-font-heading="serif">

<!-- Monospace -->
<html data-font-body="mono">

Heading Scale

All headings follow a mathematical scale (1.15x) for perfect proportions. Each heading has a corresponding CSS variable.

Heading 1 (48px)

var(--a-font-size-h1) • Used for page titles

Heading 2 (36px)

var(--a-font-size-h2) • Major section headings

Heading 3 (24px)

var(--a-font-size-h3) • Subsection headings

Heading 4 (18px)

var(--a-font-size-h4) • Minor headings

Heading 5 (16px)

var(--a-font-size-h5) • Small headings

Heading 6 (14px)

var(--a-font-size-h6) • Extra small headings

CSS Reference

--a-font-size-h1: 3rem;     /* 48px */
--a-font-size-h2: 2.25rem;  /* 36px */
--a-font-size-h3: 1.5rem;   /* 24px */
--a-font-size-h4: 1.125rem; /* 18px */
--a-font-size-h5: 1rem;     /* 16px */
--a-font-size-h6: 0.875rem; /* 14px */

Line Height & Vertical Rhythm

Line height is crucial for readability. Axiom01 uses a mathematical system where all line heights are proportional.

Tight Line Height (1.2)
This is body text with tight line height, useful for headings or when space is limited.

--a-line-height-tight

Heading Line Height (1.3)
This is applied to all h1-h6 elements for optimal heading appearance.

--a-line-height-heading

Base Line Height (1.6)
Default body text line height for comfortable reading across paragraphs and longer text blocks.

--a-line-height-base

Loose Line Height (2.0)
Use for emphasized or highlighted text where extra breathing room helps readability.

--a-line-height-loose

Why Vertical Rhythm Matters

When line heights, spacing, and font sizes follow a mathematical scale, content feels organized and professional. Axiom01 automatically manages this through design tokens.

Reading Width

Axiom01 automatically constrains paragraph width to 65 characters for optimal readability. This prevents eye strain from lines that are too long.

This paragraph is constrained to 65 characters width (--ax-reading-width). Notice how it naturally breaks lines at comfortable intervals. This width is optimal for reading without having to track your eyes across long distances, which causes fatigue.

Reading Width Variants

<!-- Narrow: 60 characters -->
<html data-layout-width="narrow">

<!-- Standard: 65 characters (default) -->
<html data-layout-width="standard">

<!-- Wide: 70 characters -->
<html data-layout-width="wide">

Text Hierarchy & Emphasis

Use semantic HTML elements combined with CSS variables to create visual hierarchy:

Strong/Bold: <strong> or <b> for important content

Italic/Emphasis: <em> or <i> for emphasized content

Highlighted: <mark> for highlighted or marked text

Code: <code> for inline code samples

Muted Text: .text-muted for secondary information

Text Utilities

<!-- Primary text (default) -->
<p class="text-primary">Primary content</p>

<!-- Secondary text -->
<p class="text-secondary">Secondary information</p>

<!-- Muted text -->
<p class="text-muted">Supplementary or disabled content</p>

<!-- Success -->
<p class="success">Success message</p>

<!-- Error -->
<p class="error">Error message</p>
