# Contributing to Axiom01

We welcome contributions to Axiom01! By contributing, you're helping to make this framework better for everyone.

## How to Contribute

1.  **Fork the repository.**
2.  **Create a new branch** for your feature or bug fix: `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/issue-description`.
3.  **Triage your change scope** into one primary bucket before implementation:
    - `bug`
    - `docs`
    - `component-parity`
    - `drupal-integration`
4.  **Make your changes.** Ensure your code adheres to the Axiom01 Styling Guide and best practices.
5.  **Run the quality lane:** `npm test`
6.  **Run release build verification:** `npm run build`
7.  **Check definition of done** in `docs/MAINTENANCE_WORKFLOW.md`.
8.  **Commit your changes** with a clear and descriptive commit message.
9.  **Push your branch** to your forked repository.
10. **Open a Pull Request** to the `main` branch of the original Axiom01 repository.

## Code Style

Please follow the guidelines outlined in the `AXIOM01_STYLING_GUIDE.md` for HTML, CSS, and JavaScript.

## Reporting Bugs

If you find a bug, please open an issue on GitHub with a clear description, steps to reproduce, and expected behavior.

## Feature Requests

We'd love to hear your ideas! Open an issue on GitHub to propose new features or improvements.

## Maintenance references

- `docs/MAJOR_RELEASE_SCOPE.md` for core-supported vs experimental/deferred scope.
- `docs/MAINTENANCE_WORKFLOW.md` for triage rubric, DoD checklists, and cadence.
- `starter-themes/axiom01-drupal11-starter-theme/` for Drupal 11 integration scaffold.

Thank you for contributing!
