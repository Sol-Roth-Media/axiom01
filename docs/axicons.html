Three Icon Styles: Regular, Thin & Inverted

Axicons includes three distinct styles of every icon, giving you complete visual flexibility for any design need. See the difference below:

Regular (Bold)

2.5px strokes • Prominent • Primary UI

Thin (Elegant)

0.5px strokes • Refined • Dense layouts

Inverted (Modern)

Solid + cutout • Modern • Badges

Regular Icons (Bold)

2.5px strokes for prominence and visibility. Use for primary navigation, action buttons, CTAs, and anywhere icons need to stand out and be easily tappable.

Thin Icons (Elegant)

0.5px strokes with simplified details. Perfect for data-dense layouts, dashboards, tables, and designs that need refinement and subtlety without sacrificing clarity.

Inverted Icons (Modern)

Solid background with white cutout symbol. High contrast and modern aesthetic. Perfect for badges, status indicators, highlights, and featured content that demands visual impact.

Why Three Styles?

  • Regular: CSS can't change SVG strokes easily - bold variants solve this
  • Thin: Some layouts need delicate, refined icons - no CSS hacks required
  • Inverted: Modern design trend - solid backgrounds with negative space symbols
  • Complete: All three means no workarounds, no custom modifications, just semantic naming

<div>
    <!-- Primary action: Regular -->
    <button><span class="axicon render" data-name="Settings"></span> Configure</button>
    
    <!-- Secondary action: Thin -->
    <a href="#"><span class="axicon render" data-name="HelpThin"></span> Help</a>

Design Tip: Use Regular for interactive elements, Thin for informational content, and Inverted for status/badges. This creates natural visual hierarchy and guides user attention.

Using Axicons with Axiom01

Axicons integrate seamlessly with the Axiom01 framework. They use the same design tokens, follow the same philosophy, and work beautifully out of the box.

1. Direct HTML Usage

Use the axicon class with semantic naming:

<span class="axicon render" data-name="ChatGPT"></span>
<span class="axicon render" data-name="Smartwatch"></span>
<span class="axicon render" data-name="Docker"></span>

2. With Axiom Components

Icons work naturally within Axiom's semantic components:

<button>
    <span class="axicon render" data-name="Settings"></span>
    Configure

<a href="#">

    Fork Repository
</a>

3. Styling with Design Tokens

Icons automatically use Axiom's color system via design tokens:

/* Icons inherit from current text color */
.axicon {
    width: var(--a-space-l);
    height: var(--a-space-l);
    stroke: currentColor;
    fill: none;
}

/* Size variants using tokens */

    width: var(--a-space-m);
    height: var(--a-space-m);
}

.axicon.large {
    width: var(--a-space-xl);
    height: var(--a-space-xl);

/* Color tokens */

    stroke: var(--a-color-primary);
}

.success {
    stroke: var(--a-color-success);

    stroke: var(--a-color-error);
}

4. JavaScript Integration

Access icons programmatically in Axiom components:

// Find icon by name
const icon = axicons.find(i => i.name === 'ChatGPT');

// Render icon dynamically
function createIcon(name, size = 'md') {
    const icon = axicons.find(i => i.name === name);
    if (!icon) return null;
    
    const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
    svg.setAttribute('viewBox', '0 0 24 24');
    svg.setAttribute('class', `axicon ${size}`);
    svg.innerHTML = icon.svgContent;
    
    return svg;
}

// Usage
const element = document.getElementById('icon-container');
element.appendChild(createIcon('Smartwatch', 'large'));

Using Axicons Independently

Axicons can be used standalone, outside of Axiom01. They're framework-agnostic and have zero dependencies.

1. Standalone HTML

<!-- Include the axicons library -->
<script src="axicons.js"></script>

<!-- Use with any HTML -->
<div id="icon-container"></div>

<script>
// Find and render an icon
const icon = axicons.find(i => i.name === 'Docker');
const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
svg.setAttribute('viewBox', '0 0 24 24');
svg.innerHTML = icon.svgContent;
document.getElementById('icon-container').appendChild(svg);
</script>

2. React Integration

import { axicons } from './axicons.js';

function Icon({ name, size = 24 }) {
    const icon = axicons.find(i => i.name === name);
    
    return (
        <svg 
            viewBox="0 0 24 24" 
            width={size} 
            height={size}
            dangerouslySetInnerHTML={{ __html: icon.svgContent }}
        />
    );
}

// Usage
export function App() {
    return (
        <div>
            <Icon name="ChatGPT" size={48} />
            <Icon name="Docker" size={32} />
            <Icon name="Smartwatch" size={24} />
        </div>
    );
}

3. Vue Integration

<template>
    <svg 
        viewBox="0 0 24 24" 
        :width="size" 
        :height="size"
        v-html="iconContent"
    />
</template>

<script>
import { axicons } from '@/assets/axicons.js';

export default {
    props: {
        name: String,
        size: {
            type: Number,
            default: 24
        }
    },
    computed: {
        iconContent() {
            const icon = axicons.find(i => i.name === this.name);
            return icon ? icon.svgContent : '';
        }
    }
}
</script>

<!-- Usage -->
<Icon name="ChatGPT" :size="48" />
<Icon name="Docker" :size="32" />
