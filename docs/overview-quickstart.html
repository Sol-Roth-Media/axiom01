Overview & Quickstart

Beautiful, accessible interfaces in minutes. Semantic HTML, not utility class chaos.

What is Axiom01?

Axiom01 is a semantic-first UI framework that turns clean HTML into beautiful, accessible interfaces. No utility classes. No component naming chaos. Just write meaningful markup and let CSS handle the styling.

68 Components

Everything from buttons to video players. Production-ready, accessible, and customizable.

149 Design Tokens

Colors, spacing, typography, shadows. Change one variable, your entire site transforms.

13.3 KB CSS

Gzipped. Includes every component. Zero dependencies. Production-ready.

Why Axiom01 is Different

Most frameworks force you to choose between clean code and beautiful design. Axiom01 gives you both.

Semantic HTML

Write meaningful markup using <article>, <section>, <header>. Axiom understands your structure.

One Class Per Component

class="card" instead of class="flex rounded shadow p-6 bg-white". Minimal, expressive, readable.

Accessible by Default

WCAG 2.1 AA compliance built-in. Keyboard navigation, focus states, contrast — all automatic.

Installation

Get Axiom01 running in your project in two ways.

Option 1: npm Recommended

Install via npm, yarn, or pnpm:

npm install axiom01

Import in your CSS:

@import 'axiom01/css/axiom.min.css';

Option 2: CDN

Add directly to your HTML <head>:

<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/axiom01/css/axiom.min.css">
<script src="https://cdn.jsdelivr.net/npm/axiom01/js/axicons.js"></script>
No build step. No configuration. No dependencies. Start using components immediately.

Your First Components

Buttons

Semantic buttons use a single variant class on the <button> element. When you style a link as a button, add the shared .button root and the variant class.

<button class="primary">Save Changes</button>
<button class="secondary">Cancel</button>
<button class="danger">Delete</button>

Cards

Structured content containers using semantic child elements — no sub-classes:

<article class="card">
  <header>
    <h3>Card Title</h3>
  </header>
  <p>Your content here.</p>
  <footer>
    <button class="primary">Action</button>
  </footer>
</article>

Forms

Fully accessible form controls with consistent styling — no extra classes needed:

<form>
  <label for="email">Email Address</label>
  <input type="email" id="email" required>

  <label for="message">Message</label>
  <textarea id="message" rows="5"></textarea>

  <button type="submit" class="primary">Send</button>
</form>

Key Concepts

CSS Variables for Global Control

Customize your entire interface by adjusting a few CSS variables:

:root {
  --a-space-unit: 1rem;       /* Global spacing scale */
  --a-color-primary: #2563eb; /* Brand color */
  --a-heading-scale: 1.15;    /* Typography scale */
}

[data-theme="dark"] {
  --a-color-surface: #0f172a;
  --a-color-text: #f8fafc;
}

Semantic Color System

Colors that communicate meaning, not just aesthetics:

  • --a-color-primary — Main interactive elements
  • --a-color-secondary — Alternative actions
  • --a-color-success — Positive outcomes
  • --a-color-warning — Caution, alerts
  • --a-color-error — Errors, destructive actions
  • --a-color-info — Informational messages

Dark Mode Support

Automatic theme switching — all tokens adapt with a single attribute:

<html data-theme="light"></html>
<html data-theme="dark"></html>
