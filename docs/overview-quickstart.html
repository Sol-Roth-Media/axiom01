Overview & Quickstart

Build beautiful, semantic interfaces in minutes. The framework that respects your code.

What is Axiom01?

Axiom01 is a semantic-first UI framework that turns clean HTML into beautiful, accessible interfaces. Stop wrestling with utility classes and component naming conventions. Write meaningful markup, and let CSS do the heavy lifting.

73 Components

Everything from buttons to video players. All production-ready, accessible, and customizable.

149 Design Tokens

Colors, spacing, typography, shadows, borders. One variable, entire site transforms.

13.3 KB CSS

Gzipped. Includes ALL components. Zero dependencies. Production-ready.

Why Axiom01 is Different

Most frameworks force you to choose: clean code OR beautiful design. Axiom01 gives you both.

Semantic HTML

Write meaningful markup using <article>, <section>, <header>. Axiom understands your structure.

One Class Per Component

class="card" instead of class="max-w-sm rounded shadow-lg...". Minimal, expressive, readable.

Accessible by Default

WCAG 2.1 AA compliance built-in. Keyboard navigation, focus states, contrast all handled automatically.

Installation & Setup

Get Axiom01 running in your project with just a few commands.

Option 1: npm (Recommended)

Install via npm, yarn, or pnpm:

npm install axiom01
yarn add axiom01
pnpm add axiom01

Then import in your CSS or JavaScript:

/* In your CSS file */
@import 'axiom01/css/axiom.min.css';
// In your JavaScript
import 'axiom01/css/axiom.min.css';

Option 2: CDN (No Build Tools)

Add these links to your HTML <head> for instant usage:

<!-- CSS Framework -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/axiom01/css/axiom.min.css">

<!-- Icons (Optional) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/axiom01/css/axicons.css">
<script src="https://cdn.jsdelivr.net/npm/axiom01/js/axicons.js"></script>

✨ That's it! Start using components immediately in your HTML. No build step, no configuration, no dependencies.

Key Concepts

CSS Variables for Global Control

Customize your entire interface by adjusting a single variable:

/* Change spacing globally */
--a-space-unit: 1rem;

/* Adjust primary color */
--a-color-primary: #2563eb;

/* Update typography scale */
--a-heading-scale: 1.15;

/* Toggle dark mode */
[data-theme="dark"] {
  --ax-surface: #0f172a;
  --ax-text: #f8fafc;
}

Semantic Color System

Colors that communicate meaning:

  • --a-color-primary - Main interactive element color
  • --a-color-success - Positive action or status
  • --a-color-warning - Caution or attention needed
  • --a-color-error - Error or destructive action
  • --a-color-info - Informational message

Responsive Design

Mobile-first approach with automatic responsive behavior:

<!-- Automatically responsive -->
<div class="grid grid-cols-3">
  <article class="card">...</article>
  <article class="card">...</article>
  <article class="card">...</article>
</div>

<!-- Stacks on mobile, 3 columns on desktop -->

Dark Mode Support

Automatic theme switching. Set once, works everywhere:

<!-- Add data-theme to html element -->
<html data-theme="dark">
  ...
</html>

Common Patterns & Examples

Layout with Container & Grid

<main>
  <section>
    <div class="container">
      <h1>Page Title</h1>
      <div class="grid grid-cols-3" style="gap: var(--a-space-l);">
        <article class="card">...</article>
        <article class="card">...</article>
        <article class="card">...</article>
      </div>
    </div>
  </section>
</main>

Alert & Status Messages

<!-- Success Alert -->
<div class="alert success">
  </span>
  Your changes have been saved successfully.
</div>

<!-- Error Alert -->
<div class="alert error">
  </span>
  Something went wrong. Please try again.
</div>

Modal Dialog

<dialog class="modal">
  <div>
    <header>
      <h2>Confirm Action</h2>
    </header>
    <main>
      <p>Are you sure you want to continue?</p>
    </main>
    <footer>
      <button class="secondary">Cancel</button>
      <button class="primary">Confirm</button>
    </footer>
  </div>
</dialog>

Performance & Accessibility

Performance

  • 13.3 KB CSS - All components included
  • 1.0 KB JS - Optional, for dynamic features
  • Zero dependencies - Pure CSS and vanilla JavaScript
  • Instant load - Works without build tools

Accessibility

  • WCAG 2.1 AA - Compliant by default
  • Keyboard navigation - Full support

  • Motion preference - Respects prefers-reduced-motion
