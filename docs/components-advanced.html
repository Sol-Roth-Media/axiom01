Component Browser (Tool)

The Component Browser is an interactive tool designed to showcase and demonstrate all Axiom01 components in a live environment. It allows developers to quickly preview components, inspect their structure, and understand their behavior.

Usage

The Component Browser itself is a meta-component. It uses a tabbed interface to load individual component documentation pages (or live examples) into an iframe. This allows for isolated testing and viewing of each component.

<div class="component-browser" data-component="component-browser">
    <!-- Header, search, filter, and tabs go here -->
    <div class="preview-container">
        <iframe id="component-preview-iframe" src="docs/components/accordion.html" title="Component preview"></iframe>
    </div>
</div>

JavaScript Integration

The component-browser.js script handles the dynamic loading of component previews into the iframe, manages tab switching, and provides search and filtering capabilities. It listens for clicks on .tab-button elements and updates the iframe's src attribute accordingly.

Accessibility

The tabbed interface uses ARIA attributes (role="tablist", role="tab", aria-selected") to ensure keyboard navigation and screen reader compatibility. The search input has an aria-label for clarity.

Customization

The visual appearance of the Component Browser can be customized using CSS variables, similar to other Axiom01 components. Key variables include:

  • --a-component-browser-bg: Background color of the browser.
  • --a-component-browser-border: Border color.
  • --a-tab-button-bg: Background for tab buttons.
  • --a-tab-button-color: Text color for tab buttons.
  • --a-tab-button-active-bg: Background for active tab.

Data Attributes

  • data-component="component-browser": Initializes the component.
  • data-src on .tab-button: Specifies the URL of the component documentation/preview to load.
  • data-category on .tab-button: Used for filtering components by category.
