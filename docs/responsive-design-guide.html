Responsive Design Guide

Understand how AXIOM01 components adapt across different screen sizes using mobile-first design principles.

Framework Breakpoints

AXIOM01 uses four responsive breakpoints to adapt layouts across all devices. All media queries are mobile-first, meaning constraints start at mobile and are enhanced for larger screens.

Mobile
< 768px

Default: 1-2 column layouts, full-width elements

Tablet
768px - 1023px

Transitional: Auto-fit layouts, flexible sizing

Desktop
1024px - 1439px

Multi-column: 3-4 columns, enhanced features

Large Desktop
1440px+

Optimal: Maximum spacing and sizing

Touch Targets & Accessibility

Mobile responsiveness includes accessibility by default:

  • Minimum tap target: 44px - Ensures easy interaction on mobile
  • Focus states visible - Always accessible via keyboard
  • Color contrast WCAG 2.1 AA - 4.5:1+ ratio maintained at all sizes
  • Touch-friendly spacing - Gaps between interactive elements on mobile
  • Text size 16px+ - Prevents automatic zoom on iOS
  • Reduced motion respected - Animations disabled for users with prefers-reduced-motion

Common Responsive Patterns

Pattern 1: Stack to Multi-Column

Components stack vertically on mobile, arrange horizontally on desktop:

@media (max-width: 768px) {
  .grid { grid-template-columns: 1fr; }  /* 1 column */
}

@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }  /* 3 columns */
}

Pattern 2: Hide/Show Elements

Simplify layout on mobile, show full features on desktop:

@media (max-width: 768px) {
  .desktop-only { display: none; }  /* Hide on mobile */
}

@media (min-width: 1024px) {
  .desktop-only { display: block; }  /* Show on desktop */
}

Pattern 3: Responsive Typography

Font sizes scale appropriately:

h1 { font-size: 1.5rem; }  /* Mobile */

@media (min-width: 1024px) {
  h1 { font-size: 2.5rem; }  /* Desktop */
}

Testing Responsive Design

Browser DevTools

  1. Open DevTools (F12 or right-click → Inspect)
  2. Toggle Device Toolbar (Ctrl+Shift+M or Cmd+Shift+M)
  3. Select device or enter custom dimensions
  4. Test at key breakpoints: 375px, 768px, 1024px, 1440px

Recommended Test Sizes

  • 320px - Small phone
  • 375px - Standard phone (iPhone 12)
  • 480px - Large phone
  • 768px - Tablet (iPad)
  • 1024px - Laptop
  • 1440px - Desktop
  • 1920px - Large desktop

Testing Checklist

  • ☐ Text readable at all sizes
  • ☐ Touch targets 44px+ on mobile
  • ☐ No horizontal scroll on mobile
  • ☐ Images scale proportionally
  • ☐ Buttons accessible via keyboard
  • ☐ Focus states visible everywhere
  • ☐ Color contrast maintained
  • ☐ Animations smooth (60fps)
