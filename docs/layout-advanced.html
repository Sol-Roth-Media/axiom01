Advanced Layout Guide

Axiom01's layout system is built upon the robust and flexible foundation of native CSS Grid, complemented by a cascading spacing system. This guide explores how to leverage these features to create complex, responsive, and maintainable layouts with minimal effort and semantic markup.

The Axiom01 Grid System

The core of Axiom01's layout is the .grid class, which applies display: grid and a default gap based on --a-space-l. You can customize the grid columns directly in your CSS or by using utility classes (if you choose to extend Axiom01 with them).

Basic Grid Usage

To create a responsive grid, simply apply the .grid class to a container and define your column structure. Axiom01's default .grid is designed to be flexible.

<div class="grid" style="grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));">
    <div class="grid-item">Item 1</div>
    <div class="grid-item">Item 2</div>
    <div class="grid-item">Item 3</div>
    <div class="grid-item">Item 4</div>
</div>

Defining Columns and Rows

You can explicitly define grid columns and rows using inline styles or custom CSS. The --a-grid-gutter variable controls the spacing between grid items.

<div class="grid" style="grid-template-columns: 1fr 2fr 1fr; grid-template-rows: auto 100px;">
    <div class="grid-item">Header</div>
    <div class="grid-item" style="grid-column: span 2;">Main Content</div>
    <div class="grid-item">Sidebar</div>
    <div class="grid-item" style="grid-column: 1 / -1;">Footer</div>
</div>

Nested Grids

CSS Grid allows for powerful nesting, enabling you to build complex layouts by placing grids within grid items.

<div class="grid" style="grid-template-columns: 1fr 1fr;">
    <div class="grid-item">Parent Item 1</div>
    <div class="grid-item">
        <div class="nested-grid-example">
            <div class="grid-item">Nested 1</div>
            <div class="grid-item">Nested 2</div>
        </div>
    </div>
</div>

Responsive Layouts with Breakpoints

Axiom01 provides a set of predefined CSS variables for breakpoints, allowing you to create responsive designs using standard media queries.

Variable Value Description
--a-breakpoint-xs480pxExtra small devices (e.g., portrait phones)
--a-breakpoint-sm576pxSmall devices (e.g., landscape phones)
--a-breakpoint-md768pxMedium devices (e.g., tablets)
--a-breakpoint-lg992pxLarge devices (e.g., laptops)
--a-breakpoint-xl1200pxExtra large devices (e.g., desktops)
--a-breakpoint-xxl1400pxExtra extra large devices (e.g., large desktops)
 
/* Example: Two columns on medium screens and up */
.my-responsive-grid {
    display: grid;
    grid-template-columns: 1fr; /* Single column by default */
    gap: var(--a-space-m);
}

@media (min-width: var(--a-breakpoint-md)) {
    .my-responsive-grid {
        grid-template-columns: 1fr 1fr; /* Two columns on medium screens and up */
    }
}

@media (min-width: var(--a-breakpoint-lg)) {
    .my-responsive-grid {
        grid-template-columns: 1fr 2fr 1fr; /* Three columns on large screens and up */
    }
}

The Cascading Spacing System in Layouts

Axiom01's unique cascading spacing system ensures consistent and harmonious layouts. All padding, margin, and gap values are derived from a single --a-space-unit variable, which can be adjusted globally.

<!-- Example of using spacing variables -->
<div style="padding: var(--a-padding-xl); margin-bottom: var(--a-margin-l);">
    <p>This content has extra large padding and large bottom margin.</p>
</div>

<div class="grid" style="gap: var(--a-space-xxl);">
    <div class="grid-item">Wide Gap Item</div>
    <div class="grid-item">Wide Gap Item</div>
</div>

By consistently using these variables, you can maintain a perfect visual rhythm throughout your application, and easily adjust the overall density of your layout by changing just one CSS variable.
