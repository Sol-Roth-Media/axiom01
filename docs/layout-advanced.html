Master CSS Grid and Flexbox with Axiom01's semantic-first approach. Build responsive, beautiful layouts with minimal code.

CSS Grid System

Axiom01 uses pure CSS Grid. Define column structures directly in CSS, not with utility classes. Use fixed column counts per breakpoint for predictable, overflow-free layouts.

Start mobile-first. All layouts begin with a single column.

Standard Breakpoints

Axiom01 defines 6 breakpoints for consistent responsive design across all projects.

Name Width Device Typical Columns
--a-breakpoint-xs 480px Portrait phones 1-2
--a-breakpoint-sm 576px Landscape phones 2
--a-breakpoint-md 768px Tablets 2
--a-breakpoint-lg 992px Laptops 3
--a-breakpoint-xl 1200px Desktops 4
--a-breakpoint-xxl 1400px Large desktops 5

Mobile-First Pattern

/* Start: 1 column (mobile) */
.grid { grid-template-columns: 1fr; }

/* Tablets: 2 columns */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* Laptops: 3 columns */
@media (min-width: 992px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Desktops: 4 columns */
@media (min-width: 1200px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}

Always Use Variables

/* Good */
.card { padding: var(--a-space-l); margin: var(--a-space-m); }

/* Avoid */
.card { padding: 24px; margin: 16px; }

Flexbox for 1D Layouts

Use Flexbox for single-dimension layouts (rows or columns). Combine with Grid for powerful 2D compositions.

Navigation Example

<nav class="main-nav">
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>

/* CSS */
.main-nav {
  display: flex;
  gap: var(--a-space-l);
  align-items: center;
  justify-content: space-between;
}

Button Groups

<div class="button-group">
  <button class="primary">Save</button>
  <button class="secondary">Cancel</button>
</div>

/* CSS */
.button-group {
  display: flex;
  gap: var(--a-space-m);
  flex-wrap: wrap;
}

Best Practices

✅ DO: Mobile-First

  • Start with 1 column (mobile)
  • Add columns at larger breakpoints
  • Test on real devices, not just browsers
  • Prioritize readability over density

✅ DO: Use CSS Variables

  • All spacing: var(--a-space-*)
  • All colors: var(--a-color-*)
  • All sizing: var(--a-*)
  • Ensures consistency and easy customization

✅ DO: Semantic HTML

  • Use proper tags: <nav>, <main>, <section>
  • Minimal classes—one per component root
  • Style descendants, not every element
  • Follow Axiom01 philosophy: HTML first, CSS second

❌ DON'T: Use auto-fit / auto-fill

  • Unpredictable at high resolutions
  • Can cause horizontal overflow
  • Use fixed columns: repeat(N, 1fr)
  • Control layout with media queries

❌ DON'T: Stack Too Many Columns

  • 5+ columns gets cramped and hard to read
  • Consider carousel for many items
  • Readability beats density
  • Test on actual devices at different resolutions

Real-World Examples

Card + Responsive Grid

Combine cards with grids for content-rich layouts. Cards adapt their internal layout at breakpoints.

<article class="card">
  <header><h2>Title</h2></header>
  <div class="card-grid">
    <img src="image.jpg" alt="">
    <div>Content here</div>
  </div>
</article>

Fluid Typography

Use CSS clamp() for responsive font sizes that scale smoothly between breakpoints.

h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
p { font-size: clamp(0.95rem, 1.5vw, 1.1rem); }

Container Queries (Modern)

Adapt layouts based on container width, not viewport width—powerful for reusable components.

@container (min-width: 600px) {
  .component { grid-template-columns: repeat(2, 1fr); }
}

Nested Grids

Place a grid inside a grid item for complex, hierarchical layouts.

<div class="grid">
  <div>Column 1</div>
  <div class="nested-grid">
    <div>Nested 1</div>
    <div>Nested 2</div>
  </div>
</div>
