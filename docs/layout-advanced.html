Advanced Layout Guide

Master CSS Grid and Flexbox with Axiom01's semantic-first approach. Build responsive, beautiful layouts with minimal code and maximum control.

The Axiom01 Grid System

Axiom01's layout foundation is pure CSS Grid: simple, semantic, and powerful. Define your grid structure directly in CSS, not in HTML with utility classes.

🎯 Core Principle

Use CSS Grid with semantic class names. Avoid repeat(auto-fit, minmax()) at high resolutions—use fixed column counts per breakpoint for predictable layouts.

Basic Grid: 1 Column (Mobile Default)

Start mobile-first: 1 column, then scale up at breakpoints.

.demo-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: var(--a-space-m);
}

Responsive Breakpoints

Axiom01 defines 6 standard breakpoints. Design mobile-first, then progressively enhance at each breakpoint.

Breakpoint Size Device Typical Use
--a-breakpoint-xs 480px Portrait phones Primary mobile
--a-breakpoint-sm 576px Landscape phones Large phones
--a-breakpoint-md 768px Tablets Mid-size screens
--a-breakpoint-lg 992px Laptops 3-column layouts
--a-breakpoint-xl 1200px Desktops 4-column layouts
--a-breakpoint-xxl 1400px Large desktops 5+ column layouts

Mobile-First Pattern

/* Default: 1 column (mobile) */
.my-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: var(--a-space-m);
}

/* 2 columns at tablets and up */
@media (min-width: 768px) {
  .my-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* 3 columns at laptops and up */
@media (min-width: 992px) {
  .my-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* 4 columns at desktops and up */
@media (min-width: 1200px) {
  .my-grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

The Cascading Spacing System

All spacing derives from a single --a-space-unit. Change one variable, scale your entire layout proportionally.

Spacing Scale

Six levels of semantic spacing, each calculated from the base unit:

--a-space-xs
4px
--a-space-s
8px
--a-space-m
16px
--a-space-l
24px
--a-space-xl
40px
--a-space-xxl
64px

Always Use Variables

/* ✅ CORRECT: Using Axiom spacing */
.card {
  padding: var(--a-space-l);
  margin-bottom: var(--a-space-xl);
  gap: var(--a-space-m);
}

/* ❌ WRONG: Hardcoded pixels */
.card {
  padding: 24px;
  margin-bottom: 40px;
  gap: 16px;
}

Flexbox for 1D Layouts

Use Flexbox for single-dimension layouts (rows or columns). CSS Grid handles 2D layouts. Use both together for powerful compositions.

Flex Direction: Row (Default)

Items flow horizontally, wrapping on smaller screens.

.flex-demo {
  display: flex;
  gap: var(--a-space-m);
  flex-wrap: wrap;
}

Responsive Typography

Use CSS clamp() for fluid typography that scales with viewport.

h1 {
  font-size: clamp(1.5rem, 5vw, 3rem);
}

Container Queries

Adapt layouts based on container width, not viewport width.

@container (min-width: 600px) {
  .component {
    grid-template-columns: repeat(2, 1fr);
  }
}

Best Practices

✅ DO: Mobile-First Design

  • Start with 1 column layouts
  • Add columns at larger breakpoints
  • Test on real devices, not just browsers

✅ DO: Use CSS Variables

  • All spacing: var(--a-space-*)
  • All colors: var(--a-color-*)
  • All sizing: var(--a-*)

✅ DO: Semantic HTML

  • Use proper tags: <nav>, <main>, <section>, <article>
  • Minimal classes (one per component)
  • Style descendants, not everything

❌ DON'T: Avoid auto-fit / auto-fill

  • Unpredictable at high resolutions
  • Use fixed columns: repeat(N, 1fr)
  • Control behavior with media queries

❌ DON'T: Stack Too Many Columns

  • 5+ columns gets cramped and hard to read
  • Consider carousel or pagination for many items
  • Prioritize readability over density
