Advanced Layout Guide

Master CSS Grid and Flexbox with Axiom01. Build semantic, responsive layouts that work perfectly at every resolution using the principle of mobile-first design.

Grid Philosophy

Axiom01 embraces a semantic-first approach to layout. Instead of relying on unpredictable repeat(auto-fit, minmax()) patterns, we use fixed column counts per breakpoint for layouts that never overflow.

❌ Avoid: auto-fit / auto-fill

These create unpredictable layouts at high resolutions:

/* Problems at high resolutions */
.grid { grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); }

✅ Prefer: Fixed Columns per Breakpoint

Use mobile-first with explicit breakpoints:

/* Mobile first */
.grid { grid-template-columns: 1fr; }

/* Tablet */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* Desktop */
@media (min-width: 1200px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}

Responsive Breakpoints

Axiom01 defines 6 standard breakpoints for consistent responsive design:

Breakpoint Width Device Columns
--a-breakpoint-xs 480px Portrait phones 1
--a-breakpoint-sm 576px Landscape phones 1-2
--a-breakpoint-md 768px Tablets 2
--a-breakpoint-lg 992px Laptops 3
--a-breakpoint-xl 1200px Desktops 4
--a-breakpoint-xxl 1400px Large desktops 5

Advanced: Responsive Card Layout

Cards that adapt layout based on content and viewport. Combines grid with semantic HTML.

Image + Content Card (Responsive)

Card Title

This card layout uses CSS Grid with 2 columns on desktop, stacking to 1 column on mobile. Perfect for product cards, team members, or content previews.

<article class="card-example">
  <img src="image.jpg" alt="">
  <div>
    <h4>Title</h4>
    <p>Content</p>
  </div>
</article>

/* Desktop: 2 columns */
.card-example {
  grid-template-columns: 200px 1fr;
}

/* Mobile: 1 column */
@media (max-width: 600px) {
  .card-example {
    grid-template-columns: 1fr;
  }
}

Advanced: Responsive Grid Reordering

Use CSS Grid's `order` property to rearrange items at different breakpoints without changing HTML.

  • ✅ Semantic HTML doesn't change (accessibility preserved)
  • ✅ Content hierarchy can adapt to screen size
  • ✅ No need to duplicate HTML or hide elements
  • ✅ Screen readers follow HTML order (not visual order)
  • ❌ Don't use for critical content ordering - HTML order is most important

    • Best Practices

    ✅ DO: Mobile-First

    • Start with 1 column
    • Add columns at larger breakpoints
    • Test on real devices

    ✅ DO: Use CSS Variables

    • Spacing: var(--a-space-*)
    • Colors: var(--a-color-*)
    • Breakpoints: var(--a-breakpoint-*)

    ✅ DO: Semantic HTML

    • Use <section>, <article>, <nav>
    • Minimal classes (one per component)
    • Style descendants, not every element

    ❌ DON'T: Use auto-fit / auto-fill

    • Unpredictable at high resolutions
    • Can cause overflow
    • Use fixed columns instead

    ❌ DON'T: Stack Too Many Columns

    • 5+ columns gets cramped
    • Readability > density
    • Use carousel or pagination for many items
