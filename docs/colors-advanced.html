Colors & Design Tokens

Axiom01 uses a semantic color system based on Material Design 3 principles. Colors are named by their role, not their hue — so they adapt automatically across themes and dark mode.

Color System Overview

Rather than naming colors red or blue, Axiom01 defines colors by purpose: primary, error, surface. This makes them meaningful, maintainable, and theme-ready.

Instead of "use red for errors", define "use --a-color-error" — which adapts automatically for dark mode and respects users with color blindness when paired with text.

Primary Color Family

Your brand color — used for the most important actions and states.

TokenRoleUse for
--a-color-primaryBrand colorAction buttons, links, focus rings
--a-color-on-primaryText on primaryText inside primary-colored elements
--a-color-primary-containerLight primaryBadge backgrounds, card accents, chips
--a-color-on-primary-containerText on containerText inside primary-container elements
 
button.primary {
  background: var(--a-color-primary);
  color: var(--a-color-on-primary);
}

.badge {
  background: var(--a-color-primary-container);
  color: var(--a-color-on-primary-container);
}

Semantic Status Colors

Status colors communicate state. Use the container variants for alert backgrounds; use the base variants for borders and icons.

TokenRoleUse for
--a-color-successPositive stateConfirmations, completed actions
--a-color-success-containerLight successAlert backgrounds, subtle badges
--a-color-warningCautionary stateWarnings before irreversible actions
--a-color-warning-containerLight warningWarning alert backgrounds
--a-color-errorNegative stateValidation errors, destructive actions
--a-color-error-containerLight errorError alert backgrounds
--a-color-infoInformationalTips, hints, neutral notices
--a-color-info-containerLight infoInfo alert backgrounds
 
.alert.success {
  background: var(--a-color-success-container);
  color: var(--a-color-on-success-container);
  border-left: 4px solid var(--a-color-success);
}

.alert.error {
  background: var(--a-color-error-container);
  color: var(--a-color-on-error-container);
  border-left: 4px solid var(--a-color-error);
}

Surface & Neutral Colors

Surface colors define the background hierarchy. Neutral text colors provide readable contrast at every level.

TokenRoleUse for
--a-color-surfacePrimary backgroundPage backgrounds, cards, containers
--a-color-surface-variantSecondary backgroundHover states, striped rows, input fills
--a-color-on-surfacePrimary textMain body text, headings
--a-color-on-surface-variantSecondary textLabels, helper text, captions
--a-color-outlineBorders & dividersCard borders, input borders, <hr>

Understanding "On" Colors

Any token prefixed with on- is the foreground color that goes on top of the named background. This pairing guarantees WCAG contrast compliance.

Background var(--a-color-primary) → text uses var(--a-color-on-primary)
Background var(--a-color-error-container) → text uses var(--a-color-on-error-container)

Understanding "Container" Colors

Container colors are lighter, tinted versions of their parent. Use them for backgrounds when the full-strength color would be too heavy.

Success — uses success-container background

Error — uses error-container background

/* Full-strength: prominent actions */
button.primary { background: var(--a-color-primary); }

/* Container: subtle backgrounds */
.badge        { background: var(--a-color-primary-container); }

Always Use Tokens, Never Hardcode

✓ Correct

button {
  background: var(--a-color-primary);
  color: var(--a-color-on-primary);
  border-color: var(--a-color-outline);
}

✗ Avoid

button {
  background: #2563eb;
  color: #ffffff;
  border-color: #e2e8f0;
}

Hardcoded colors break dark mode, break custom themes, and create maintenance debt. Tokens are automatic.

Dark Mode

All tokens adapt automatically. Add data-theme="dark" and every color responds:

<!-- Light mode -->
<html data-theme="light">

<!-- Dark mode -->
<html data-theme="dark">

Toggle it via JavaScript:

const toggle = () => {
  const current = document.documentElement.getAttribute('data-theme');
  document.documentElement.setAttribute('data-theme',
    current === 'dark' ? 'light' : 'dark'
  );
};
