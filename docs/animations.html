Animation Philosophy

Animation in Axiom01 is intentional, never decorative.

Purposeful

Every animation communicates state, guides focus, or improves perceived performance. Decoration is never the goal.

Performant

All animations use GPU-accelerated properties (transform, opacity) and complete in 150–500ms for snappy responsiveness.

Accessible

Respects user preferences with prefers-reduced-motion. No animation blocks interaction or harms usability.

Add Animation with One Class: `.animate`

Axiom01's semantic approach. Just add `.animate` to any component for intelligent, automatic animations.

The Pattern

Default (one class):

<!-- Card animates with fadeInUp -->
<article class="card animate">
  Card content
</article>

<!-- Button animates with scaleIn -->
<button class="animate">
  Click me
</button>

<!-- Modal animates with slideUp -->
<div class="modal animate">
  Modal content
</div>

Override (add animation class):

<!-- Card with custom animation -->
<article class="card  animation-demo reveal-fade">
  Content
</article>

<!-- Staggered sequence -->
<article class="card  animation-demo delay-1"></article>
<article class="card  animation-demo delay-2"></article>
<article class="card  animation-demo delay-3"></article>

Why Component Defaults?

Each component serves a different purpose and appears in different contexts. Rather than one-size-fits-all animations, Axiom01 gives each component the animation that best communicates its role:

  • Cards need gentle, content-focused entrance (fadeInUp)
  • Buttons need attention-grabbing emphasis (scaleIn)
  • Modals need clear state change emphasis (slideUp)
  • Alerts need top-down warning signal (fadeInDown)
  • Tooltips need quick, subtle reveal (fadeIn)

What Does `.animate` Do?

Adding .animate to a component triggers its intelligent default animation:

  • Framework detects component type
  • Applies best animation for that type
  • Sets optimal timing (300–500ms)
  • Uses professional easing curves
  • Respects prefers-reduced-motion

No thinking required—just add `.animate`.

Fade Animations

Gentle opacity changes for introductions and transitions.

Use Case: Page loads, modal entries, content reveals, focus states.

CSS Usage

.reveal-fade {
  animation: fadeIn 300ms ease-out;
}

.reveal-fade-up {
  animation: fadeInUp 300ms ease-out;
}

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(0.5rem);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

Loading & Status Animations

Continuous loops for loading states and feedback.

Use Case: Loading spinners, progress indicators, status updates, background processes.

CSS Usage

.status-spin {
  animation: spin 1s linear infinite;
}

.status-pulse {
  animation: pulse 2s ease-in-out infinite;
}

@keyframes spin {

  to { transform: rotate(360deg); }

Timing & Easing Guidelines

Choose the right duration and easing for your animation type.

Recommended Durations

0–100ms Instant feedback
150–200ms Quick, snappy
300ms Standard transitions
500ms Important, deliberate
800ms+ Entrance, emphasis

Common Easing Functions

ease-out Arrivals, entrances
ease-in Departures, exits
ease-in-out State changes
linear Continuous (spinners)

Accessibility & prefers-reduced-motion

Animations must respect user preferences.

CSS Pattern

/* Default: With animation */
.element {
  animation: fadeIn 300ms ease-out;
}

/* User prefers reduced motion: Disable all animations */
@media (prefers-reduced-motion: reduce) {
  .element {
    animation: none !important;
    opacity: 1;  /* Ensure element is visible */
  }
}

Testing prefers-reduced-motion

macOS: System Preferences → Accessibility → Display → Reduce Motion

Windows: Settings → Ease of Access → Display → Show animations

Firefox: about:config → ui.prefersReducedMotion → 1

JavaScript Detection

const prefersReduced = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
).matches;

if (!prefersReduced) {
  // Apply animation
} else {
  // Use fallback
}

Performance Best Practices

Ensure animations run smoothly at 60 FPS.

GPU-Accelerated Properties

Safe to animate:

  • opacity
  • transform (translate, scale, rotate)
  • filter

Never Animate These

Causes layout recalculation:

  • width, height
  • left, top, right, bottom
  • margin, padding
  • font-size

Good Example

Use GPU-accelerated transform:

.dropdown.menu {
  transform: scaleY(0);
  transition: transform 300ms;
}

.dropdown.is-open .dropdown.menu {
  transform: scaleY(1);
}

Bad Example

Don't animate expensive properties:

/* ❌ WRONG: Triggers layout recalc */
.dropdown.menu {
  max-height: 0;
  transition: max-height 300ms;
}

.dropdown.is-open .dropdown.menu {
  max-height: 500px;
}

Animation Quality Checklist

Before shipping an animated component, verify these points.

Design & Purpose

  • Animation communicates state change
  • Improves UX, not decoration
  • Doesn't distract from content
  • Speed appropriate (150–500ms)
  • Doesn't move elements under cursor

Implementation

  • Uses GPU-accelerated properties
  • CSS animations, not JavaScript
  • Professional easing
  • Uses CSS variables
  • No hardcoded values

Accessibility

  • Respects prefers-reduced-motion
  • Doesn't block keyboard access
  • Focus still visible without animation
  • Content actionable if animation fails
  • Tested with screen readers

Performance

  • No layout thrashing
  • No excessive stacking
  • 60 FPS verified in DevTools
  • Minimal bundle impact
  • Doesn't block interaction
