Animations

System Reference

Animation in Axiom01 is intentional, never decorative. Every animation communicates state, guides focus, or improves perceived performance using GPU-accelerated properties.

Philosophy

Purposeful

Every animation communicates state, guides focus, or improves perceived performance. Decoration is never the goal.

Performant

All animations use GPU-accelerated properties (transform, opacity) and complete in 150–500ms for snappy responsiveness.

Accessible

Respects user preferences with prefers-reduced-motion. No animation blocks interaction or harms usability.

Automatic Component Animations

Axiom01 handles semantic animations natively. The following components automatically trigger entrance or interaction animations when they are used appropriately. No utility classes needed.

Modals & Dialogs

The <dialog> element automatically scales up and fades in using a custom cubic-bezier timing function when [open] is added.

<dialog open>
  Modal content smoothly scales in
</dialog>

Tooltips

Adding the data-tooltip attribute automatically injects a tooltip that slides up and fades in on hover or focus.

<button data-tooltip="Helper text">
  Hover me
</button>

Drawers

The <aside class="drawer"> component automatically slides in from the edge of the screen when activated.

<aside class="drawer" data-side="left">
  Sidebar slides in
</aside>

Interactive Elements

Form controls (like Selectable Cards) provide instant, highly-optimized visual feedback via :has() and CSS transitions on interaction.
