HTML Structure

<div class="form-container">
    <div class="form-progress">
        <div class="progress-step active">
            <div class="progress-dot">1</div>
            <div class="progress-label">Step 1</div>
        </div>
    </div>
    
    <div class="form-content">
        <div class="form-step active">
            <h2 class="form-step-title">Title</h2>
            <div class="form-field">
                <label class="form-label">Label</label>
                <input type="text" class="form-input">
            </div>
        </div>
    </div>
    
    <div class="form-actions">
        <button class="form-button secondary">Previous</button>
        <button class="form-button primary">Next</button>
    </div>
</div>

Features

  • Progress Indicator - Visual step tracking
  • Step Navigation - Previous/Next controls
  • Form Validation - Validate before advancing
  • Field Organization - Logical grouping of inputs
  • Success State - Completion confirmation
  • State Persistence - Remember entered data

Use Cases

  • Account Registration - Multi-step signup flow
  • Checkout - E-commerce purchase process
  • Onboarding - Setup wizard for new users
  • Surveys - Multi-page questionnaires
  • Applications - Job or grant applications
  • Configuration - Complex app setup

Accessibility

  • Use semantic form elements
  • Include proper labels on all inputs
  • Announce current step to screen readers
  • Support keyboard navigation
  • Show validation error messages clearly
  • Maintain focus management between steps

Best Practices

  • Clear Progress - Show which step user is on
  • Logical Grouping - Related fields on same step
  • Validation - Validate before advancing
  • Save Progress - Remember entered data
  • Error Messages - Clear, actionable feedback
  • Estimated Time - Show expected duration
