HTML Structure

<form>
    <!-- Progress Indicator -->
    <ol class="steps-progress" role="tablist">
        <li class="step-item" data-active="true" data-step="1" role="tab">
            <div class="step-number">1</div>
            <span class="step-label">Step 1</span>
        </li>
    </ol>

    <!-- Form Sections -->
    <div class="form-sections" role="region">
        <section class="form-section" data-active="true" id="step1">
            <h2>Step 1 Title</h2>
            <div class="field">
                <label for="input1">Field Label</label>
                <input type="text" id="input1">
            </div>
        </section>
    </div>

    <!-- Navigation -->
    <div class="form-controls">
        <button type="button" class="button" data-variant="secondary" id="prevBtn">
            Previous
        </button>
        <button type="button" class="button" data-variant="primary" id="nextBtn">
            Next
        </button>
    </div>
</form>

Key Features

  • Progress Indicator - Visual step tracking with completed state
  • Step Navigation - Previous/Next buttons for movement
  • Data Attributes - `data-active` for visibility, `data-completed` for state
  • Semantic HTML - Proper form structure with ARIA roles
  • Success State - Completion confirmation message
  • Accessible - Full keyboard and screen reader support

Data Attributes

  • `data-active="true"` - Marks currently displayed step
  • `data-completed="true"` - Marks completed step in progress
  • `data-step="1|2|3"` - Step identifier for navigation

ARIA Attributes

  • `role="tablist"` - Progress indicator is tab list
  • `role="tab"` - Individual step items
  • `aria-selected="true|false"` - Current tab indicator
  • `role="region"` - Form sections container
  • `aria-labelledby` - Links sections to titles

Use Cases

  • Account Registration - Multi-step signup flow
  • Checkout - E-commerce purchase process
  • Onboarding - Setup wizard for new users
  • Surveys - Multi-page questionnaires
  • Applications - Job or grant applications
  • Configuration - Complex app setup

Best Practices

  • Show clear progress indicator at all times
  • Group related fields on the same step
  • Display step titles and descriptions
  • Validate before advancing to next step
  • Allow going back to previous steps
  • Show summary/review before final submission
  • Display success confirmation after submission
  • Support keyboard navigation throughout
