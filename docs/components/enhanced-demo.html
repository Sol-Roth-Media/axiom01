JavaScript API

Manual Initialization

// After DOM ready
Axiom.initQueue(element)
Axiom.initSegmentedControl(element)
Axiom.initChipInput(element)
Axiom.initBacklit(element)

Event Listeners

// Segmented Control change
element.addEventListener(
  'segmented-change',
  (e) => console.log(e.detail.value)
)

Component Tracking

// Check initialized
Axiom.initializedComponents.has(el)

// Get instance
Axiom.initializedComponents
  .get(el).instance

Cleanup

// Remove single
Axiom.destroyComponent(element)

// Remove all
Axiom.destroyAllComponents()

Keyboard Shortcuts

Component Key Action
Segmented Control Tab Move between buttons
Segmented Control ← / → Navigate left/right
Segmented Control ↑ / ↓ Navigate up/down
Chip Input Enter / , Add chip
Chip Input Backspace Remove last chip (when empty)
Queue Drag Reorder items

When to Use Enhanced Demo

Do Use

  • Use this component for its intended purpose
  • Follow the semantic structure
  • Implement proper accessibility
  • Respect responsive design principles
  • Use design tokens for customization

Don't Use

  • Don't modify core structure unnecessarily
  • Avoid hardcoding styles or colors
  • Don't skip accessibility features
  • Avoid responsive design gotchas
  • Don't duplicate when a component exists

Consider

  • Your specific use case and context
  • User accessibility needs (WCAG AAA)
  • Dark mode and theme variations
  • Mobile and desktop experiences
  • Performance optimization
