Notification

Alert users to important information with semantic notifications. Supports multiple types: success, error, warning, and info.

Live Examples

Success! Your changes have been saved.

Error! Something went wrong. Please try again.

Note: New features are now available.

Markup

Notifications use semantic HTML with data-variant for type styling:

<div role="alert" data-variant="success">
				<span class="axicon render" data-name="CheckCircle"></span>
				<div>
				<strong>Success!</strong> Your action completed.
				</div>
				<button aria-label="Close notification">&times;</button>
				</div>

				<!-- Variants: success, error, warning, info -->

Data Attributes

  • `data-variant="success|error|warning|info"` - Notification type (colors and icons)

ARIA Attributes

  • `role="alert"` - Announces notification to screen readers immediately
  • `aria-label` - Describes close button purpose

Features

  • Semantic HTML - Uses <div> with role="alert"
  • Icon Support - Include icons for visual clarity
  • Close Button - Allow users to dismiss
  • Type Variants - Four semantic notification types
  • Token-Based Colors - Uses design tokens for styling
  • Accessibility - ARIA roles for screen readers

Accessibility

  • `role="alert"` - Announces notification to screen readers immediately on load
  • `aria-label` - Labels close button for keyboard users
  • Color + Icon - Never rely on color alone to convey type
  • Focus Management - Close button is always keyboard accessible
  • Clear Text - Use descriptive, actionable messages

Responsive Behavior

This component automatically adapts to different screen sizes:

Mobile (< 768px)

On mobile devices:

  • Optimized for small screens
  • Touch-friendly spacing maintained (44px minimum)
  • Text wraps naturally without overflow
  • Content density optimized for readability

Desktop (1024px+)

On desktop displays:

  • Enhanced visual features
  • Hover states and transitions
  • Full keyboard navigation support
  • Optimal spacing and sizing

Framework Integration

Responsive behavior is automatic:

  • Framework handles media queries
  • No custom CSS required
  • Mobile-first approach applied
  • Accessibility maintained at all sizes

Best Practices

  • Keep messages concise and actionable
  • Use appropriate type for the message content
  • Include icons to reinforce the message type
  • Allow users to dismiss notifications
  • Position consistently on the page
  • Use color with icon/text, not color alone
