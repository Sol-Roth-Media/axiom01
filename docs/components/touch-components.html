Touch Gesture Reference

👆

Tap

Single finger press and release. Primary interaction.

👆👆

Double Tap

Two rapid taps. Common for zoom on photos.

👇

Long Press

Hold finger down for 500ms+. Context menus.

👉

Swipe

Fast directional drag. Navigation between views.

✌️

Pinch

Two fingers moving apart/together. Zoom control.

🔄

Rotate

Two fingers rotating. Image/map rotation.

HTML Structure

<div class="touch-target" id="touchElement">
    <div class="touch-icon">👆</div>
    <div class="touch-label">Tap to activate</div>
</div>

<script>
const element = document.getElementById('touchElement');

// Single tap
element.addEventListener('touchstart', (e) => {
    // Handle touch start
});

element.addEventListener('touchend', (e) => {
    // Handle touch end
});

// Swipe detection
let startX = 0;
element.addEventListener('touchstart', (e) => {
    startX = e.touches[0].clientX;
});

element.addEventListener('touchend', (e) => {
    const endX = e.changedTouches[0].clientX;
    if (endX < startX - 50) {
        // Swiped left
    } else if (endX > startX + 50) {
        // Swiped right
    }
});
</script>

Features

  • Tap Detection - Single and double tap recognition
  • Swipe Gestures - Left, right, up, down swiping
  • Long Press - Extended touch duration detection
  • Pinch Zoom - Multi-touch scaling
  • Rotation - Two-finger rotation detection
  • Mobile Optimized - Touch-action CSS properties

Use Cases

  • Mobile Navigation - Swipe between screens
  • Photo Galleries - Swipe and pinch to zoom
  • Maps - Pan, pinch, and rotate
  • Sliders - Swipe to change values
  • Context Menus - Long-press to open options
  • Card Swiping - Tinder-style interactions

Accessibility

  • Provide touch-action CSS properties
  • Support keyboard fallbacks for all gestures
  • Include visual feedback on touch interactions
  • Minimum 44×44px touch targets recommended
  • Use semantic HTML for touch elements
  • Announce gesture results to screen readers

Best Practices

  • Touch Targets - Make buttons 44×44px minimum
  • Visual Feedback - Show active states clearly
  • Gesture Discovery - Hint users about swipes
  • Debounce Gestures - Prevent rapid repeated triggers
  • Haptic Feedback - Vibrate on important actions
  • Fallback Controls - Buttons for non-gesture users
