Drawer

Slide-in panel using native <dialog>. Full keyboard trap, Escape key, and backdrop built in.

Right Drawer

<button class="primary"
        data-dialog-open="nav-drawer">
  Open Drawer
</button>

<dialog class="drawer" id="nav-drawer">
  <header>
    <h2>Navigation</h2>
    <button aria-label="Close"
            data-dialog-close="nav-drawer">
      <span class="axicon render" data-name="X"></span>
    </button>
  </header>
  <div>
    <nav><ul>
      <li><a href="/" aria-current="page">Dashboard</a></li>
      <li><a href="/settings">Settings</a></li>
    </ul></nav>
  </div>
</dialog>

Left Drawer

<!-- Add data-side="left" for left-side drawer -->
<dialog class="drawer" data-side="left" id="filter-drawer">
  <header>...</header>
  <div>...</div>
</dialog>

Semantic Structure

ElementRole
dialog.drawerContainer — native modal with backdrop
headerTitle + close button row
divScrollable content area inside the dialog
footer (optional)Action buttons
data-side="left"Slides from left instead of right

Why Native <dialog>?

  • Built-in focus trap — keyboard users cannot escape accidentally
  • Escape closes automatically — no JS needed
  • Native ::backdrop — no extra overlay element
  • Screen reader announces it as a dialog region
