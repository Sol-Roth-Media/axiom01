Hero

Large-impact section for page introductions and primary calls-to-action. Uses semantic HTML with button component.

Basic Hero Section

Welcome to Axiom01

Build beautiful, semantic websites with minimal CSS and maximum accessibility.

<article style="background: linear-gradient(...); padding: var(--a-space-xl); text-align: center; color: white;">
    <h2>Welcome to Axiom01</h2>
    <p>Build beautiful, semantic websites...</p>
    <div style="display: flex; gap: var(--a-space-m); justify-content: center;">
        <button class="button" data-variant="primary">Get Started</button>
        <button class="button" data-variant="secondary">View Docs</button>
    </div>
</article>

Key Features

  • Semantic HTML - Uses `
    ` or `
    `, no custom classes
  • Flex Layout - Buttons centered and responsive with gap spacing
  • Button Component - Uses `.button` with `data-variant` for styling
  • Gradient Background - Uses design token colors in CSS linear-gradient
  • Design Tokens Only - All colors, spacing, sizing from var(--a-*)
  • Responsive - Buttons wrap on mobile, text scales

Styling Options

  • Gradient Background - `linear-gradient(135deg, var(--a-color-primary) 0%, var(--a-color-primary-container) 100%)`
  • Button Variant 1 - `data-variant="primary"` for main CTA
  • Button Variant 2 - `data-variant="secondary"` for secondary action
  • Flex Gap - Use `gap: var(--a-space-m)` for consistent spacing
  • Text Colors - White text on gradient background
  • Padding - Use `var(--a-space-xl)` for generous hero padding

Accessibility

  • Use semantic `
    ` or `
    ` elements
  • Use semantic `
  • Maintain high color contrast (white text on gradient)
  • Buttons have visible focus states for keyboard navigation
  • Limit to 2-3 buttons maximum (reduces cognitive load)
  • Text scales properly on all screen sizes
  • Proper heading hierarchy (h1, h2, h3)

Use Cases

  • Landing Pages - Primary call-to-action section at page top
  • Product Pages - Feature showcase introduction with CTA buttons
  • Service Highlights - Large banner sections explaining key benefits
  • Event Announcements - Highlight important events with signup buttons
  • Feature Promotions - Showcase specific product features or updates

Best Practices

  • Keep headline short and compelling
  • Write clear, action-oriented button text
  • Use maximum 2 buttons (primary + secondary action)
  • Ensure text is readable at all screen sizes
  • Use gradient or solid background color for contrast
  • Position hero near top of page for maximum visibility
  • Use design tokens for all styling (no hardcoded colors)
  • Test keyboard navigation and focus states
