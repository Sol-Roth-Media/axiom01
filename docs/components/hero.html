HTML Structure

Basic Hero with Buttons

<section class="hero">
  <div class="container">
    <h1>Your Headline</h1>
    <p class="tagline">Optional subtitle or description</p>
    <div class="actions">
      <button class="primary">Get Started</button>
      <button class="outline">Learn More</button>
    </div>
  </div>
</section>

Hero with Links as Buttons

<section class="hero">
  <div class="container">
    <h1>Headline</h1>
    <p>Description</p>
    <div class="actions">
      <a href="/signup" class="primary">Sign Up</a>
      <a href="/learn" class="outline">Learn More</a>
    </div>
  </div>
</section>

button.outline

Transparent background with white border and text.

CSS Philosophy

No Custom Classes for Buttons

Following AXIOM01 philosophy, buttons don't require a redundant .button class. Just use <button> or <a role="button"> with optional modifier classes like .primary or .outline.

The framework styles all buttons automatically through CSS selectors.

Centered Actions by Default

The .actions container uses justify-content: center to center buttons horizontally. Buttons wrap on smaller screens with flexible gap sizing.

Accessibility

Best Practices

  • Use semantic <section> for hero container
  • Use semantic <button> or <a> for interactive elements
  • Maintain high color contrast on gradient backgrounds
  • Buttons have visible focus states for keyboard navigation
  • Limited number of buttons (2-3 max) reduces cognitive load
  • Text is readable on all screen sizes
