Tabs

Axiom01's tabs component organizes related content into separate panels. Supports keyboard navigation and multiple styling variants for flexible content organization.

Live Examples

Documentation Tabs

Component Overview

The tabs component provides an organized way to display related content. Each tab contains a panel with relevant information. Users can switch between tabs to view different content without leaving the page.

Code Example

Implementation is straightforward with semantic HTML and accessible ARIA attributes. The component supports keyboard navigation with arrow keys and full screen reader support for accessibility.

Configuration

Customize tabs with CSS variables for colors, spacing, and transitions. Support for vertical tabs, icon-only tabs, and disabled states. Flexible enough for any use case.

Frequently Asked Questions

Q: Can I have nested tabs? A: Yes, but use with caution for usability. Q: Mobile responsive? A: Fully responsive with horizontal scrolling on small screens.

HTML Structure

<div class="tabs-container">
    <div class="tabs-nav" role="tablist">
        <button class="tab-button active" role="tab" aria-controls="panel1">
            Tab 1
        </button>
        <button class="tab-button" role="tab" aria-controls="panel2">
            Tab 2
        </button>
    </div>
    
    <div id="panel1" class="tab-content active" role="tabpanel">
        <p>Content for tab 1</p>
    </div>
    <div id="panel2" class="tab-content" role="tabpanel">
        <p>Content for tab 2</p>
    </div>
</div>

Features

  • Tab Navigation - Click or keyboard to switch tabs
  • Content Panels - Organized content areas per tab
  • Active State - Clear indication of current tab
  • Icon Support - Optional icons on tabs
  • Keyboard Navigation - Arrow keys and Enter support
  • ARIA Support - Full accessibility for screen readers

Use Cases

  • Documentation - Multiple sections of content
  • Settings Pages - Grouped configuration options
  • Product Details - Description, reviews, specs tabs
  • Dashboard - Different data views or filters
  • Form Steps - Multi-section form submission
  • Navigation - Alternative to dropdown menus

Accessibility

  • Use role="tablist" on tab container
  • Use role="tab" on tab buttons
  • Use role="tabpanel" on content areas
  • Include aria-controls linking tabs to panels
  • Support keyboard: Tab, Arrow keys, Home, End
  • Maintain focus management

Best Practices

  • Clear Labels - Tab titles should be clear
  • Logical Ordering - Order tabs by importance
  • Icon + Text - Use both for clarity
  • Content Organization - Related content per tab
  • Keyboard Support - Full arrow key navigation
  • Mobile Responsive - Scroll if needed on small screens
