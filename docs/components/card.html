Card

Semantic card component for grouping related content. Single `.card` class with semantic HTML structure and `data-variant` for styling options.

Default Card

Card Title

This is a simple card with semantic HTML structure. Content goes here.

<article class="card">
    <header>
        <h3>Card Title</h3>
    </header>
    <p>Card content here.</p>
    <footer>
        <button class="button" data-variant="primary">Action</button>
    </footer>
</article>

Markup

Card component uses semantic `<article>` element with single `.card` class:

<article class="card">
    <img src="..." alt="...">
    <header>
        <h2>Title</h2>
        <p>Subtitle</p>
    </header>
    <!-- Main content -->
    <p>Card body content...</p>
    <footer>
        <button class="button">Action</button>
    </footer>
</article>

Semantic Structure

  • `<article class="card">` - Card container (semantic)
  • `<header>` - Card header section (semantic, optional)
  • `<h3>`, `<h2>` - Card title (semantic heading)
  • `<p>`, content - Card body content (semantic)
  • `<footer>` - Card actions section (semantic, optional)

Variants

Use `data-variant` attribute for different card styles:

  • No variant - Default card styling
  • `data-variant="outlined"` - Emphasized border style
  • `data-variant="elevated"` - Shadow-based elevation

Key Principles

  • Single Class - Only `.card` class on container
  • Semantic HTML - Use `<article>`, `<header>`, `<footer>`
  • No Element Classes - No `.card-header`, `.card-body`, `.card-footer`
  • Data Attributes for Variants - Use `data-variant` for styling options
  • Natural Content Flow - Structure matches semantic meaning

Accessibility

  • Use `<article>` for self-contained content
  • Include proper heading hierarchy
  • Use `<header>` and `<footer>` for semantic organization
  • Ensure sufficient color contrast
  • Include descriptive `alt` text for images
  • Make buttons keyboard accessible

Best Practices

  • Keep card content focused and related
  • Use consistent card layouts
  • Include clear calls-to-action
  • Add images or icons for visual interest
  • Use proper semantic elements for organization
  • Test on multiple screen sizes
  • Ensure keyboard navigation works throughout
