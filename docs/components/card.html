Card

Axiom01's card component is a flexible and extensible content container. Cards group related information in a visually appealing format, perfect for dashboards, product listings, or blog posts.

HTML Structure

Cards use semantic HTML with optional header, body, and footer sections:

<article class="card">
    <header class="card-header">
        <h3>Card Title</h3>
    </header>
    <div class="card-body">
        <p>Card content goes here.</p>
    </div>
    <footer class="card-footer">
        <button class="button">Action</button>
    </footer>
</article>

Variants

Cards support multiple style variants:

<!-- Default (outlined) -->
<article class="card">...</article>

<!-- Outlined (emphasized border) -->
<article class="card outlined">...</article>

<!-- Elevated (shadow-based) -->
<article class="card elevated">...</article>

With Images

Cards can display images or visual content:

<article class="card">
    <img src="image.jpg" alt="Description" class="card-image-src">
    <header class="card-header">
        <h3>Title</h3>
    </header>
    <div class="card-body">
        <p>Content...</p>
    </div>
</article>

Accessibility

  • Use semantic <article> or <section> elements as the card container
  • Always include a heading within the card header for context
  • Ensure adequate color contrast between text and background
  • Make interactive elements (buttons) keyboard accessible
  • Use proper semantic HTML for content structure

Best Practices

  • Consistent Sizing: Keep cards in a grid for visual harmony
  • Clear Hierarchy: Use headings and spacing to guide focus
  • Concise Content: Keep text brief and scannable
  • Responsive: Cards should stack on mobile devices
  • Actions: Include clear call-to-action buttons when needed
  • Visual Interest: Use images, badges, or icons to enhance cards
