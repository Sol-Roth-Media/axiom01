Masonry

Responsive masonry layout for displaying content in columns.

Masonry Component

A responsive grid layout component with three powerful modes: masonry (CSS Grid auto-fill), file browser, and gallery. Perfect for portfolios, image galleries, and file management interfaces.

Overview

The Masonry component adapts to three different use cases:

  • Masonry Mode: CSS Grid with auto-fill for flowing layouts
  • File Mode: Icon-based file browser with download capabilities
  • Gallery Mode: Image gallery with hover overlays and aspect ratio variations

Basic Masonry Layout

HTML Structure

<div data-component="masonry"
     data-mode="masonry"
     data-columns="3">
</div>

Attributes

Attribute Values Description
data-mode "masonry" | "files" | "gallery" Layout mode to use
data-columns 1-6 Number of columns (masonry mode only)
data-gap CSS value Gap between items (default: var(--a-space-l))

JavaScript API

const masonryEl = document.querySelector('[data-component="masonry"]');
const api = window.componentInstances.get(masonryEl);

// Add item
api.addItem({ id: 1, name: "Item", image: "url" });

// Set all items
api.setItems([...]);

// Get items
const items = api.getItems();

// Change columns
api.setColumns(4);

// Change mode
api.setMode("gallery");

Data Formats

Masonry Mode

{
  id: "item-1",
  title: "Project Name",
  description: "Project description",
  image: "url-to-image.jpg"
}

File Mode

{
  id: "file-1",
  name: "document.pdf",
  type: "pdf" /* or: image, video, audio, document, folder */,
  size: 1024 // in bytes
}

Gallery Mode

{
  id: "photo-1",
  image: "url-to-image.jpg",
  title: "Photo Title",
  description: "Photo description",
  aspect: "wide" /* or: "tall", default is normal */
}

Events

Event Detail
file-selected { item, index }
gallery-selected { item, index }
item-removed { item, index }
item-added { item }

Best Practices

  • Gallery Images: Use optimized JPG/WebP with aspect ratios
  • File Icons: Component auto-detects file type from the type attribute
  • Performance: Virtual scrolling recommended for 1000+ items
  • Responsive: Adjust columns based on viewport with media queries
