HTML Structure

Basic Select

<select>
  <option value="">Choose an option</option>
  <option value="opt1">Option 1</option>
  <option value="opt2">Option 2</option>
  <option value="opt3">Option 3</option>
</select>

Select with Grouped Options

<select>
  <optgroup label="Fruits">
    <option>Apple</option>
    <option>Banana</option>
  </optgroup>
  <optgroup label="Vegetables">
    <option>Carrot</option>
    <option>Broccoli</option>
  </optgroup>
</select>

Multiple Select

<select multiple size="5">
  <option selected>Item 1</option>
  <option selected>Item 2</option>
  <option>Item 3</option>
  <option>Item 4</option>
</select>

Select with Datalist (Autocomplete)

<input list="browsers" placeholder="Type or select...">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
</datalist>

JavaScript API

Getting the Selected Value

const select = document.querySelector('select');

// Get the selected option value
const selectedValue = select.value;
console.log(selectedValue); // "option1"

// Get the selected option text
const selectedText = select.options[select.selectedIndex].text;
console.log(selectedText); // "Option 1"

// Listen for changes
select.addEventListener('change', (e) => {
  console.log('Selected:', e.target.value);
});

Programmatic Value Setting

const select = document.querySelector('select');

// Set the selected value
select.value = 'option2';

// Reset to first option
select.selectedIndex = 0;

// Disable the select
select.disabled = true;

// Get all selected options (for multiple)
const selected = Array.from(select.selectedOptions);
selected.forEach(opt => console.log(opt.value));

Accessibility

Best Practices

  • Always pair <select> with <label> for clarity
  • Use descriptive option text
  • Provide a placeholder option (empty value first option)
  • Use <optgroup> for logical grouping
  • Screen readers announce all options automatically
  • Native keyboard support requires no code
  • Works on all devices (desktop, tablet, mobile)

AXIOM01 Philosophy

Why Use Native <select>

The native <select> element demonstrates AXIOM01 principles perfectly:

  • Semantic: <select> tells the browser exactly what it is
  • Accessible: Built-in screen reader support, keyboard navigation
  • Minimal Code: Zero JavaScript needed for full functionality
  • Framework-Only Styling: CSS handles appearance, one class if needed
  • Works Everywhere: All browsers, all devices, responsive by default

Custom dropdown components require hundreds of lines of JavaScript and lose built-in accessibility. Use <select> first.
