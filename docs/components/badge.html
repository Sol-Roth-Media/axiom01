Badge

Axiom01's badge component displays small, labeled indicator elements. Use badges to highlight status, count, or categorize content with semantic color coding.

Basic HTML

<!-- Default badge -->
<span class="badge badge-default">Default</span>

<!-- Primary badge -->
<span class="badge badge-primary">Primary</span>

<!-- Status badges -->
<span class="badge badge-success">Success</span>
<span class="badge badge-warning">Warning</span>
<span class="badge badge-error">Error</span>
<span class="badge badge-info">Info</span>

With Indicator Dot

Add the badge-dot class to display a colored dot next to the text:

<span class="badge badge-success badge-dot">Online</span>
<span class="badge badge-warning badge-dot">Pending</span>
<span class="badge badge-error badge-dot">Offline</span>

Color Variants

  • badge-default - Neutral badge with border
  • badge-primary - Primary brand color
  • badge-secondary - Secondary color
  • badge-success - Green for successful/positive states
  • badge-warning - Orange for warnings/alerts
  • badge-error - Red for errors/critical states
  • badge-info - Blue for informational content

Use Cases

  • Status Indicators: Show online/offline, draft/published, active/inactive
  • Labels: Categorize items with tags (Bug, Feature, Documentation)
  • Counts: Display notification badges, unread messages, etc.
  • Achievements: Highlight badges, certifications, or special status
  • Metadata: Show versions, priority levels, or other attributes

Accessibility

  • Badges with dot indicators provide visual + textual information
  • Use semantic color coding that's intuitive (green = good, red = error)
  • Ensure sufficient contrast between text and background
  • Never rely on color alone to convey meaning - include text
  • For status indicators, consider using aria-label for additional context

Best Practices

  • Keep Text Short: Badges work best with 1-2 words
  • Use Semantic Colors: Match colors to meaning (green=success, red=error)
  • Consistent Sizing: All badges use the same height for visual harmony
  • Avoid Nesting: Don't place badges inside interactive elements
  • Context Matters: Place badges near relevant content
