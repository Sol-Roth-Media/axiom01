CSS Classes & Data Attributes

<ol role="tablist">

Container for step indicator. Use semantic <ol> for ordered list.

<li role="tab">

Individual step. Semantic list item with tab role.

data-active="true"

Marks current active step. Only one step should have this at a time.

data-completed="true"

Marks completed steps. Shows as filled/checkmark.

data-variant="vertical"

Displays steps vertically instead of horizontally.

data-variant="vertical-responsive"

Vertical on mobile, horizontal on desktop.

JavaScript API

Update Active Step

const steps = document.querySelectorAll('[role="tab"]');

// Move to step 3
steps[2].setAttribute('data-active', 'true');
steps[1].removeAttribute('data-active');

// Mark previous steps as completed
steps.forEach((step, index) => {
  if (index < 2) {
    step.setAttribute('data-completed', 'true');
  }
});

Move to Next Step

function nextStep() {
  const steps = document.querySelectorAll('[role="tab"]');
  const active = document.querySelector('[role="tab"][data-active]');
  const activeIndex = Array.from(steps).indexOf(active);
  
  // Mark current as completed
  active.setAttribute('data-completed', 'true');
  active.removeAttribute('data-active');
  
  // Move to next step
  if (activeIndex < steps.length - 1) {
    steps[activeIndex + 1].setAttribute('data-active', 'true');
  }
}

Get Current Step

const currentStep = document.querySelector('[role="tab"][data-active]');
if (currentStep) {
  const stepNumber = currentStep.querySelector('span:first-child').textContent;
  console.log(`Currently on step ${stepNumber}`);
}

Best Practices

  • Use semantic <ol> for ordered steps
  • Add role="tablist" to container and role="tab" to items
  • Include step labels and numbers for clarity
  • Update ARIA attributes when steps change
  • Provide step descriptions for complex processes
  • Ensure color alone doesn't convey state (use icons/text)
  • Test with screen readers (NVDA, JAWS, VoiceOver)

AXIOM01 Philosophy

Why Use Semantic Steps

Steps embody AXIOM01 principles:

  • Semantic: Native <ol> list indicates ordered sequence
  • Accessible: Screen readers announce steps and states automatically
  • Data-Driven: States controlled by data attributes, not class mutations
  • Minimal JavaScript: Pure data attribute manipulation
  • Responsive: Layout adapts to screen size via CSS
  • Maintainable: Clear HTML structure, no custom classes

Custom step implementations often break accessibility by using divs instead of lists. Use semantic steps first.
