Empty State Component

Provide helpful guidance when content is unavailable, loading, or when users need to take action to see content.

Basic Empty State

Standard empty state with call-to-action for when no content is available.

No items found

There are no items to display right now.

<section class="empty" role="status">
  <figure aria-hidden="true">
    <i class="fas fa-inbox"></i>
  </figure>
  <h3>No items found</h3>
  <p>There are no items to display right now.</p>
  <button class="primary">Add New Item</button>
</section>

Search Results Empty State

Empty state specifically for search results with multiple action options.

No Results Found

We couldn't find anything matching "your search".

<section class="empty search" role="status" aria-label="No search results">
  <figure aria-hidden="true">🔍</figure>
  <h3>No Results Found</h3>
  <p>We couldn't find anything matching "<span class="highlight">your search</span>".</p>
  <footer>
    <button class="secondary">Clear Search</button>
    <button class="tertiary">Browse All Items</button>
  </footer>
</section>

Loading State

Loading indicator for when content is being fetched.

Loading...

Please wait while we fetch your content.

<section class="empty loading" role="status" aria-label="Loading content">
  <figure aria-hidden="true">
    <i class="fas fa-spinner fa-spin"></i>
  </figure>
  <h3>Loading...</h3>
  <p>Please wait while we fetch your content.</p>
</section>

Error State

Error state for when something goes wrong with content loading.

Something went wrong

We encountered an error while loading your content. Please try again.

<section class="empty error" role="alert" aria-label="Content loading error">
  <figure aria-hidden="true">
    <i class="fas fa-exclamation-triangle"></i>
  </figure>
  <h3>Something went wrong</h3>
  <p>We encountered an error while loading your content.</p>
  <footer>
    <button class="primary">Try Again</button>
    <button class="secondary">Contact Support</button>
  </footer>
</section>

Accessibility Features

  • Live Regions: Proper role attributes for screen readers
  • Semantic Structure: Uses section, figure, and footer elements
  • ARIA Labels: Clear descriptions for different states
  • Focus Management: Primary actions are easily accessible
  • Clear Messaging: Helpful instructions and next steps
