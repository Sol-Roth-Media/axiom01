Editor Component

The Editor component in Axiom01 provides foundational styling for standard HTML <textarea> elements, making them visually consistent with the rest of the framework. For advanced rich text editing capabilities (like WYSIWYG features, formatting tools, etc.), Axiom01 is designed to integrate seamlessly with third-party JavaScript rich text editor libraries.

This approach allows developers the flexibility to choose their preferred rich text editor while ensuring the base input area adheres to Axiom01's design system.

Usage Guidelines

To use Axiom01's base editor styling:

  • Apply the .editor-placeholder class to your <textarea> element.
  • Ensure your <textarea> has an associated <label> for accessibility.

For integrating a rich text editor library:

  • Initialize your chosen third-party library (e.g., CKEditor, TinyMCE, Quill) on the <textarea> element that has the .editor-placeholder class.
  • Most modern rich text editors offer extensive customization options, including the ability to leverage CSS custom properties (variables) for theming. This allows you to align the editor's UI with your Axiom01 theme.
  • Refer to the documentation of your chosen rich text editor for specific integration and theming instructions.

Third-Party Integration

Axiom01's editor styling is a wrapper for popular rich text editor libraries. Some widely used options include:

  • CKEditor: A powerful WYSIWYG editor with extensive features.
  • TinyMCE: Another popular and highly customizable rich text editor.
  • Quill: A modern, open-source WYSIWYG editor built for compatibility and extensibility.

When integrating, prioritize libraries that support CSS variable-based theming to maintain visual consistency with Axiom01.

Accessibility

While Axiom01 provides a semantic base, the full accessibility of a rich text editor largely depends on the chosen third-party library and its configuration. When selecting and implementing an editor:

  • Ensure the library itself is WCAG compliant.
  • Properly associate labels with the editor instance.
  • Verify keyboard navigation and screen reader compatibility for all editor controls and content areas.
  • Provide clear instructions for users on how to create accessible content within the editor.

Customization

The base styling of the .editor-placeholder can be customized using the following CSS variables:

  • --a-color-surface: Background color of the textarea.
  • --a-color-on-surface: Text color within the textarea.
  • --a-color-outline: Border color of the textarea.
  • --a-border-radius-medium: Border radius.
  • --a-space-m: Internal padding.
  • --a-font-family-sans, --a-font-size-base: Font styles.

For customizing the integrated rich text editor, consult its specific theming documentation.
