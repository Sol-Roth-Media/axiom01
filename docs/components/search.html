HTML Structure

The search component uses a minimal HTML structure, relying on data-* attributes for JavaScript targeting and semantic elements for accessibility.

<!-- Search Toggle Button (can be placed anywhere, e.g., in header) -->
<button class="search toggle" aria-label="Open Search" aria-controls="search-dialog" aria-expanded="false">
<i class="fas fa-search" aria-hidden="true"></i>
</button>

<!-- Search Overlay Structure -->
<div id="search-modal" class="search-modal is-hidden" data-component="search" aria-modal="true" role="dialog">
<div class="search-modal-content">
    <header>
        <h3>Search Documentation</h3>
        <button class="close" aria-label="Close Search">&times;</button>
    </header>
    <div class="body">
        <input type="search" id="search-modal-input" placeholder="Search components, guides, and more...">
        <div id="search-modal-results" class="search-results">
            <!-- Search results will be dynamically inserted here -->
        </div>
    </div>
</div>
</div>

JavaScript Initialization

The index-page-manager.js script automatically initializes any element with data-component="search". The search functionality relies on the Fuse.js library.

The search component expects a global searchData array to be available, which contains the content to be searched. This data should be an array of objects, where each object has at least a title, category, and url property.

Required Markup Contract

  • Use data-component="search" on the modal/dialog root element.
  • Use a trigger button with data-search-toggle (or the legacy .search.toggle class pattern).
  • Inside the modal, include elements using data-search-close, data-search-input, and data-search-results.
  • The component manages aria-expanded on the trigger and aria-hidden on the modal state.
// Example searchData structure (must be globally accessible before search.js initializes)
const searchData = [
{ title: "Accordion", category: "feedback", url: "docs/components/accordion.html" },
{ title: "AI Chat", category: "ai", url: "docs/components/ai-chat.html" },
// ... more data
];

// The initSearchModal function in index-page-manager.js will automatically pick this up.
// Ensure Fuse.js is loaded before index-page-manager.js.

Accessibility

The search component is designed with accessibility in mind:

  • The main search overlay has aria-modal="true" and role="dialog".
  • Toggle and close buttons have descriptive aria-label attributes.
  • The toggle button's aria-controls links it to the search dialog, and aria-expanded indicates its state.
  • Focus is managed to trap within the modal when open and return to the trigger when closed.
  • Keyboard navigation is supported: Escape key closes the modal, and arrow keys can navigate search results.

Customization

The appearance of the search modal can be customized using CSS variables defined in css/axiom.css:

  • --a-color-overlay-bg: Background color of the modal overlay.
  • --a-color-surface: Background color of the search content area.
  • --a-color-outline: Border color for input and separators.
  • --a-color-on-surface: Text color.
  • --a-color-on-surface-variant: Secondary text color (e.g., placeholders, small text).
  • --a-color-primary: Primary accent color.
  • --a-border-radius-medium: General border radius.
  • --a-shadow-large: For the modal's shadow.
  • --a-space-*: Various spacing variables for padding and margins.
