Search

Axiom01's search component provides a powerful, accessible way to find content. Features real-time filtering, keyboard navigation, and customizable results display.

HTML Structure

<div class="search-container">
				<span class="search-icon">
				<span class="axicon render" data-name="Search"></span>
				</span>

				<input
				type="text" class="search-input"
				placeholder="Search..."
				aria-label="Search"
				aria-autocomplete="list"
				aria-controls="results"
				>

				<button class="search-clear" aria-label="Clear search">
				<span class="axicon render" data-name="X"></span>
				</button>
				</div>

				<div class="search-results" id="results" role="listbox">
				<div class="search-result-item" role="option">
				<h3 class="result-title">Result Title</h3>
				<p class="result-desc">Result description</p>
				</div>
				</div>

JavaScript Implementation

const searchInput = document.getElementById('searchInput');
				const searchResults = document.getElementById('searchResults');
				const searchData = [
				{ title: 'Product A', desc: 'Description A' },
				{ title: 'Product B', desc: 'Description B' },
				// ... more data
				];

				searchInput.addEventListener('input', (e) => {
				const query = e.target.value.toLowerCase();
				const clearBtn = document.querySelector('.search-clear');

				if (query) {
				clearBtn.classList.add('visible');
				const results = searchData.filter(item =>
				item.title.toLowerCase().includes(query)
				);

				displayResults(results);
				} else {
				clearBtn.classList.remove('visible');
				searchResults.classList.remove('visible');
				}
				});

				function displayResults(results) {
				searchResults.innerHTML = results.length
				? results.map(r => `
				<div class="search-result-item">
				<h3 class="result-title">${r.title}</h3>
				<p class="result-desc">${r.desc}</p>
				</div>
				`).join('')
				: '<div class="search-no-results">No results found</div>';

				searchResults.classList.add('visible');
				}

Features

  • Real-time Search - Instant filtering as user types
  • Clear Button - Quick way to reset search
  • Search Icon - Visual indication of search field
  • No Results Message - Feedback when nothing matches
  • Keyboard Navigation - Full keyboard support
  • Filter Support - Filter results by category

Use Cases

  • Product Search - E-commerce product discovery
  • Documentation - API or guide search
  • User Directory - Find team members or contacts
  • Content Discovery - Blog or article search
  • Settings - Search for options and preferences
  • Command Palette - Quick command lookup

Accessibility

  • Use semantic input element with appropriate aria-label
  • Include aria-autocomplete="list" for autocomplete behavior
  • Use role="listbox" for results container
  • Use role="option" for individual results
  • Support keyboard navigation (arrows, enter, escape)
  • Announce search results count to screen readers

Best Practices

  • Debounce Input - Wait before filtering for large datasets
  • Highlight Matches - Emphasize matching text in results
  • Limit Results - Show max 10 results to avoid clutter
  • Case Insensitive - Match regardless of case
  • Clear on Submit - Navigate to result and clear input
  • Search History - Optional: show recent searches
