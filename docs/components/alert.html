Semantic HTML Structure

Alert component uses semantic HTML for proper meaning and screen reader compatibility.

Basic Alert Structure

The alert component uses semantic <div> with .alert class, paired with an icon and content area:

<div class="alert success" role="alert">
	<span class="axicon render" data-name="Check-Circle"></span>
	<div>
		<strong>Alert Title</strong>
		<p>Alert message content here.</p>
	</div>
</div>

Framework Classes

Only semantic framework classes. No utilities. No custom styles.

Available Classes

Class Purpose Example
.alert Main alert container <div class="alert">
.success Success state styling <div class="alert success">
.info Information state styling <div class="alert info">
.warning Warning state styling <div class="alert warning">
.error Error state styling <div class="alert error">

Accessibility & Best Practices

WCAG 2.1 AA compliance and inclusive design patterns.

Accessibility Features

  • Semantic Role: Use role="alert" for urgent alerts to notify screen reader users immediately
  • Color + Icon: Don't rely on color alone; use icons to indicate severity
  • Clear Text: Message text should be clear and actionable
  • High Contrast: Colors meet WCAG AA standards for contrast
  • Focus Management: Manage focus appropriately for alert placement
  • Keyboard Navigation: Ensure any interactive elements are keyboard accessible

Design Tokens

All styling uses design tokens for consistency and theming:

/* Alert styling uses semantic color tokens */
.alert {
	background-color: var(--a-color-surface);
	border: 1px solid var(--a-color-outline);
	padding: var(--a-space-l);
	border-radius: var(--a-border-radius-base);
	display: flex;
	align-items: center;
	gap: var(--a-space-m);
}

.alert.success {
	background-color: var(--a-color-success-container);
	color: var(--a-color-on-success-container);
	border-color: var(--a-color-success);
}
