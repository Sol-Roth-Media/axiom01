Alert

Contextual feedback messages. One root class — alert — with a semantic modifier for type. Fully accessible with ARIA live regions.

Alert Types

<div class="alert info" role="status">
  <span class="axicon render" data-name="Info"></span>
  <div>
    <strong>Information</strong>
    <p>This is an informational message.</p>
  </div>
</div>

<div class="alert success" role="status">
  <span class="axicon render" data-name="Check-Circle"></span>
  <div>
    <strong>Success</strong>
    <p>Your changes have been saved.</p>
  </div>
</div>

<div class="alert warning" role="alert">
  <span class="axicon render" data-name="Alert-Triangle"></span>
  <div>
    <strong>Warning</strong>
    <p>Please review before continuing.</p>
  </div>
</div>

<div class="alert error" role="alert">
  <span class="axicon render" data-name="Alert-Circle"></span>
  <div>
    <strong>Error</strong>
    <p>An error occurred. Please try again.</p>
  </div>
</div>

Dismissible Alerts

Add a close button with data-alert-close for dismissible alerts:

<div class="alert success" role="status">
  <span class="axicon render" data-name="Check-Circle"></span>
  <div>
    <strong>File uploaded</strong>
    <p>Your file has been saved successfully.</p>
  </div>
  <button class="close" aria-label="Dismiss alert" data-alert-close>
    <span class="axicon render" data-name="X"></span>
  </button>
</div>

Text-Only Alert

<div class="alert info" role="status">
  <p>Your session will expire in 5 minutes.</p>
</div>

Variants

ClassUseARIA Role
alert infoNeutral information, tips, hintsrole="status"
alert successConfirmations, completed actionsrole="status"
alert warningPotential issues requiring attentionrole="alert"
alert errorFailed actions, validation errorsrole="alert"

Accessibility

  • Use role="alert" for critical messages (errors, warnings) — announced immediately by screen readers
  • Use role="status" for non-critical messages (success, info) — announced politely
  • Include aria-live="polite" on a container if injecting alerts dynamically
  • Icons are decorative — always pair with text, mark icons aria-hidden="true"
  • Dismiss button needs aria-label — the X icon alone is not accessible
