Alert Component

Alerts provide contextual feedback messages for typical user actions. They can be used for success messages, warnings, errors, and general information.

Basic Alerts

Basic alerts are static and non-dismissible. Use them for information that should persist on the page.

This is a basic informational alert.
<div class="alert alert-info">
    <i class="fas fa-info-circle alert-icon"></i>
    <div class="alert-content">This is a basic informational alert.</div>
</div>

Alert Colors

Use contextual color classes to indicate the type of message.

Success! Your profile has been updated.
Warning: Your trial period is about to expire.
Error: Could not save your changes. Please try again.
<!-- Success -->
<div class="alert alert-success">...</div>

<!-- Warning -->
<div class="alert alert-warning">...</div>

<!-- Error -->
<div class="alert alert-error">...</div>

Dismissible Alerts

Add the `.alert-dismissible` class and a close button to create a dismissible alert. The JavaScript for this is handled automatically by `axiom.js`.

You can dismiss this success alert.
<div class="alert alert-success alert-dismissible">
    <i class="fas fa-check-circle alert-icon"></i>
    <div class="alert-content">You can dismiss this success alert.</div>
    <button class="alert-close" data-alert-close>&times;</button>
</div>

Dynamic Alerts (Toasts)

Use the `AxiomComponents.Alert.show()` JavaScript method to create dynamic "toast" notifications. These are useful for providing feedback on asynchronous actions.

// Show a success toast
AxiomComponents.Alert.show('success', 'This is a success toast.');

// Show an error toast that auto-dismisses after 5 seconds
AxiomComponents.Alert.show('error', 'This will disappear in 5s.', { autoDismiss: 5000 });

Accessibility

  • Alerts have a `role` of `alert` to announce them to screen readers.
  • Icons are used for visual cues, but are not essential for understanding the alert's meaning.
  • Dismissible alerts have a close button with an `aria-label` for screen readers.
