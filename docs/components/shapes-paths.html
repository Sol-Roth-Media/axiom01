Basic Shapes

<!-- Circle -->
				<svg viewBox="0 0 120 120">
				<circle cx="60" cy="60" r="40" stroke="currentColor" fill="none"/>
				</svg>

				<!-- Rectangle -->
				<svg viewBox="0 0 120 120">
				<rect x="20" y="20" width="80" height="80" rx="8" stroke="currentColor" fill="none"/>
				</svg>

				<!-- Polygon (Triangle) -->
				<svg viewBox="0 0 120 120">
				<polygon points="60,20 100,100 20,100" stroke="currentColor" fill="none"/>
				</svg>

Path Element

The <path> element is the most powerful SVG shape for creating custom paths:

<!-- Bezier Curve -->
				<svg viewBox="0 0 120 120">
				<path d="M20 100 Q60 20 100 100" stroke="currentColor" fill="none"/>
				</svg>

				<!-- Wave Path -->
				<svg viewBox="0 0 120 120">
				<path d="M10 60 Q30 40 50 60 T90 60" stroke="currentColor" fill="none"/>
				</svg>

				<!-- Custom Shape -->
				<svg viewBox="0 0 120 120">
				<path d="M60 10 L75 40 L105 40 L80 60 L90 90 L60 70 L30 90 L40 60 L15 40 L45 40 Z" fill="currentColor"/>
				</svg>

Using CSS and Variables

Style SVG shapes with CSS variables for consistent theming:

<svg viewBox="0 0 120 120">
				<circle cx="60" cy="60" r="40"
				stroke="var(--a-color-primary)"
				fill="var(--a-color-primary-container)"
				stroke-width="2"/>
				</svg>

				<style>
				svg circle {
				transition: stroke 0.3s ease;
				}

				svg:hover circle {
				stroke: var(--a-color-secondary);
				}
				</style>

Accessibility

  • Always include a <title> element inside SVG for context
  • Use aria-labelledby to reference the title
  • Add role="img" for decorative SVGs that convey meaning
  • Use semantic color coding (primary, success, warning, error)
  • Ensure sufficient contrast for visibility
  • Provide aria-hidden="true" for purely decorative SVGs

Use Cases

  • Data Visualization: Charts, graphs, progress indicators
  • Diagrams: Flowcharts, process flows, hierarchies
  • Decorative Elements: Borders, patterns, backgrounds
  • Interactive Graphics: Animated shapes, interactive paths
  • Custom Icons: Brand-specific or specialized icons
  • Animations: Animated strokes, morphing shapes

Best Practices

  • Use viewBox: Always include viewBox for scalable SVGs
  • CSS Styling: Use CSS variables for colors that work with dark mode
  • Semantic Markup: Include titles and descriptions
  • Performance: Optimize paths and remove unnecessary elements
  • Accessibility: Always provide context and alternative text
  • Consistency: Match stroke widths and styling with design system
