JavaScript API

Getting and Setting Values

const slider = document.querySelector('input[type="range"]');

// Get current value
console.log(slider.value); // "50"

// Set value programmatically
slider.value = 75;

// Get min/max
console.log(slider.min);   // "0"
console.log(slider.max);   // "100"
console.log(slider.step);  // "1"

Event Handling

const slider = document.querySelector('input[type="range"]');

// Listen for changes (fires during dragging)
slider.addEventListener('input', (e) => {
  console.log('Current value:', e.target.value);
});

// Listen for change (fires after release)
slider.addEventListener('change', (e) => {
  console.log('Final value:', e.target.value);
});

// Update output element
const output = document.querySelector('output');
slider.addEventListener('input', (e) => {
  output.textContent = e.target.value;
});

Accessibility

Best Practices

  • Always pair with <label> for clarity
  • Use <output> to display current value
  • Provide text alternative input as backup
  • Test keyboard navigation thoroughly
  • Screen readers announce min, max, and current value
  • Ensure sufficient thumb size for touch devices
  • Avoid rapid changes that cause confusion

AXIOM01 Philosophy

Why Use Native Range Input

The native range input embodies AXIOM01 principles:

  • Semantic: <input type="range"> tells the browser exactly what it is
  • Accessible: Built-in keyboard support and screen reader announcements
  • Minimal Code: No custom JavaScript needed for basic functionality
  • Touch-Friendly: Works perfectly on mobile devices
  • Browser Native: Uses OS-level controls
  • Responsive: Scales automatically to container

Custom sliders require hundreds of lines of code, complex JavaScript, and often break on edge cases. Use <input type="range"> first.
