Sidebar

Axiom01's sidebar component is a flexible layout component for navigation, supplementary content, or application menus. It adapts responsively to different screen sizes.

HTML Structure

<div class="sidebar-layout">
    <aside role="complementary" aria-label="Sidebar Navigation">
        <h3>Navigation</h3>
        <nav>
            <ul>
                <li><a href="#" class="active">Dashboard</a></li>
                <li><a href="#">Projects</a></li>
                <li><a href="#">Team</a></li>
                <li><a href="#">Settings</a></li>
            </ul>
        </nav>
    </aside>
    <main>
        <h4>Main Content</h4>
        <p>Content goes here...</p>
    </main>
</div>

Features

  • Flexible Layout - Use flexbox or CSS Grid for layout
  • Responsive Design - Adapts from fixed width to collapsible/stacked on mobile
  • Icon Support - Display icons with text or collapse to icon-only
  • Semantic HTML - Uses <aside> and <nav> for proper structure
  • Active States - Highlight current page/section with active class
  • Accessibility - Proper ARIA labels and semantic roles

CSS Implementation

/* Sidebar layout */
.sidebar-layout {
    display: flex;
}

/* Sidebar styling */
.sidebar-layout aside {
    width: 250px;
    background: var(--a-color-primary-container);
    padding: var(--a-space-l);
    border-right: 1px solid var(--ax-border);
}

/* Navigation links */
.sidebar-layout nav a {
    display: block;
    padding: var(--a-space-m);
    color: var(--a-color-primary);
    text-decoration: none;
    border-radius: var(--a-border-radius-md);
    transition: background-color var(--a-transition-base) var(--a-ease-out);
}

.sidebar-layout nav a:hover {
    background: rgba(0, 102, 204, 0.1);
}

.sidebar-layout nav a.active {
    background: var(--a-color-primary);
    color: white;
}

/* Main content */
.sidebar-layout main {
    flex: 1;
    padding: var(--a-space-l);
}

/* Responsive */
@media (max-width: 768px) {
    .sidebar-layout {
        flex-direction: column;
    }
    
    .sidebar-layout aside {
        width: 100%;
        border-right: none;
        border-bottom: 1px solid var(--ax-border);
    }
}

Use Cases

  • Application Navigation - Main navigation for web applications and dashboards
  • Documentation Sites - Table of contents or navigation sidebar
  • Blog Sidebars - Recent posts, categories, or search
  • Admin Panels - Menu and settings navigation
  • E-commerce - Product filters and categories

Accessibility

  • Semantic HTML - Use <aside> with role="complementary"
  • Navigation - Use <nav> element for navigation items
  • ARIA Labels - Add aria-label to identify sidebar purpose
  • Active State - Indicate current page with visual and text indicators
  • Keyboard Navigation - All links and buttons must be keyboard accessible
  • Mobile Considerations - Ensure sidebar doesn't block content on small screens

Best Practices

  • Clear Navigation - Keep sidebar navigation simple and organized
  • Visual Hierarchy - Group related items, use indentation for hierarchy
  • Responsive Collapse - Sidebar should collapse or stack on mobile
  • Consistent Width - Standard sidebar widths: 200-300px
  • Easy to Hide - On mobile, provide button to show/hide sidebar
  • Scrollable Content - Long sidebars should have internal scroll
