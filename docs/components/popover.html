Popover Component

Contextual overlay panels triggered by a details element. Ideal for dropdown menus, action menus, and contextual information panels. Uses native HTML `<details>` for semantic structure.

HTML Structure

Uses native `<details>` and `<summary>` elements with a `<nav>` for menu content:

<details class="popover">
  <summary>Menu</summary>
  <nav>
    <a href="#">Option 1</a>
    <a href="#">Option 2</a>
    <button>Action</button>
  </nav>
</details>

Features

  • Semantic HTML: Uses native `<details>` and `<summary>` elements
  • Keyboard Navigation: Space/Enter to toggle, Escape to close
  • Auto-close: Popovers auto-close when clicking outside
  • Flexible Content: Support links, buttons, or any content inside nav
  • Positioning: Automatically positioned relative to trigger button
  • Accessibility: Full keyboard and screen reader support

JavaScript Behavior

Automatic global handling provided by AXIOM01 core:

  • Global click handler closes all open details elements
  • Escape key closes all open popovers and dialogs
  • Multiple popovers on page are handled independently

Styling Customization

Use AXIOM01 design tokens for consistent styling:

/* Override default popover styling */
details.popover nav {
  /* --z-dropdown controls z-index */
  /* --a-shadow-md controls shadow depth */
  /* --a-color-outline controls border color */
  /* --a-color-surface controls background */
}

/* Style menu items */
details.popover nav a,
details.popover nav button {
  /* --a-space-m controls padding */
  /* --a-color-on-surface controls text color */
  /* --a-transition-base controls hover animation */
}

Design Tokens Used

  • --a-space-s - Padding in summary button
  • --a-space-l - Horizontal padding in summary
  • --a-space-m - Vertical padding for menu items
  • --a-color-outline - Border color
  • --a-color-surface - Background color
  • --a-color-on-surface - Text color
  • --a-color-primary - Hover state color
  • --a-shadow-md - Popover shadow
  • --a-transition-base - Smooth transitions
  • --z-dropdown - Z-index stacking
  • --a-border-radius-base - Border radius

Best Practices

  • Clear Triggers: Make the summary text clearly indicate it's clickable (use "Menu ▼" or similar)
  • Consistent Options: Use similar action types grouped together (links vs buttons)
  • Keyboard Support: All items should be keyboard accessible
  • Mobile Friendly: Popovers work well on mobile with touch interactions
  • Accessibility: Always use semantic HTML elements within the nav
