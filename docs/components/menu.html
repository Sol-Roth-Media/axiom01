Menu

Context and action menus using <menu role="menu">. Keyboard navigable, ARIA-compliant.

Context Menu

<menu role="menu" data-open="true" aria-label="Item actions">
  <button role="menuitem">
    <span class="axicon render" data-name="Edit"></span> Edit
  </button>
  <button role="menuitem" data-variant="danger">
    <span class="axicon render" data-name="Trash"></span> Delete
  </button>
</menu>

Toggle Visibility

// Show
menu.dataset.open = 'true';
// Hide
delete menu.dataset.open;

// Close on outside click
document.addEventListener('click', e => {
  if (!menu.contains(e.target) && !trigger.contains(e.target)) {
    delete menu.dataset.open;
  }
});

Keyboard Navigation

menu.addEventListener('keydown', e => {
  const items = [...menu.querySelectorAll('[role="menuitem"]')];
  const idx = items.indexOf(document.activeElement);
  if (e.key === 'ArrowDown') items[(idx + 1) % items.length].focus();
  if (e.key === 'ArrowUp') items[(idx - 1 + items.length) % items.length].focus();
  if (e.key === 'Escape') { delete menu.dataset.open; trigger.focus(); }
});

Accessibility

  • Trigger button needs aria-haspopup="menu" and aria-expanded
  • Arrow keys navigate between menuitem buttons
  • Escape closes and returns focus to trigger
  • Use <hr> to separate groups — not wrapper divs
