CSS Classes & Attributes

menu[role="menu"]

Semantic menu container with ARIA role.

button[role="menuitem"]

Individual menu item.

data-open="true|false"

Toggles menu visibility.

aria-haspopup="menu"

Indicates button opens menu.

aria-expanded="true|false"

Shows if menu is open/closed.

<hr>

Visual divider between menu items.

JavaScript API

Toggle Menu

const trigger = document.querySelector('button[aria-haspopup="menu"]');
const menu = document.getElementById('menu-1');

trigger.addEventListener('click', () => {
  const isOpen = menu.getAttribute('data-open') === 'true';
  menu.setAttribute('data-open', !isOpen);
  trigger.setAttribute('aria-expanded', !isOpen);
});

Handle Menu Item Click

const items = document.querySelectorAll('[role="menuitem"]');

items.forEach(item => {
  item.addEventListener('click', (e) => {
    console.log('Selected:', item.textContent);
    
    // Close menu after selection
    menu.setAttribute('data-open', 'false');
    trigger.setAttribute('aria-expanded', 'false');
  });
});

AXIOM01 Philosophy

Why Use Semantic Menus

Menus embody AXIOM01 principles:

  • Semantic: Native <menu> with proper ARIA roles
  • Accessible: Screen readers understand menu structure
  • Keyboard: Full keyboard navigation support
  • Minimal: No framework dependencies
  • Flexible: Works with any trigger element
  • Progressive: Enhanced with JavaScript
