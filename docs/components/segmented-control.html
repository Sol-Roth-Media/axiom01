CSS Classes & Data Attributes

.segmented-control

Semantic <fieldset> container for grouped buttons.

<legend>

Accessible label for the button group.

data-selected="true"

Marks the currently selected/active button.

disabled

Native HTML disabled attribute for inactive buttons.

aria-label

For icon-only buttons, provide accessible label.

JavaScript API

Select Option

const fieldset = document.querySelector('.segmented-control');
const buttons = fieldset.querySelectorAll('button');

// Click handler
buttons.forEach((btn, index) => {
  btn.addEventListener('click', () => {
    // Remove selection from all buttons
    buttons.forEach(b => b.removeAttribute('data-selected'));
    
    // Select clicked button
    btn.setAttribute('data-selected', 'true');
    
    console.log(`Selected option: ${index}`);
  });
});

Get Selected Value

function getSelectedOption() {
  const selected = document.querySelector(
    '.segmented-control button[data-selected="true"]'
  );
  return selected ? selected.textContent : null;
}

console.log(getSelectedOption()); // "Option 2"

Set Selected Value

function setSelectedOption(index) {
  const buttons = document.querySelectorAll(
    '.segmented-control button'
  );
  
  buttons.forEach(btn => btn.removeAttribute('data-selected'));
  if (buttons[index]) {
    buttons[index].setAttribute('data-selected', 'true');
  }
}

// Set to 3rd option
setSelectedOption(2);

Features

Semantic Fieldset

Uses <fieldset> and <legend> for proper form grouping.

Mutually Exclusive

Only one button can be selected at a time.

Icon Support

Combine text and icons, or use icons only.

Flexible Count

Support any number of options with responsive layout.

Disabled State

Native disabled attribute for inactive options.

Accessible

ARIA labels and semantic HTML throughout.

Accessibility

Best Practices

  • Use semantic <fieldset> with <legend>
  • Ensure one option is always selected (data-selected)
  • Use native disabled for unavailable options
  • Provide aria-label for icon-only buttons
  • Manage focus between button options
  • Announce selection changes to screen readers
  • Use keyboard navigation (Tab, Arrow keys)
  • Test with NVDA, JAWS, VoiceOver

AXIOM01 Philosophy

Why Use Semantic Segmented Controls

Segmented controls embody AXIOM01 principles:

  • Semantic: Native <fieldset> and <legend> for grouping
  • Accessible: Screen readers announce option groups and selections
  • Form-Native: Works with form validation and submission
  • State-Driven: Data attributes manage selection state
  • Minimal Markup: Just buttons in a fieldset
  • Responsive: Layout adapts to screen size

Avoid custom button group implementations that lose semantic meaning. Use fieldsets.
