Page Header Component

Full-width page header section with breadcrumb navigation and page title. Establishes visual hierarchy and navigation context for inner pages.

Session 2 Backport Production Ready

Overview

The Page Header component combines a full-width background section with breadcrumb navigation and a page title. It's commonly used on inner pages (not the homepage) to establish context and allow users to navigate back to higher-level pages.

Features

  • ✅ Full-width background with centered content
  • ✅ Breadcrumb navigation with arrow separators (→)
  • ✅ Current page highlighted in breadcrumb
  • ✅ Semantic HTML structure
  • ✅ Responsive typography (scales on mobile)
  • ✅ Theme-aware colors
  • ✅ Accessible ARIA attributes
  • ✅ Dark mode compatible

Usage

Basic Structure

<section class="page-header">
  <div class="page-header-container">
    <nav class="breadcrumb" aria-label="Breadcrumb">
      <ol>
        <li><a href="/">Home</a></li>
        <li><a href="/admin">Admin</a></li>
        <li><span aria-current="page">Users</span></li>
      </ol>
    </nav>
    <h1>User Management</h1>
  </div>
</section>

Breadcrumb Navigation

The breadcrumb navigation uses semantic <nav> and <ol> elements with arrow separators between items.

Structure

  • Each breadcrumb item is a <li>
  • Links use the <a> tag
  • The current page uses <span aria-current="page">
  • Arrows (→) are added automatically via CSS

Last Item (Current Page)

The last breadcrumb item represents the current page. It should:

  • Not be a link
  • Use aria-current="page" for accessibility
  • Be highlighted (bold) to stand out
  • Not have an arrow separator

CSS Variables

The Page Header uses these CSS variables:

/* Colors */
--a-color-surface           /* Background color */
--a-color-on-surface        /* Text color */
--a-color-primary           /* Link color */
--a-color-outline           /* Border color */
--a-color-on-surface-variant /* Secondary text (arrows) */

/* Typography */
--a-font-weight-bold        /* Bold font weight */

/* Spacing */
--a-space-xl                /* 2rem - vertical padding */
--a-space-m                 /* 1rem - horizontal padding */

/* Border radius */
--a-border-radius-base      /* Border radius (if needed) */

Responsive Behavior

The component scales responsively:

  • Desktop (768px+): Full padding and large typography
  • Tablet (768px): Slightly reduced padding
  • Mobile (<768px): Compact padding, smaller font sizes, arrows scale down

Accessibility

  • Semantic <nav> with aria-label="Breadcrumb"
  • Ordered list (<ol>) indicates hierarchy
  • aria-current="page" indicates current page
  • High contrast text and links
  • Touch-friendly link targets (minimum 44x44px)
  • Keyboard navigable

Dark Mode

The component automatically adapts to dark mode using CSS variables. No additional styling required.

Common Use Cases

  • Admin dashboard pages
  • Product category pages
  • Documentation pages
  • Blog post pages
  • User profile pages
  • Search results pages

Best Practices

  • Keep breadcrumb paths short (3-5 levels)
  • Use clear, concise labels
  • Always include a link to the home page as the first item
  • Don't repeat the page title in the breadcrumb
  • Ensure all links are functional
