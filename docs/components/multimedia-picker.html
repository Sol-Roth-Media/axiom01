Multimedia Picker

Semantic multimedia picker component for browsing and selecting media files. Uses tabbed interface for organization and multi-select with selection display in a cohesive container.

Structure & Styling

  • Container - `
    ` with border, border-radius, padding (cohesive card-like component)
  • Tab Group - `[role="tablist"]` with flex layout, border, and background (grouped pill-style tabs)
  • Tab Buttons - `.button` with `data-variant` for primary/secondary states
  • Tab Panels - `[role="tabpanel"]` with semantic structure
  • Fieldsets - `
    ` and `` for form organization
  • Selection Area - Bordered section showing selected items
  • Action Buttons - Right-aligned with flex layout

Tab Group Styling

The `[role="tablist"]` container provides the grouped pill-style appearance:

[role="tablist"] {
    display: flex;
    flex-wrap: wrap;
    gap: var(--a-space-s);
    padding: var(--a-space-m);
    border: 1px solid var(--a-color-border);
    border-radius: var(--a-border-radius-base);
    background: var(--a-color-background-secondary);
    margin-bottom: var(--a-space-l);
}

Key Features

  • Cohesive Container - Component wrapped in bordered article element
  • Tab Group - Pills grouped together with border and background
  • Tab States - Active/inactive with visual distinction
  • Fieldset Organization - Proper form grouping with legend
  • Selection Display - Shows selected items in separated section
  • Action Buttons - Right-aligned Cancel/Select pair
  • Semantic HTML - All ARIA roles and attributes

Use Cases

  • Media Library - Browse and select multiple media files
  • File Manager - Organized file selection interface
  • Content Creation - Pick media for posts/articles
  • Gallery Management - Organize and select photos
  • Asset Library - Browse project media files

Accessibility

  • Use proper `role="tablist"`, `role="tab"`, `role="tabpanel"` attributes
  • Include `aria-selected` on tabs to indicate active state
  • Use `aria-controls` to link tabs to panels
  • Use `aria-labelledby` for screen reader descriptions
  • Support keyboard navigation (Tab, Arrow keys, Enter)
  • Use `
    ` and `` for form grouping
  • Link labels to inputs with `for` attributes

Best Practices

  • Wrap entire component in bordered container for cohesion
  • Group tabs together with border and background color
  • Use semantic button elements with data-variant
  • Always include legends for fieldsets
  • Right-align action buttons (Cancel, Select, Save)
  • Show selected items in separated section with visual distinction
  • Use design tokens for all spacing, colors, borders
  • Test keyboard navigation (Tab, Arrow keys)
  • Verify screen reader announcements
