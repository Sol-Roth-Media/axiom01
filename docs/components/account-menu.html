Account Menu Component

A dropdown menu component for user authentication and profile access. Displays logged-in user information with links to profile, settings, and logout. Session 2 Backport

Overview

The Account Menu component provides a consistent, accessible way to display user authentication status and provide quick access to user-related actions. It automatically shows different content for authenticated and anonymous users.

Features

  • ✅ Shows logged-in username with profile icon
  • ✅ Dropdown menu for profile, settings, logout
  • ✅ Shows "Log in" link for anonymous users
  • ✅ Fully keyboard accessible (arrow keys, Escape)
  • ✅ Click-outside to close
  • ✅ Smooth animations and hover effects
  • ✅ Mobile responsive (full-width on small screens)
  • ✅ Dark mode compatible
  • ✅ WCAG 2.1 AA compliant

Basic Usage

  • My Profile
  • Settings
  • Logout

HTML Structure

<div class="account-menu" data-component="account-menu">
  <button class="account-button" 
          aria-expanded="false" 
          aria-haspopup="menu">
    <span class="account-icon">👤</span>
    <span class="account-name">John Doe</span>
    <span class="account-toggle">⋮</span>
  </button>
  
  <ul class="account-dropdown" role="menu">
    <li><a href="/profile" role="menuitem">My Profile</a></li>
    <li><a href="/settings" role="menuitem">Settings</a></li>
    <li class="divider"></li>
    <li><a href="/logout" role="menuitem">Logout</a></li>
  </ul>
</div>

Anonymous Users

Log in 
<div class="account-menu">
  <a href="/login" class="login-link">Log in</a>
</div>

JavaScript

The component is initialized automatically when the page loads. The JavaScript is located at js/components/account-menu.js.

Keyboard Shortcuts

  • Enter or Space - Toggle dropdown
  • Arrow Up - Navigate to previous menu item
  • Arrow Down - Navigate to next menu item
  • Home - Jump to first menu item
  • End - Jump to last menu item
  • Escape - Close dropdown
  • Click Outside - Close dropdown

Required Markup Contract

  • Use data-component="account-menu" on the root container.
  • Include one .account-button toggle button and one .account-dropdown menu.
  • Menu links should use role="menuitem" for keyboard navigation.
  • The component manages aria-expanded, aria-controls, and aria-hidden automatically.

/* Spacing variables */

--a-space-s             /* 0.5rem - padding on items */
--a-space-m             /* 1rem - padding on button */

Accessibility

  • Semantic HTML with proper roles (menu, menuitem)
  • ARIA attributes: aria-expanded, aria-haspopup, aria-controls, aria-hidden, aria-label
  • Keyboard navigation support (arrows, Home/End, Escape, Enter)
  • Focus management within dropdown
  • High contrast in both light and dark modes
  • Screen reader friendly

Browser Support

  • Chrome/Edge (latest)
  • Firefox (latest)
  • Safari (latest)
  • Mobile browsers (iOS Safari, Chrome Mobile)
