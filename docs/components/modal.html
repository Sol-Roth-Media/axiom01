Modal

Modals are used to display content in a layer above the main page, often for critical information or user input.

Live Demo

Click the button below to see a live modal in action. The `data-modal-target` attribute on the button corresponds to the `id` of the modal element.

<!-- The Button -->
<button class="button primary" data-modal-target="demo-modal">Open Modal</button>

<!-- The Modal -->
<div class="modal" id="demo-modal" data-component="modal">
    <div class="modal-content">
        <header>
            <h3>Modal Title</h3>
            <button class="close" aria-label="Close modal">×</button>
        </header>
        <section>
            <p>This is the main content of the modal. You can put any HTML you want in here.</p>
        </section>
        <footer>
            <button class="button secondary close">Close</button>
            <button class="button primary">Save Changes</button>
        </footer>
    </div>
</div>

Accessibility

Axiom's modal script handles critical accessibility features for you out of the box.

Focus Trapping

When the modal is open, focus is trapped within it, so users cannot accidentally interact with the underlying page.

Escape Key

Users can close the modal at any time by pressing the Escape key, which is a standard and expected behavior.

Roles & Labels

Ensure your modal has `role="dialog"` and is properly labeled with `aria-labelledby` to provide context for screen readers.

Modal Title

This is the main content of the modal. You can put any HTML you want in here, including forms, images, or videos.

The modal script will automatically handle closing when the background, escape key, or an element with the `.close` class is clicked.
