Modal

Semantic modal dialog component for focused user interactions. Single `.modal` class with proper ARIA roles and keyboard management.

Live Example

<div class="modal" aria-modal="true" role="dialog" aria-labelledby="modal-title">
				<div>
				<header>
				<h2 id="modal-title">Modal Title</h2>
				<button aria-label="Close modal">×</button>
				</header>
				<p>Modal content here.</p>
				<footer>
				<button class="button secondary">Cancel</button>
				<button class="button primary">Confirm</button>
				</footer>
				</div>
				</div>

Markup Structure

Modal uses semantic HTML with single `.modal` class:

<div class="modal" aria-modal="true" role="dialog" aria-labelledby="modal-title">
				<!-- Centered container div -->
				<div>
				<!-- Header section -->
				<header>
				<h2 id="modal-title">Title</h2>
				<button aria-label="Close modal">×</button>
				</header>

				<!-- Main content -->
				<p>Modal body content...</p>

				<!-- Footer with actions -->
				<footer>
				<button class="button secondary">Cancel</button>
				<button class="button primary">Confirm</button>
				</footer>
				</div>
				</div>

Key Features

  • Semantic Structure - Uses `
    `, `
    ` for organization
  • Single Class - Only `.modal` class on container
  • ARIA Attributes - Full accessibility: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
  • Focus Management - Modal receives and traps focus
  • Dismissal Options - Close button, cancel button, or escape key
  • Responsive Design - Adapts to all screen sizes
  • No BEM Classes - No `.modal-header`, `.modal-body`, `.modal-footer`

JavaScript Implementation

Control modal visibility with simple JavaScript:

// Open modal
				const openModal = (modalId) => {
				const modal = document.getElementById(modalId);
				modal.style.display = 'flex';
				// Set focus to modal or first focusable element
				modal.focus();
				};

				// Close modal
				const closeModal = (modalId) => {
				document.getElementById(modalId).style.display = 'none';
				};

				// Close on escape key
				document.addEventListener('keydown', (e) => {
				if (e.key === 'Escape') {
				const modal = document.querySelector('.modal[style*="flex"]');
				if (modal) closeModal(modal.id);
				}
				});

				// Close on outside click
				const modal = document.getElementById('modal-id');
				modal.addEventListener('click', (e) => {
				if (e.target === modal) {
				closeModal('modal-id');
				}
				});

Semantic Structure Details

  • **`.modal`** - Main container with single class
  • **Centered ` `** - Child div for centering content
  • **`
    `** - Title and close button (semantic)
  • **Content** - Any semantic elements (paragraphs, forms, etc.)
  • **`
    `** - Action buttons (semantic)

ARIA Attributes

  • `role="dialog"` - Identifies as dialog/modal
  • `aria-modal="true"` - Indicates modal behavior
  • `aria-labelledby="modal-title"` - Links to modal title
  • `aria-label` on close button - Describes purpose

Accessibility

  • Use semantic `
    ` and `
    ` elements
  • Include proper ARIA roles and attributes
  • Trap focus within modal (don't allow tab outside)
  • Support escape key to close
  • Restore focus to triggering element after close
  • Use descriptive button labels
  • Ensure sufficient color contrast
  • Test with keyboard and screen readers

Responsive Behavior

Modal dialogs adapt to all screen sizes:

Mobile (< 768px)

On mobile, modals occupy most of the screen:

  • Modal width: 90vw (90% viewport width)
  • Max-height: 90vh (fits within viewport)
  • Close button clearly visible
  • Content scrolls if needed
  • Touch-friendly close targets (44px+)

Desktop (1024px+)

Desktop modals use optimal sizing:

  • Fixed width (typically 500px-800px)
  • Centered on screen
  • Semi-transparent backdrop
  • Keyboard navigation (ESC to close)

Framework Integration

Modals use semantic HTML with responsive CSS:

  • Framework handles viewport constraints
  • Backdrop layer automatic
  • Focus trap and keyboard handling built-in
  • Responsive sizing based on media queries

Best Practices

  • Use modals for critical user actions only
  • Keep modal content focused and relevant
  • Include clear action buttons (Primary + Secondary)
  • Always provide a close button
  • Use semantic HTML structure
  • Implement keyboard support (Escape, Tab trap)
  • Use single `.modal` class with semantic elements
  • Test focus management thoroughly
