Modal

Axiom01's modal component displays important content in a focused overlay. Modals are designed to draw user attention and block interaction with the rest of the page until the modal is dismissed.

HTML Structure

Modals use a semantic overlay pattern with proper accessibility attributes:

<!-- Modal Overlay -->
<div class="modal-overlay" id="modal-example" role="dialog" aria-modal="true" aria-labelledby="modal-title">
    <div class="modal-content">
        <!-- Header -->
        <div class="modal-header">
            <h2 id="modal-title">Modal Title</h2>
            <button class="modal-close" aria-label="Close modal">&times;</button>
        </div>

        <!-- Body -->
        <div class="modal-body">
            <p>Modal content goes here.</p>
        </div>

        <!-- Footer -->
        <div class="modal-footer">
            <button class="button secondary">Cancel</button>
            <button class="button primary">Confirm</button>
        </div>
    </div>
</div>

Features

  • Overlay Pattern - Darkened background prevents interaction with page content
  • Focus Management - Modal receives focus while open
  • Dismissal Options - Close button, cancel button, or click outside
  • Smooth Animation - Slide up animation on open
  • Responsive Design - Adapts to small screens
  • Multiple Variants - Basic, form, confirmation styles

JavaScript Implementation

Control modal visibility with JavaScript:

// Open modal
const openModal = (modalId) => {
    document.getElementById(modalId).classList.add('open');
};

// Close modal
const closeModal = (modalId) => {
    document.getElementById(modalId).classList.remove('open');
};

// Close on outside click
const modal = document.getElementById('modal-example');
modal.addEventListener('click', (e) => {
    if (e.target === modal) {
        closeModal('modal-example');
    }
});

Use Cases

  • Confirmations - "Are you sure?" dialogs before destructive actions
  • Forms - Create, edit, or upload forms in dedicated modal
  • Alerts - Important notifications requiring user acknowledgment
  • Information - Detailed information without leaving page
  • Authentication - Login or signup flows
  • Onboarding - Guided tours or help overlays

Accessibility

  • Semantic Role - Use role="dialog" and aria-modal="true"
  • Labeling - Include aria-labelledby pointing to title
  • Keyboard Navigation - Escape key closes modal, Tab stays within modal
  • Focus Management - Focus moves to modal when opened, returns when closed
  • Screen Reader Support - Proper heading hierarchy and button labels
  • Contrast - Overlay has sufficient contrast with background

Best Practices

  • Clear Purpose - Modal should have one clear purpose
  • Minimal Content - Keep modal focused and avoid scrolling if possible
  • Clear Actions - Primary and secondary buttons should be obvious
  • Easy Dismissal - Always provide a close button and allow outside click
  • No Nested Modals - Avoid modal within modal
  • Mobile Friendly - Ensure modal is readable on small screens
  • Prevent Action - Modal should block interaction with page content
