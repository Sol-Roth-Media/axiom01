Native <progress> Element

Built-in to all modern browsers for displaying task progress.

Semantic Meaning

Tells assistive technologies exactly what it represents.

Determinate/Indeterminate

Show specific progress or indeterminate loading state.

Screen Reader Support

HTML Structure

Basic Progress Bar

<progress value="50" max="100"></progress>

Indeterminate Progress

<!-- No value attribute = indeterminate (loading) state -->
<progress></progress>

With Label and Percentage

<label for="task-progress">
  Upload: <span id="percent">45</span>%
</label>
<progress id="task-progress" value="45" max="100"></progress>

Multiple Progress Bars

<h3>Task Progress</h3>
<p>Task 1: <progress value="90" max="100"></progress></p>
<p>Task 2: <progress value="60" max="100"></progress></p>
<p>Task 3: <progress value="30" max="100"></progress></p>

JavaScript API

Setting Progress Value

const progress = document.querySelector('progress');

// Get current progress
console.log(progress.value); // 50
console.log(progress.max);   // 100

// Set progress value
progress.value = 75;

// Calculate percentage
const percentage = (progress.value / progress.max) * 100;
console.log(percentage); // 75

Simulating Upload

const progress = document.querySelector('progress');
const percentDisplay = document.querySelector('#percent');

// Simulate upload progress
let current = 0;
const interval = setInterval(() => {
  current += Math.random() * 30;
  if (current >= 100) {
    current = 100;
    clearInterval(interval);
  }
  progress.value = current;
  percentDisplay.textContent = Math.floor(current);
}, 500);

Accessibility

Best Practices

  • Always label progress bars with <label>
  • Screen readers announce percentage automatically
  • Use determinate progress when known, indeterminate for unknown
  • Provide text percentage alongside visual bar
  • Test with screen readers (NVDA, JAWS, VoiceOver)
  • Don't rely solely on color to convey status
  • Use aria-live for dynamic updates

AXIOM01 Philosophy

Why Use Native <progress>

The native progress element embodies AXIOM01 principles:

  • Semantic: <progress> tells the browser exactly what it represents
  • Accessible: Screen readers announce progress percentage automatically
  • Minimal Code: No custom JavaScript for basic progress
  • Browser Native: Optimized by browser vendor
  • Responsive: Scales to container automatically
  • Performant: Zero layout thrashing

Custom progress bars often require complex CSS and JavaScript, breaking accessibility in the process. Use <progress> first.
