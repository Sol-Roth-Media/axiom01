Component Browser

The Component Browser is a powerful meta-component designed to showcase and interact with other Axiom01 components. It provides a dynamic interface with tabbed navigation, search, and category filtering, making it an ideal tool for exploring and demonstrating your component library. This component is particularly useful for documentation sites, design systems, or any application where you need to present a collection of UI elements in an organized and interactive manner.

Usage Guidelines

To implement the Component Browser, use the following semantic HTML structure. The data-component="component-browser" attribute on the main container is essential for Axiom01's JavaScript to initialize its functionality.

  • Wrap the entire browser in a <div class="component-browser" data-component="component-browser">.
  • The header section (<header class="component-browser-header">) typically contains component count, search input, category filter, and a "View in New Window" button.
  • The tabs are contained within <div class="tabs-container"><div class="tabs" role="tablist">. Each tab is a <button class="tab-button" role="tab">.
    • data-src: This attribute is crucial. It specifies the path to the HTML file containing the component's documentation or live example that will be loaded into the iframe.
    • data-category: This attribute is used for filtering components. Assign a category (e.g., "feedback", "forms", "layout") to each tab button.
    • aria-selected="true|false": Indicates the currently active tab. This attribute is dynamically managed by the component's JavaScript.
  • The component preview is rendered inside an <iframe> within <div class="preview-container"><div class="preview">. The iframe should have a unique id (e.g., component-preview-iframe) and a descriptive title for accessibility.

JavaScript Integration

The core functionality of the Component Browser (tab switching, iframe loading, search, and filtering) is handled automatically by Axiom01's JavaScript. The presence of the data-component="component-browser" attribute on the main container triggers the loading and initialization of js/components/component-browser.js.

Note: The live example above includes inline JavaScript for demonstration purposes within this documentation page. In a real application, the component's logic would be fully managed by js/components/component-browser.js, which is initialized by axiom.js.

Accessibility

The Component Browser is designed with accessibility in mind:

  • ARIA Roles: Uses role="tablist" for the tab container and role="tab" for individual tab buttons, providing semantic meaning for assistive technologies.
  • ARIA States: aria-selected="true|false" on tabs dynamically indicates their active state to screen readers.
  • Keyboard Navigation: Tabs are fully navigable via keyboard (Tab key to move focus, Space or Enter to activate).
  • Search Input: Includes an aria-label for screen reader clarity, describing the purpose of the input field.
  • Iframe Title: The preview iframe has a descriptive title attribute, which is essential for screen reader users to understand the content of the embedded frame.

Customization

You can customize the Component Browser's appearance by overriding the following CSS variables in your site.css or a custom theme file:

  • --a-color-surface, --a-color-surface-variant, --a-color-on-surface: Control background and text colors for various parts of the browser.
  • --a-color-outline: Defines the color for borders and separators.
  • --a-color-primary, --a-color-primary-variant: Used for active tab indicators, hover states, and primary accents.
  • --a-space-s, --a-space-m, --a-space-l, --a-space-xl: Adjust internal spacing and padding within the component.
  • --a-border-radius-medium: Controls the rounded corners of elements.
  • --a-shadow-small: Defines subtle shadow effects for depth.

Code Example

<div class="component-browser" data-component="component-browser">
    <header class="component-browser-header">
        <div class="component-browser-info">
            <h4>Component Library (<span id="component-count">...</span> Components)</h4>
            <p>Explore our collection of components.</p>
        </div>
        <div class="component-browser-controls">
            <input type="text" id="component-search-input" placeholder="Search components..." aria-label="Search components">
            <select id="component-category-filter" aria-label="Filter by category">
                <option value="all">All Categories</option>
                <option value="feedback">Feedback</option>
                <option value="forms">Forms</option>
                <option value="layout">Layout</option>
            </select>
            <button id="view-fullscreen-button" class="button secondary">
                <i class="fas fa-external-link-alt" aria-hidden="true"></i>
                <span>View in New Window</span>
            </button>
        </div>
    </header>
    <div class="tabs-container">
        <div class="tabs" role="tablist" aria-label="Component tabs">
            <button class="tab-button active" data-src="alert.html" role="tab" aria-selected="true" data-category="feedback"><span>Alert</span></button>
            <button class="tab-button" data-src="button.html" role="tab" data-category="forms"><span>Button</span></button>
            <button class="tab-button" data-src="card.html" role="tab" data-category="layout"><span>Card</span></button>
        </div>
    </div>
    <div class="preview-container">
        <div class="preview">
            <iframe id="component-preview-iframe" src="alert.html" title="Component preview"></iframe>
        </div>
    </div>
</div>
