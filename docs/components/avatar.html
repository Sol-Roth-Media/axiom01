Basic HTML

<!-- Avatar with initials -->
<div class="avatar avatar-md avatar-primary">JD</div>

<!-- Avatar with image -->
<div class="avatar avatar-md">
    <img src="user.jpg" alt="John Doe">
</div>

<!-- Avatar with icon -->
<div class="avatar avatar-md avatar-primary">
    <span class="axicon render" data-name="User"></span>
</div>

Sizes

  • avatar-sm - 32px (small)
  • avatar-md - 48px (medium, default)
  • avatar-lg - 64px (large)

Color Variants

  • avatar-primary - Primary brand color
  • avatar-success - Green (positive)
  • avatar-warning - Orange (caution)
  • avatar-error - Red (error)
  • avatar-neutral - Gray (neutral)

With Status Indicators

Add a status dot to show user availability:

<div class="avatar-with-status">
    <div class="avatar avatar-md avatar-primary">JD</div>
    <div class="avatar-status"></div> <!-- Online -->
</div>

<div class="avatar-status away"></div> <!-- Away -->
<div class="avatar-status offline"></div> <!-- Offline -->

Use Cases

  • User Profiles: Display user picture or initials in headers, cards, or lists
  • Comments & Discussions: Show who made a comment or post
  • Team Members: Display team avatars in project pages
  • Chat & Messaging: Show status and picture in conversations
  • Assigned Tasks: Show who is assigned to a task

Accessibility

  • When using images, always include an alt attribute with the user's name
  • If avatar is interactive, use a button element
  • Status indicators should be accompanied by text when possible
  • Ensure sufficient contrast between avatar and background
  • Use semantic color coding (green = online, gray = offline, orange = away)

Best Practices

  • Consistent Sizing: Use the same avatar size within a list or component
  • Initials: Use first letter of first and last name (e.g., "JD" for John Doe)
  • Status Context: Only show status indicators where relevant
  • Meaningful Colors: Use colors that mean something or group users by team
  • Fallbacks: Always have a fallback if image fails to load
