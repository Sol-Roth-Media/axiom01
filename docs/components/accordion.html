Accordion Component

The Accordion component allows you to toggle the visibility of content sections, providing a clean and organized way to display information. It's ideal for FAQs, feature lists, or any area where space is limited and content needs to be revealed on demand.

Usage

To implement an accordion, use the following semantic HTML structure. The `data-component="accordion"` attribute on the parent container enables Axiom's JavaScript to initialize the component. Each accordion item should have a header (button) and a content area.

HTML Structure

<div class="accordion-example" data-component="accordion">
    <div class="accordion-item">
        <button class="accordion-header" aria-expanded="false" aria-controls="content-1">
            Accordion Item #1
            <i class="fas fa-chevron-down"></i>
        </button>
        <div id="content-1" class="accordion-content" role="region" aria-labelledby="header-1">
            <p>This is the first item's accordion body. It is shown by default, until the collapse plugin adds the appropriate classes that we use to style each element. It's also worth noting that just about any HTML can go within the <code>.accordion-body</code>, though the transition does limit overflow.</p>
        </div>
    </div>
    <div class="accordion-item">
        <button class="accordion-header" aria-expanded="false" aria-controls="content-2">
            Accordion Item #2
            <i class="fas fa-chevron-down"></i>
        </button>
        <div id="content-2" class="accordion-content" role="region" aria-labelledby="header-2">
            <p>This is the second item's accordion body. You can include various content types here, such as text, images, or other components.</p>
            <ul>
                <li>List item one</li>
                <li>List item two</li>
            </ul>
        </div>
    </div>
    <div class="accordion-item">
        <button class="accordion-header" aria-expanded="false" aria-controls="content-3">
            Accordion Item #3
            <i class="fas fa-chevron-down"></i>
        </button>
        <div id="content-3" class="accordion-content" role="region" aria-labelledby="header-3">
            <p>And here's some amazing placeholder content. This is the last item's accordion body. Nothing more, nothing less.</p>
        </div>
    </div>
</div>

Accessibility

Axiom's Accordion component is built with accessibility in mind:

  • role="button": The header is a button, making it focusable and operable via keyboard.
  • aria-expanded: Indicates whether the accordion panel is currently expanded or collapsed. Toggled by JavaScript.
  • aria-controls: Links the header button to its controlled content panel.
  • role="region": Identifies the content panel as a perceivable section of content.
  • Keyboard Navigation:
    • Space or Enter: Activates the accordion header to toggle its content.
    • Tab: Moves focus between accordion headers and other interactive elements.

Customization

The Accordion component can be customized using Axiom's CSS variables. You can adjust colors, spacing, and border styles to match your theme.

.accordion-item {
    border-color: var(--a-color-outline);
    border-radius: var(--a-border-radius-m);
}
.accordion-header {
    background-color: var(--a-color-surface-variant);
    color: var(--a-color-on-surface-variant);
}
.accordion-header:hover {
    background-color: var(--a-color-surface-variant-hover);
}
.accordion-content {
    background-color: var(--a-color-surface);
    color: var(--a-color-on-surface);
}

JavaScript Integration

The accordion's functionality is handled automatically by Axiom's JavaScript when the data-component="accordion" attribute is present on the main container. The JavaScript toggles the aria-expanded attribute on the header and the is-open class on the content, which controls its visibility and animation.

For smooth transitions, the CSS uses `max-height` and `overflow: hidden` in conjunction with the `is-open` class. When the `is-open` class is added, the `max-height` is set to a value large enough to contain the content, allowing for a smooth reveal.
