Accordion Component

Collapsible content sections that allow users to show and hide information in a compact, organized way. Perfect for FAQs, feature lists, and content organization.

Basic Usage

<div class="accordion">
  <div class="accordion-item">
    <h3 class="accordion-header">
      <button class="accordion-button" aria-expanded="false" aria-controls="panel1">
        What is Axiom01?
      </button>
    </h3>
    <div class="accordion-panel" id="panel1">
      <div class="accordion-content">
        <p>Axiom01 is a semantic-first UI framework that prioritizes clean HTML, accessibility, and minimal class usage.</p>
      </div>
    </div>
  </div>

  <div class="accordion-item active">
    <h3 class="accordion-header">
      <button class="accordion-button" aria-expanded="true" aria-controls="panel2">
        How do I get started?
      </button>
    </h3>
    <div class="accordion-panel" id="panel2">
      <div class="accordion-content">
        <p>Simply include the Axiom01 CSS file in your HTML head and start writing semantic HTML with minimal classes.</p>
        <ul>
          <li>Add the CSS link to your HTML</li>
          <li>Write clean, semantic HTML</li>
          <li>Apply single component classes</li>
        </ul>
      </div>
    </div>
  </div>
</div>

Always Open Variant

For accordions that should stay expanded and not collapse:

<div class="accordion always-open">
  <div class="accordion-item active">
    <h3 class="accordion-header">
      <button class="accordion-button" aria-expanded="true" aria-controls="info-panel" disabled>
        Important Information
      </button>
    </h3>
    <div class="accordion-panel" id="info-panel">
      <div class="accordion-content">
        <p>This content is always visible and cannot be collapsed.</p>
      </div>
    </div>
  </div>
</div>

CSS Variables

.accordion {
  --accordion-border-color: var(--a-color-outline);
  --accordion-header-bg: var(--a-color-surface);
  --accordion-content-bg: var(--a-color-surface-variant);
  --accordion-border-radius: var(--a-border-radius-medium);
  --accordion-padding: var(--a-space-m);
}

Accessibility Features

  • Proper heading hierarchy with semantic HTML
  • ARIA attributes for screen readers (aria-expanded, aria-controls)
  • Keyboard navigation support (Enter, Space, Arrow keys)
  • Focus management and indicators
  • Screen reader announcements for state changes

JavaScript Integration

// Basic accordion functionality
document.querySelectorAll('.accordion-button').forEach(button => {
  button.addEventListener('click', function() {
    const item = this.closest('.accordion-item');
    const panel = item.querySelector('.accordion-panel');
    const isExpanded = this.getAttribute('aria-expanded') === 'true';
    // Toggle current item
    this.setAttribute('aria-expanded', !isExpanded);

Best Practices

  • Use semantic HTML for structure
  • Minimize class usage
  • Test accessibility with keyboard and screen readers
  • Customize with CSS variables for your theme
