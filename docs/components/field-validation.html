Field Validation Component

Form field wrapper with validation states. Provides semantic markup and data attributes for displaying error messages, focus states, and validation feedback using AXIOM01 design tokens.

HTML Structure

Use the `.field` wrapper with data attributes for state management:

<div class="field">
  <label for="input-id"><strong>Field Label</strong></label>
  <input type="text" id="input-id" placeholder="Placeholder text">
</div>

<!-- With error state -->
<div class="field" data-invalid="true">
  <label for="error-input">Invalid Field</label>
  <input type="text" id="error-input">
  <span class="error-message">Error message here</span>
</div>

Data Attributes

  • data-invalid="true" - Marks field as invalid, applies error styling
  • data-focused="true" - Applied automatically on focus, applies focus styling

JavaScript Behavior

AXIOM01 core automatically manages focus states:

// Focus management is automatic via event delegation
document.addEventListener('focusin', function (e) {
  const fieldWrapper = e.target.closest('.field');
  if (fieldWrapper) {
    fieldWrapper.setAttribute('data-focused', 'true');
  }
});

document.addEventListener('focusout', function (e) {
  const fieldWrapper = e.target.closest('.field');
  if (fieldWrapper) {
    fieldWrapper.removeAttribute('data-focused');
  }
});

Validation Example (JavaScript)

const emailInput = document.querySelector('#demo-email');
const field = emailInput.closest('.field');

emailInput.addEventListener('blur', function () {
  const isValid = /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(this.value);
  
  if (!isValid && this.value) {
    field.setAttribute('data-invalid', 'true');
    const errorMsg = field.querySelector('.error-message');
    if (!errorMsg) {
      const span = document.createElement('span');
      span.className = 'error-message';
      span.textContent = 'Please enter a valid email address';
      field.appendChild(span);
    }
  } else {
    field.removeAttribute('data-invalid');
    const errorMsg = field.querySelector('.error-message');
    if (errorMsg) errorMsg.remove();
  }
});

Design Tokens Used

  • --a-space-m - Field margin-bottom
  • --a-space-s - Label margin-bottom
  • --a-font-weight-medium - Label weight
  • --a-color-error - Invalid field color
  • --a-color-error-light - Invalid field background
  • --a-color-primary - Focus state color
  • --a-color-outline - Border color
  • --a-border-radius-base - Input border radius
  • --a-transition-base - State transitions
  • --focus-outline-width - Focus outline width
  • --focus-outline-offset - Focus outline offset

CSS Classes

  • .field - Main field wrapper container
  • .error-message - Error message text styling
  • Input elements automatically inherit field styling

Best Practices

  • Always label inputs: Use `<label for="id">` paired with input id
  • Show errors clearly: Use error-message span with data-invalid attribute
  • Support keyboard: Tab navigation and enter submission work automatically
  • Accessibility: Use `<strong>` for required field indicators
  • Real-time validation: Validate on blur, not on every keystroke
  • Helper text: Use small text below label for additional guidance
