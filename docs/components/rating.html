Rating Component

An accessible, interactive star rating component for collecting user feedback and displaying ratings.

Overview

The Rating component allows users to select or view star-based ratings. It's fully keyboard accessible, supports hover previews, and can be used in read-only mode for displaying ratings.

Basic Usage

Interactive Rating

<div class="rating" data-component="rating" data-count="5"></div>

Read-Only Rating Display

<div class="rating" data-component="rating" data-count="5" data-readonly></div>

Features

  • Interactive Stars: Click to select a rating, hover to preview
  • Keyboard Navigation: Use arrow keys to navigate between stars
  • Read-Only Mode: Display ratings without interaction
  • Accessibility: Full ARIA support and semantic HTML
  • Customizable: Set any number of stars (1-10)
  • Events: Dispatches custom `rating-changed` event

Attributes

Attribute Description Default
data-component="rating" Required. Enables the rating component N/A
data-count Number of stars to display 5
data-readonly Makes the rating display-only (no interaction) N/A

Examples

3-Star Rating

<div class="rating" data-component="rating" data-count="3"></div>

10-Star Rating

<div class="rating" data-component="rating" data-count="10"></div>

Use Cases

  • Product Reviews: Let customers rate products
  • Feedback Forms: Rate service quality, satisfaction
  • Content Pages: Display average ratings alongside reviews
  • Ratings Display: Show aggregated ratings with read-only mode
  • Quality Metrics: Rate quality, difficulty, or other metrics

JavaScript API

After the component is initialized, you can interact with it programmatically:

// Get the component element
const ratingEl = document.querySelector('.rating');

// Listen for rating changes
ratingEl.addEventListener('rating-changed', (e) => {
    console.log('Rating set to:', e.detail.rating);
});

// Set rating programmatically (if not readonly)
// Note: Use the component's internal methods if exposed

Accessibility

  • ARIA Roles: Uses `role="radiogroup"` for semantic meaning
  • Keyboard Support: Arrow keys to navigate, Enter/Space to select
  • Labels: Each star has descriptive `aria-label`
  • Focus Visible: Clear focus indicators for keyboard users
  • Screen Readers: Proper announcements with ARIA attributes

CSS Customization

You can customize the appearance using CSS variables:

.rating {
    --star-color: var(--a-color-outline);
    --star-hover-color: var(--a-color-warning);
    --star-active-color: var(--a-color-primary);
    --star-size: 2rem;
}

Or override the CSS classes directly:

.rating .star {
    font-size: 2.5rem; /* Larger stars */
    color: var(--a-color-secondary); /* Different color */
}

Integration with Forms

<form>
    <label for="product-rating">How would you rate this product?</label>
    <div id="product-rating" class="rating" data-component="rating" data-count="5"></div>

    <label for="review-text">Tell us more:</label>
    <textarea id="review-text" name="review"></textarea>

    <button type="submit">Submit Review</button>
</form>
