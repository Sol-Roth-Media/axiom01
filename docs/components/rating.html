Browser APIs

Native <meter> Element

Represents scalar measurements like ratings, disk usage, temperatures.

Semantic Measurement

Tells assistive tech this is a measurement, not progress.

Color-Coded Ranges

Low, medium, and high ranges automatically color-coded.

Value Constraints

Min/max define the range, value is current measurement.

Screen Reader Support

Announces value and measurement type automatically.

Easy JavaScript Control

Simple value property for dynamic updates.

HTML Structure

Basic Meter (5-Star Rating)

<meter min="0" max="5" value="4" 
       low="2" high="4" optimum="5"></meter>

Disk Usage Meter

<label for="disk">Storage</label>
<meter id="disk" min="0" max="100" value="70"
       low="50" high="80" optimum="20"></meter>

Temperature Meter

<label for="temp">Room Temperature</label>
<meter id="temp" min="50" max="95" value="72"
       low="60" high="85" optimum="72"></meter>

Attributes Explained

<meter
  value="4"      <!-- Current measurement -->
  min="0"        <!-- Minimum value -->
  max="5"        <!-- Maximum value -->
  low="2"        <!-- Below this = low range -->
  high="4"       <!-- Above this = high range -->
  optimum="5"    <!-- Ideal value -->
></meter>

JavaScript API

Getting and Setting Values

const meter = document.querySelector('meter');

// Get current value
console.log(meter.value); // 4

// Set new value
meter.value = 3;

// Access range properties
console.log(meter.min);     // 0
console.log(meter.max);     // 5
console.log(meter.low);     // 2
console.log(meter.high);    // 4
console.log(meter.optimum); // 5

Dynamic Rating Update

const ratingMeter = document.querySelector('meter');
const ratingButtons = document.querySelectorAll('[data-rating]');

ratingButtons.forEach(btn => {
  btn.addEventListener('click', (e) => {
    const newRating = e.target.dataset.rating;
    ratingMeter.value = newRating;
    console.log('Rating set to:', newRating);
  });
});

Color Ranges

How Meter Determines Color

The browser automatically colors the meter based on where the value falls:

  • Red (Low): value < low, or value > high and optimum < low
  • Yellow (Medium): Between low and high
  • Green (Optimal): Close to optimum value

The exact color-coding varies by browser, but the semantic meaning is consistent.

Meter vs Progress

When to Use Each

Element Purpose Example
<progress> Task completion % File upload, download
<meter> Scalar measurement Rating, disk usage, temperature

Accessibility

Best Practices

  • Always pair with <label> for context
  • Screen readers announce value and range
  • Use semantic attributes (low, high, optimum)
  • Provide text alternative value nearby
  • Choose min/max/low/high logically
  • Test with screen readers
  • Don't use for progress tracking (use <progress>)

AXIOM01 Philosophy

Why Use Native <meter>

The native meter element embodies AXIOM01 principles:

  • Semantic: <meter> clearly indicates scalar measurement (not progress)
  • Accessible: Screen readers understand measurement context
  • Minimal Code: No custom JavaScript for basic meters
  • Browser Native: Uses OS-level rendering
  • Automatic Coloring: Browser intelligently colors ranges
  • Easy to Use: Simple attributes for complex semantics

Custom rating components often misuse progress bars or build from scratch. Use <meter> for its intended purpose—scalar measurements.
