Forms

Axiom01 provides semantic, accessible form elements with minimal styling for maximum flexibility. All form controls are fully keyboard accessible and support standard HTML validation.

HTML Structure

<form>
    <div class="form-group">
        <label for="email">Email Address</label>
        <input type="email" id="email" name="email" required>
        <p class="form-help">Help text here</p>
    </div>
    
    <div class="form-group">
        <label for="message">Message</label>
        <textarea id="message" name="message"></textarea>
    </div>
    
    <div class="form-actions">
        <button type="submit" class="button primary">Submit</button>
        <button type="reset" class="button secondary">Clear</button>
    </div>
</form>

Form Control Types

  • Text Input - <input type="text"> for general text
  • Email Input - <input type="email"> with built-in validation
  • Password Input - <input type="password"> for secure entry
  • Select/Dropdown - <select> for choosing from options
  • Textarea - <textarea> for multi-line text
  • Checkbox - <input type="checkbox"> for multiple selection
  • Radio Button - <input type="radio"> for single selection

Form Structure Best Practices

  • Form Groups - Wrap each control with its label in .form-group
  • Labels - Always use <label> with for attribute linked to input id
  • Help Text - Use .form-help class for additional guidance
  • Validation - Use HTML5 attributes: required, type="email", etc.
  • Actions - Group submit/reset buttons in .form-actions
  • Spacing - Form groups use var(--a-space-l) gap for vertical rhythm

Use Cases

  • Contact Forms - Simple contact information collection
  • Sign Up - User registration and account creation
  • Login - Authentication and access control
  • Settings - User preferences and configuration
  • Surveys - Feedback and data collection
  • Search - Query input and filtering

Accessibility

  • Always associate labels with inputs using for and id attributes
  • Use semantic form elements (<form>, <fieldset>, <legend>)
  • Provide clear help text and error messages
  • Use HTML5 validation attributes for client-side feedback
  • Ensure focus states are visible
  • Support keyboard navigation (Tab, Space, Enter)

Best Practices

  • Logical Order - Arrange fields in a logical, intuitive order
  • Clear Labels - Use concise, descriptive labels
  • Required Fields - Mark required fields clearly (use required attribute)
  • Error Prevention - Validate early and provide helpful error messages
  • Visual Grouping - Use form groups and spacing to organize related fields
  • Mobile First - Ensure forms work well on mobile (large touch targets)
