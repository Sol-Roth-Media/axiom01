Tooltip Component

Accessible tooltips that provide additional context on hover and focus with proper keyboard navigation support.

Basic Usage

Tooltips provide contextual information when users hover or focus on elements.

<button class="tooltip">
    Hover me
    <span role="tooltip" aria-hidden="true">This is a helpful tooltip</span>
</button>

Positioning

Tooltips support four positioning options using single modifier classes.

<button class="tooltip top">
    Top Tooltip
    <span role="tooltip" aria-hidden="true">Appears above</span>
</button>

<button class="tooltip right">
    Right Tooltip
    <span role="tooltip" aria-hidden="true">Appears to the right</span>
</button>

Rich Content Tooltips

Tooltips can contain rich content including icons and formatted text.

<button class="tooltip rich">
    <i class="fas fa-info-circle" aria-hidden="true"></i>
    Rich Tooltip
    <div role="tooltip" aria-hidden="true">
        <strong>Pro Tip:</strong><br>
        This tooltip can contain multiple lines<br>
        and even <em>formatted text</em>!
    </div>
</button>

Interactive Elements

Tooltips work with various interactive elements like links, buttons, and form controls.

Link with tooltip This link leads somewhere important 
<a href="#" class="tooltip">
    Link with tooltip
    <span role="tooltip" aria-hidden="true">This link leads somewhere important</span>
</a>

Best Practices

✅ Do

  • Use role="tooltip" for proper ARIA semantics
  • Keep tooltip content concise and helpful
  • Include aria-hidden="true" on decorative tooltips
  • Test with keyboard navigation (Tab to focus)
  • Position tooltips to avoid viewport edges

❌ Don't

  • Put essential information only in tooltips
  • Make tooltips too large or overwhelming
  • Use tooltips for interactive content
  • Create tooltips that cover important UI elements
  • Rely solely on hover for mobile users

Accessibility

Tooltip components are designed with accessibility in mind:

  • ARIA Support: Uses role="tooltip" for proper semantics
  • Keyboard Navigation: Tooltips appear on focus for keyboard users
  • Screen Reader Support: Hidden from screen readers when decorative
  • Focus Management: Tooltips don't interfere with tab order
