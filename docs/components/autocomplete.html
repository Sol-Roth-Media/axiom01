Autocomplete

Axiom01's autocomplete component provides intelligent text input suggestions as users type. It filters options dynamically and supports keyboard navigation for accessible, efficient data entry.

HTML Structure

<div class="autocomplete-wrapper">
    <label class="autocomplete-label" for="search">Search</label>
    <input 
        type="text" 
        id="search" 
        class="autocomplete-input"
        placeholder="Start typing..."
        aria-autocomplete="list"
        aria-controls="dropdown"
    >
    <ul class="autocomplete-dropdown" id="dropdown" role="listbox">
        <li class="autocomplete-option" role="option">Option 1</li>
        <li class="autocomplete-option" role="option">Option 2</li>
        <li class="autocomplete-option" role="option">Option 3</li>
    </ul>
</div>

JavaScript Implementation

const input = document.getElementById('search');
const dropdown = document.getElementById('dropdown');
const options = dropdown.querySelectorAll('.autocomplete-option');
let activeIndex = -1;

input.addEventListener('input', (e) => {
    const query = e.target.value.toLowerCase();
    const matches = Array.from(options).filter(opt => 
        opt.textContent.toLowerCase().includes(query)
    );
    
    // Hide all options first
    options.forEach(opt => opt.style.display = 'none');
    
    // Show matching options
    if (query && matches.length) {
        matches.forEach(opt => opt.style.display = 'block');
        dropdown.classList.add('open');
    } else {
        dropdown.classList.remove('open');
    }
    
    activeIndex = -1;
});

// Keyboard navigation
input.addEventListener('keydown', (e) => {
    const visibleOptions = Array.from(options).filter(
        opt => opt.style.display !== 'none'
    );
    
    if (e.key === 'ArrowDown') {
        e.preventDefault();
        activeIndex = Math.min(activeIndex + 1, visibleOptions.length - 1);
        updateActive(visibleOptions);
    } else if (e.key === 'ArrowUp') {
        e.preventDefault();
        activeIndex = Math.max(activeIndex - 1, -1);
        updateActive(visibleOptions);
    } else if (e.key === 'Enter') {
        if (activeIndex >= 0) {
            e.preventDefault();
            input.value = visibleOptions[activeIndex].textContent;
            dropdown.classList.remove('open');
        }
    } else if (e.key === 'Escape') {
        dropdown.classList.remove('open');
    }
});

function updateActive(visibleOptions) {
    visibleOptions.forEach((opt, i) => {
        opt.classList.toggle('active', i === activeIndex);
    });
}

Features

  • Real-time Filtering - Options filter as user types
  • Keyboard Navigation - Arrow keys, Enter, Escape support
  • Accessible - ARIA roles, focus management, semantic HTML
  • No Results Feedback - Shows message when no matches found
  • Click Support - Click options to select them
  • Flexible Styling - Works with any data source

Use Cases

  • Country/Region Selection - Geographic filtering
  • User Search - Finding team members or contacts
  • Product Search - E-commerce product discovery
  • Technology Stack - Selecting technologies or skills
  • Tag Input - Adding tags or categories
  • Address Autocomplete - Street and location completion

Accessibility

  • Use aria-autocomplete="list" on the input
  • Use role="listbox" on the dropdown container
  • Use role="option" on each suggestion item
  • Include aria-controls linking input to dropdown
  • Support full keyboard navigation (arrows, Enter, Escape)
  • Announce results count to screen readers

Best Practices

  • Debounce - For large datasets, debounce the filtering
  • Case Insensitive - Match regardless of case
  • Partial Match - Match anywhere in the string, not just start
  • Min Characters - Wait for 2-3 characters before showing results
  • Clear Feedback - Show "no results" when nothing matches
  • Max Results - Limit displayed options (e.g., 10) for performance
