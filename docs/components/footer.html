HTML Structure

<footer class="footer-example">
    <div class="footer-content">
        <div class="footer-brand">
            <a href="#" class="footer-brand-logo">[Ax<span class="primary">01</span>]</a>
            <p class="footer-tagline">Tagline here</p>
        </div>
        
        <div class="footer-column">
            <h4>Column Title</h4>
            <ul>
                <li><a href="#">Link 1</a></li>
                <li><a href="#">Link 2</a></li>
            </ul>
        </div>
    </div>
    
    <div class="footer-divider"></div>
    
    <div class="footer-bottom">
        <p class="footer-copyright">&copy; 2024 Company. All Rights Reserved.</p>
        <div class="footer-social">
            <a href="#" aria-label="Social Media">
                <span class="axicon render" data-name="GitHub"></span>
            </a>
        </div>
    </div>
</footer>

Footer Sections

  • Brand Section - Logo and company tagline
  • Link Columns - Organized navigation (Product, Legal, Community, etc.)
  • Divider - Visual separator between main content and bottom section
  • Bottom Section - Copyright notice and social media links
  • Social Icons - Links to social media platforms

Use Cases

  • Web Applications - Site navigation and brand presence
  • E-commerce - Product links, customer service, policies
  • Blog/Content - Archive links, categories, social sharing
  • SaaS Products - Documentation, legal, support links
  • Corporate Sites - Company info, contact, social media
  • Portfolio - About, projects, contact information

Accessibility

  • Use semantic <footer> element
  • Provide descriptive link text (not just "Click Here")
  • Include aria-label for social media icons
  • Ensure sufficient color contrast for all text
  • Make footer keyboard navigable
  • Use proper heading hierarchy (h4 for footer column titles)

Best Practices

  • Consistent Branding - Match site header style
  • Clear Information Hierarchy - Group related links logically
  • Responsive Design - Stack columns on mobile
  • Social Links - Include icons with proper ARIA labels
  • Copyright Date - Use dynamic year if possible
  • Links to Key Pages - Privacy, Terms, Contact, etc.
