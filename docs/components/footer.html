Anatomy

Brand Section

Left-aligned branding area with logo, tagline, and optional social links. Fixed width on desktop, full width on mobile.

Navigation Columns

Flexible footer navigation columns. Right-aligned on desktop, stacked on mobile. Each column contains a heading and link list.

Bottom Section

Full-width copyright and attribution area. Separated visually and appears below main content at all resolutions.

Responsive Layout

Desktop: Brand left, columns right. Mobile: Full width stacking. Uses semantic HTML and CSS flexbox.

Variants & States

Minimal Footer

Simplest form with just brand and copyright.

<footer class="ax-footer">
    <div class="ax-footer-content">
        <section class="ax-footer-brand">
            <a href="/" class="ax-footer-logo">Logo</a>
        </section>
    </div>
    <div class="ax-footer-bottom">
        <p class="ax-footer-copyright">© 2024 Company</p>
    </div>
</footer>

Full-Featured Footer

Complete footer with brand, multiple columns, social links, and copyright.

<footer class="ax-footer">
    <div class="ax-footer-content">
        <section class="ax-footer-brand">
            <a href="/" class="ax-footer-logo">[Logo]</a>
            <p class="ax-footer-tagline">Company Tagline</p>
            <div class="ax-footer-social">
                <a href="#"><i class="fab fa-github"></i></a>
            </div>
        </section>
        <nav class="ax-footer-links">
            <section class="ax-footer-column">
                <h3>Product</h3>
                <ul>
                    <li><a href="#">Features</a></li>
                </ul>
            </section>
        </nav>
    </div>
    <div class="ax-footer-bottom">
        <p class="ax-footer-copyright">© 2024 Company</p>
    </div>
</footer>

Design Principles

  • Semantic HTML: Uses proper semantic elements (footer, nav, section) for accessibility and SEO.
  • Fully Responsive: Built on CSS flexbox with mobile-first approach. Works perfectly at all resolutions.
  • CSS Variables: All colors and spacing use Axiom design tokens for consistency.
  • Accessibility: Proper ARIA labels, semantic landmarks, and keyboard navigation support.
  • No ID Dependencies: Uses class-based styling with the `ax-footer` prefix to avoid conflicts.
  • Modular: Can be easily customized by adding or removing sections.
