Component Contract

Category: display · Maturity: Experimental

  • Semantic structure: one root class and element-first descendants
  • Keyboard behavior documented and required for all interactive controls
  • ARIA mapping documented before release
  • Reduced-motion behavior uses motion-safe transitions
  • Theme tokens only: no hardcoded spacing/colors
  • Runtime lifecycle parity: init() and destroy()

Accessibility Checklist

  1. Focus order is predictable and fully keyboard operable.
  2. Interactive controls expose role/state text to assistive technology.
  3. Live updates use polite announcements only when needed.
  4. Chart-like visuals include semantic table/text fallback content.
