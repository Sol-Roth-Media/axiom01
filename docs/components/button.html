Button Component

Axiom01's button component provides clear, semantic, and accessible calls to action. Buttons are designed to be intuitive and highly customizable through CSS variables, ensuring they fit seamlessly into any theme.

Default Buttons

<button class="button">Default Button</button>
	<button class="button primary">Primary Button</button>
	<button class="button secondary">Secondary Button</button>

Button Sizes

<button class="button primary" data-size="small">Small Button</button>
	<button class="button primary">Base Button</button>
	<button class="button primary" data-size="large">Large Button</button>

Buttons with Icons

<button class="button primary"><i class="fas fa-plus"></i> Add Item</button>
	<button class="button secondary"><i class="fas fa-save"></i> Save</button>
	<button class="button"><i class="fas fa-trash"></i> Delete</button>

Disabled Buttons

<button class="button primary" disabled>Disabled Primary</button>
	<button class="button secondary" disabled>Disabled Secondary</button>
	<button class="button" disabled>Disabled Default</button>

Link Buttons

Use <a> tags with .button class for navigation.

Learn More View Details 
<a href="#" class="button primary">Learn More</a>
	<a href="#" class="button secondary">View Details</a>

Anatomy

Container

The button element that wraps all child elements. Provides padding, border, and background styling.

Icon (Optional)

An optional icon or symbol displayed at the start of the button. Use Axicons for consistency.

Label Text

The button's text content. Should be clear, concise, and action-oriented (e.g., "Save", "Submit", "Cancel").

States

Buttons have multiple states: default, hover, active, disabled, and loading. CSS handles visual transitions.

When to Use This Component

Do Use

  • For user actions like "Submit", "Save", "Send"
  • To guide users toward primary actions
  • When you need visual hierarchy in forms
  • For consistent interaction patterns
  • To indicate enabled/disabled states clearly

Don't Use

  • For text links or navigation (use <a> tags)
  • As page titles or headers
  • For decorative elements without function
  • In place of semantic HTML elements
  • With unclear or ambiguous labels

Consider

  • Using size variants for visual weight
  • Icon + text for better comprehension
  • Loading states for long operations
  • Multiple button grouping for related actions
  • Color semantic for destructive actions
