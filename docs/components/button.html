• Use <button> for actions
  • Use <a> for navigation
  • Single modifier classes: .primary, .large
  • Include aria-hidden="true" on decorative icons
  • Provide clear, descriptive button text
  • Test keyboard navigation

❌ Don't

  • Use BEM classes like .btn--primary
  • Stack utility classes: .btn .btn-lg .btn-primary
  • Use <div> or <span> as buttons
  • Rely only on color to convey meaning
  • Use vague text like "Click here"
  • Make buttons too small for touch targets
Axiom - Button Component

Button Component

Semantic buttons following Axiom01's minimal class philosophy for actions in forms, dialogs, and interactive elements.

Philosophy: Semantic Actions

Axiom01 buttons use proper semantic HTML with single modifier classes. Use <button> for actions and <a> for navigation, styled with minimal classes like .primary or .large.

<button>Default Button</button>
<button class="primary">Primary Action</button>
<button class="secondary">Secondary</button>
<button class="success">Success Action</button>
<button class="warning">Warning</button>
<button class="error">Destructive</button>

Button Sizes

Use single modifier classes for different button sizes following Axiom's minimal class approach.

<button class="small">Small Button</button>
<button>Default Size</button>
<button class="large">Large Button</button>

Button States

Buttons automatically handle focus, hover, active, and disabled states with proper accessibility support.

<button class="primary">Normal State</button>
<button class="primary" disabled>Disabled State</button>
<button class="secondary" aria-pressed="true">Pressed State</button>

Buttons with Icons

Combine buttons with icons using semantic structure, not utility classes.

<button class="primary">
    <i class="fas fa-download" aria-hidden="true"></i>
    Download
</button>

Button vs Link

Use the correct semantic element: buttons for actions, links for navigation.

Navigate to Section 
<!-- ✅ Button for actions -->
<button class="primary">Submit Form</button>

<!-- ✅ Link styled as button for navigation -->
<a href="/page" class="button secondary">Go to Page</a>

CSS Styling Approach

Axiom01 buttons follow the minimal class philosophy with semantic variants:

/* ✅ Axiom01 Way - Single Classes */
button, .button {
    padding: var(--a-space-m) var(--a-space-l);
    border-radius: var(--a-border-radius-medium);
    border: none;
    cursor: pointer;
}

button.primary {
    background: var(--a-color-primary);
    color: var(--a-color-on-primary);
}

button.large {
    padding: var(--a-space-l) var(--a-space-xl);
    font-size: var(--a-font-size-lg);
}

Accessibility Features

Built-in Accessibility

  • Semantic Elements: Uses proper <button> and <a> elements
  • Keyboard Support: Enter and Space key activation
  • Focus Management: Visible focus indicators for keyboard navigation
  • Screen Reader Support: Proper button roles and states
  • ARIA States: Supports aria-pressed, aria-expanded, aria-disabled
  • Color Independence: Meaning not conveyed by color alone
