Button

Semantic button component for user actions. Single `.button` class with `data-variant` for styling. Supports sizes, icons, and disabled states.

Default Buttons

<button class="button">Default Button</button>
<button class="button" data-variant="primary">Primary Button</button>
<button class="button" data-variant="secondary">Secondary Button</button>

Button Sizes

<button class="button" data-variant="primary" data-size="small">Small Button</button>
<button class="button" data-variant="primary">Base Button</button>
<button class="button" data-variant="primary" data-size="large">Large Button</button>

Buttons with Icons

<button class="button" data-variant="primary">
    <span class="axicon render" data-name="Plus"></span> Add Item
</button>
<button class="button" data-variant="secondary">
    <span class="axicon render" data-name="Save"></span> Save
</button>
<button class="button">
    <span class="axicon render" data-name="Trash"></span> Delete
</button>

Disabled Buttons

<button class="button" data-variant="primary" disabled>Disabled Primary</button>
<button class="button" data-variant="secondary" disabled>Disabled Secondary</button>
<button class="button" disabled>Disabled Default</button>

Link Buttons

Use semantic `` tags with `.button` class for navigation:

Learn More View Details 
<a href="#" class="button" data-variant="primary">Learn More</a>
<a href="#" class="button" data-variant="secondary">View Details</a>

Markup

Button component uses semantic `<button>` or `<a>` elements with single `.button` class:

<!-- Button element -->
<button class="button" data-variant="primary">Click Me</button>

<!-- Link element -->
<a href="#" class="button" data-variant="primary">Link Button</a>

<!-- With icon -->
<button class="button" data-variant="primary">
    <span class="axicon render" data-name="IconName"></span>
    Button Text
</button>

Variants

Use `data-variant` attribute for different button styles:

  • `data-variant="primary"` - Primary action button
  • `data-variant="secondary"` - Secondary action button
  • No variant - Default button style

Sizes

Use `data-size` attribute for different button sizes:

  • `data-size="small"` - Smaller button
  • No size attribute - Base (default) size
  • `data-size="large"` - Larger button

States

  • `disabled` - Disabled button (native HTML attribute)
  • Hover - Visual feedback on mouse over
  • Focus - Keyboard focus indicator (`:focus-visible`)
  • Active - Visual feedback on click

Accessibility

  • Use semantic `<button>` or `<a>` elements
  • Include `aria-label` for icon-only buttons
  • Use `disabled` attribute for disabled state
  • Proper focus management with `:focus-visible`
  • Sufficient color contrast (WCAG AA compliant)
  • Icons paired with text for clarity

Best Practices

  • Use semantic `<button>` for actions
  • Use `<a>` with `.button` class for navigation
  • Always include descriptive button text
  • Use icons to reinforce meaning
  • Keep button labels short and action-oriented
  • Use variants to show importance hierarchy
  • Test keyboard navigation and screen readers
