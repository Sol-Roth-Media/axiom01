# Axiom01 Component Implementation Checklist

Use this checklist for each new or improved component.

- [ ] Semantic structure audited (single root class, no BEM).
- [ ] Keyboard behavior fully defined and tested.
- [ ] ARIA role/state/property mapping documented.
- [ ] Reduced-motion behavior verified.
- [ ] Theme token usage verified (no hardcoded spacing/colors).
- [ ] Docs and demo stay in parity.
- [ ] JS lifecycle supports init/destroy parity.
- [ ] Component indexed in overview/category/browser/search.
- [ ] QA pass complete (npm test + npm run build).
