What is Axiom?

Axiom is a CSS framework designed with a semantic-first approach. Instead of relying heavily on utility classes for layout, Axiom leverages the inherent structure of your HTML5 content elements (like <main>, <section>, <article>, <aside>) combined with modern CSS (like Grid and Flexbox) to create intelligent, responsive layouts automatically. The goal is to keep your HTML clean and focused on content, letting CSS handle the presentation.

The Axiom Philosophy: Semantic First

Axiom encourages you to write meaningful HTML. By using standard HTML5 tags correctly, you not only improve accessibility and SEO but also enable Axiom to automatically apply sensible layout structures.

  • Cleaner HTML: Reduce the clutter of presentation-focused classes. Your markup stays readable and maintainable.
  • Intuitive Layouts: Common layouts emerge naturally from your content structure.
  • Maintainability: Layout adjustments are primarily handled in CSS, often without touching the HTML.
  • Flexibility: While prioritizing semantics, Axiom provides optional utility classes (like a 12-column grid) for complex scenarios or when you need precise, fine-grained control that semantic tags alone might not offer. Use them sparingly!

The ideal Axiom project uses semantic tags for the vast majority of its layout needs, resorting to utility classes only for specific overrides or complex components.

Key Features

  • Semantic Layout System: Automatic layouts based on the hierarchy of <main>, <section>, <article>, and <aside>.
  • Optional Utility Grid: A familiar 12-column grid system (.row, .col-*, .offset-*) is available for overrides and complex cases.
  • Responsive Design: Adapts gracefully to different screen sizes out-of-the-box.
  • Component Library: Includes common UI elements like buttons, forms, cards, modals, alerts, tabs, accordions, etc.
  • Theming with CSS Variables: Easily customize colors, fonts, and spacing. Includes built-in light/dark modes and multiple color schemes.
  • Minimalist & Lightweight: Focuses on providing a solid foundation without unnecessary bloat.

Getting Started with Axiom

Choose the method that best suits your project needs:

Method 1: Simple Usage (CDN / Download)

Best for quick starts, static sites, or projects without a complex build setup.

Option 1a: Using a CDN

Include the compiled Axiom CSS and JS files directly from a CDN.

<!DOCTYPE html>
<html lang="en" data-theme="light">
<head>
    <!-- ... meta tags ... -->
    <title>Axiom CDN Example</title>
    <!-- Axiom CSS (Replace with actual CDN URL) -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/axiom-css@0.1.0/dist/axiom.min.css"> <!-- Placeholder URL -->
    <!-- Your Custom CSS (Optional) -->
    <link rel="stylesheet" href="css/site.css">
</head>
<body>
    <!-- Your HTML content -->
    <!-- Axiom JS (Replace with actual CDN URL) -->
    <script src="https://cdn.jsdelivr.net/npm/axiom-css@0.1.0/dist/scripts.min.js"></script> <!-- Placeholder URL -->
</body>
</html>

Note: Replace placeholder URLs. Use specific versions.

Option 1b: Local Download

Download the compiled `axiom.css` and `scripts.js` files from the GitHub Releases page (update link).

  1. Download the files (e.g., `axiom-v0.1.zip`).
  2. Place `axiom.css` in your project's CSS directory (e.g., `css/`).
  3. Place `scripts.js` in your project's JavaScript directory (e.g., `js/`).
  4. Link them in your HTML (see Minimal Starter Template).

Method 2: Using a Build Process (NPM / Yarn)

Recommended for projects using modern front-end workflows, build tools, or requiring deeper customization.

Step 1: Install via Package Manager

# Using npm (when available)
npm install axiom-css --save-dev

# Using yarn (when available)
yarn add axiom-css --dev

(Note: Replace `axiom-css` with the actual package name.)

Step 2: Import Axiom CSS

/* In your main style.css or style.scss */
/* Import Axiom variables first (optional, for overrides) */
/* @import 'node_modules/axiom-css/src/colors_vars.css'; */
/* @import 'node_modules/axiom-css/src/axiom_vars.css'; */

/* Your custom variable overrides here */
:root { --a-color-primary-hue: 180; }

/* Import the main Axiom styles */
@import 'node_modules/axiom-css/dist/axiom.css'; /* Or from /src */

/* Your custom site styles */

Step 3: Configure Build Tool

Ensure your build process (PostCSS, Sass, Vite, Webpack, etc.) handles the import, autoprefixing, and minification.

Step 4: Customize Variables

Override CSS variables in your own CSS file before importing Axiom styles, or in `:root`.

Step 5: Use Axiom JavaScript

Import and initialize the JS module or include the compiled script.

// Example: In your main app.js
// import initializeAxiomComponents from 'axiom-css/src/scripts';
// initializeAxiomComponents();

Layouts with Axiom

Axiom primarily uses the semantic structure of your HTML to create layouts. Here's how to achieve common patterns:

Semantic Layouts (Recommended Approach)

Leverage standard HTML5 elements. Axiom applies CSS Grid/Flexbox rules to these tags automatically based on their nesting within <main> or <section>.

Single Column (Default)

Content directly within <main> or a single <section> / <article> / <aside> will naturally form a single column, constrained by the parent's width (often `main` or `.container`).

This content is in a single column by default within the main flow.

<main>...<article>...</article>...</main>

Main Content with Sidebar

Place an <article> (for main content) and an <aside> (for the sidebar) as direct children of the same <section> (or <main>). Axiom's CSS automatically applies a grid layout (e.g., 2fr 1fr) on wider screens.

Main Content Area

This is the primary content...

Sidebar

This is the sidebar content...

<section> <!-- This section becomes the grid container --> <article>...</article> <aside>...</aside> </section>

Note: The exact ratio and breakpoint are defined in `axiom.css`...

Multiple Equal Columns (e.g., Cards or Tiles)

Place multiple <article> elements (or other block elements) directly within a <section>. Axiom's CSS applies a responsive grid layout to arrange these items.

<section> <!-- This section becomes the grid container --> <article>Item 1</article> <article>Item 2</article> ... </section>

Note: The column count and minimum width are defined in `axiom.css`...
