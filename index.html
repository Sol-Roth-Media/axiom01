The Axiom01 Philosophy

Simplicity, semantics, and maintainability. Axiom01 helps you build beautiful interfaces with less markup and more meaning.

Semantic-First

Use semantic HTML elements for structure, not extra divs. Style with .card header, not .card-header-wrapper-container.

Minimal Classes

Style with concise selectors like .card header, not long class names. One class per component, semantic variants for modifications.

Accessibility Built-In

Every component includes proper ARIA attributes, keyboard navigation, and screen reader support out of the box.

Clean Markup

Fewer wrappers, logical order, and accessibility built-in.

Get Started with Axiom01

Add Axiom01 to your project in minutes, not hours

1

Include the CSS

Add Axiom01's CSS to your HTML head. That's it - no JavaScript required for basic components.

<link rel="stylesheet" href="css/axiom.css">
2

Write Semantic HTML

Use proper HTML elements with minimal, meaningful classes. Axiom01 enhances what you already know.

<article class="card">
  <header>
    <h3>Your Title</h3>
  </header>
  <p>Your content here</p>
  <footer>
    <button class="primary">Action</button>
  </footer>
</article>
3

Customize & Theme

Override CSS variables to match your brand, or choose from 20+ built-in themes.

:root {
  --a-color-primary: #your-brand-color;
  --a-font-family: 'Your-Font', sans-serif;
}
