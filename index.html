  • Ensure all buttons are reachable by keyboard (Tab key).
  • Use aria-disabled for disabled states if needed.

Accessibility Tips

  • Use aria-label for all form fields.
  • Group related fields with fieldset and legend.
  • Ensure all inputs have associated label elements.
  • Use required for mandatory fields.
  • Provide clear instructions and error messages.
  • Test forms with keyboard navigation and screen readers.
  • Use role="form" for complex forms to help assistive technologies.
  • Use autocomplete attributes for common fields (e.g., email, name).

Best Practices

  • Keep forms simple and focused.
  • Validate input on both client and server.
  • Use clear, concise labels and instructions.
  • Provide feedback for errors and successful submissions.
  • Support mobile and desktop layouts.

Media Component Demo

Accessibility Tips

  • Use alt text for all images.
  • Use aria-label for important media elements.
  • Ensure media content is keyboard and screen reader accessible.
  • Test with screen readers and keyboard navigation.

Best Practices

  • Use figure and figcaption for complex media blocks.
  • Combine images, icons, and text for richer content.
  • Use badges for status or type indication.
  • Provide captions for images and media when possible.
  • Use tabindex for custom interactive media elements.
  • Use tabindex="0" on custom interactive media elements to ensure keyboard accessibility.
  • Use role attributes for non-standard elements (e.g., role="img" for SVGs).
  • Consider aria-describedby for additional context or captions.
  • Use aria-live for dynamic media regions to announce updates to assistive technologies.
  • Test all interactive media with multiple browsers and assistive tech.
  • For video/audio, provide transcripts or captions for accessibility.
  • Use aria-hidden="true" for decorative images/icons.
  • For SVGs, use focusable="false" and aria-hidden="true" for decorative icons.
  • For carousels/slideshows, use aria-roledescription="carousel" and provide navigation controls with accessible labels.
  • For images/icons used as buttons or links, ensure aria-label or aria-labelledby describes their action.
  • For complex media, use aria-controls to associate controls with their media regions.

Axiom01 Philosophy

Simplicity, semantics, and maintainability. Axiom01 helps you build beautiful interfaces with less markup and more meaning.

  • Semantic-first Use semantic HTML elements for structure, not extra divs.
  • Minimal classes Style with concise selectors like .card header, not long class names.
  • Clean markup Fewer wrappers, logical order, and accessibility built-in.
  • Maintainable CSS Use variables, element selectors, and combine classes for variants.
  • Easy to read & extend Your codebase stays simple, flexible, and modern.

What Makes Axiom01 Different

Axiom01 is a modern, semantic-first UI framework designed for speed, clarity, and accessibility. Whether you're building a dashboard, landing page, or full web app, Axiom01 helps you create beautiful, maintainable interfaces with less effort.

Get started quickly, customize with 20+ built-in themes, and enjoy a clean, readable codebase. Axiom01 is built for developers who value simplicity, flexibility, and performance.

Semantic-First Styling

Unlike frameworks that require countless utility classes, Axiom01 styles semantic HTML elements directly, resulting in cleaner markup that's easier to read and maintain.

Class Composition Strategy

Instead of littering your HTML with utility classes, Axiom01 uses strategic class composition that respects the semantic meaning of your elements while providing all the styling power you need.

Powerful Theming Engine

With 20+ built-in themes ranging from professional light/dark modes to vibrant aesthetic options, switching the entire look of your application requires changing just one attribute.

Lightweight Foundation

Designed for performance with a focus on only including what you'll actually use, Axiom01 aims to keep your applications responsive and fast-loading.

Progressive Enhancement

Built to work with the grain of HTML and CSS, not against it, enhancing the natural capabilities of the web platform rather than fighting them.

Accessibility in Mind

Being developed with keyboard navigation and screen reader compatibility as core principles, not afterthoughts.

Minimal CSS Philosophy

Axiom01 is built on the belief that beautiful interfaces should be crafted with semantic HTML and minimal, meaningful CSS classes. Instead of stacking utility classes, Axiom01 styles your markup based on structure and meaning, making your code easier to read, maintain, and extend.

Markup Comparison

Before (Utility-heavy)

<div class="card card-header">
  <div class="card-header">Title</div>
  <div class="card-content">Content</div>
  <div class="card-footer">Actions</div>
</div>

After (Axiom01)

<article class="card">
  <header>Title</header>
  <section>Content</section>
  <footer>Actions</footer>
</article>

Result: Less markup, more meaning, and automatic styling for each section.

Theming System

Axiom01 supports instant theme switching and a semantic color system. Choose from 20+ built-in themes or create your own. All components adapt automatically to your selected theme.

Theme Switcher

Switch themes instantly using the theme switcher in the navbar. All colors and components update automatically.

<select id="theme-select" name="theme">
  <option value="light">Light</option>
  <option value="dark">Dark</option>
  <option value="ocean">Ocean</option>
  ...
</select>

Semantic Colors

Use semantic color variables for backgrounds, text, borders, and states. Example:

background: var(--a-color-success);
color: var(--a-color-on-success);
border-color: var(--a-color-outline);

See Style Guide: Colors for full reference.

Custom Themes

Create your own theme by overriding CSS variables. Example:

:root {
  --a-color-primary: #7c3aed;
  --a-color-on-primary: #fff;
  --a-bg-card: #f6f6ff;
}

See Theming Guide for details.
