How It Works: The Multi-Step Form

This component combines semantic HTML, targeted CSS, and event-driven JavaScript to create a seamless experience.

HTML Structure

The form uses a main container with `data-component="multi-step-form"`. Each step is a `div` with a `data-step` attribute, and only the `active` step is displayed.

<div data-component="multi-step-form">
  <!-- Step Headers -->
  <div class="multi-step-header">...</div>
  
  <form>
    <!-- Form Steps -->
    <div class="form-step active" data-step="1">...</div>
    <div class="form-step" data-step="2">...</div>
    
    <!-- Action Buttons -->
    <div class="form-actions">...</div>
  </form>
</div>

Validation and Feedback

We use standard HTML5 validation attributes like `required` and `minlength`. The JavaScript hooks into the browser's built-in `checkValidity()` method to test inputs before proceeding.

<input type="email" id="email" required>
<div class="error-message"></div>

When an input is invalid, its `validationMessage` is displayed in the `.error-message` container, providing immediate and accessible feedback.
