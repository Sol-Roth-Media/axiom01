Getting Started

Axiom01 is a lightweight, semantic-first CSS and JavaScript framework designed to help you build beautiful, accessible interfaces with minimal markup. Unlike traditional utility-first frameworks, Axiom01 emphasizes semantic HTML and CSS variables for cleaner, more maintainable code.

Key Features

  • Semantic HTML First: Work with your existing HTML structure
  • CSS Variables: Easy customization without complex build tools
  • Accessibility Built-in: ARIA attributes and keyboard navigation included
  • Multiple Themes: 20+ built-in themes with dark mode support
  • Performance Focused: Minimal JavaScript, optimized CSS

The Axiom Philosophy

Axiom01 is built on core principles that set it apart from other UI frameworks:

Minimal Class Usage

Unlike frameworks that require dozens of utility classes, Axiom01 leverages semantic HTML and CSS variables to reduce markup bloat and improve maintainability.

<!-- Traditional approach -->
<div class="bg-white p-4 rounded shadow-lg border">
  <h3 class="text-lg font-bold mb-2">Title</h3>
  <p class="text-gray-600">Content</p>
</div>

<!-- Axiom01 approach -->
<article class="card">
  <h3>Title</h3>
  <p>Content</p>
</article>

CSS Variables Over Classes

Axiom01 uses CSS custom properties extensively to create consistent design patterns, enable easy theming, and allow dynamic style changes without JavaScript.

:root {
  --a-color-primary: hsl(210, 80%, 50%);
  --a-space-base: 16px;
  --a-border-radius-base: 8px;
}

.card {
  background: var(--a-color-surface);
  padding: var(--a-space-base);
  border-radius: var(--a-border-radius-base);
}

Accessibility First

Accessibility isn't an afterthought—it's built into the framework from the ground up with ARIA attributes, keyboard navigation, and WCAG-compliant color contrast.

<button data-modal-target="example-modal"
        aria-haspopup="dialog">
  Open Modal
</button>

<div id="example-modal" class="modal"
     role="dialog"
     aria-modal="true"
     aria-labelledby="modal-title">
  <h2 id="modal-title">Modal Title</h2>
  <!-- Modal content -->
</div>

Performance Focused

Axiom01 is designed to be lightweight and performant with minimal JavaScript dependencies, no external libraries, and optimized CSS with low specificity.

  • CSS: ~50KB minified
  • JS: ~15KB minified
  • No external dependencies
  • Tree-shakeable components

Installation

Option 1: Download

Download the latest release from GitHub:

git clone https://github.com/Sol-Roth-Media/axiom01.git

Option 2: NPM

Install via npm for Node.js projects:

npm install axiom01

Option 3: CDN

Link directly to the hosted files:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/axiom01@latest/css/axiom.css">
<script src="https://cdn.jsdelivr.net/npm/axiom01@latest/js/axiom.js"></script>

Basic Usage

1. Include CSS and JavaScript

<!DOCTYPE html>
<html lang="en" data-theme="light">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Axiom01 Site</title>
    <link rel="stylesheet" href="css/axiom.css">
</head>
<body class="axiom01">
    <!-- Your content here -->
    <script src="js/axiom.js"></script>
</body>
</html>

2. Use Semantic HTML

Axiom01 works best with semantic HTML elements:

<main class="container">
    <section>
        <h1>Welcome to My Site</h1>
        <p>This is a paragraph with automatic styling.</p>

        <article class="card">
            <header>
                <h2>Card Title</h2>
            </header>
            <section>
                <p>Card content goes here.</p>
            </section>
            <footer>
                <button class="btn primary">Action</button>
            </footer>
        </article>
    </section>
</main>

<div class="alert error">
    <i class="fas fa-exclamation-triangle"></i>
    <strong>Error!</strong> There was a problem.

Cards

Flexible card layouts with semantic structure:

Card Title

This is the card content area. It can contain any HTML content.

<article class="card">
    <header>
        <h3>Card Title</h3>
    </header>
    <section>
        <p>Card content goes here.</p>
    </section>

        <button class="btn primary">Action</button>

Modal

Accessible modal dialogs with keyboard navigation:

<button data-modal-target="demo-modal">Open Modal</button>

<div id="demo-modal" class="modal" aria-hidden="true">

        <header>
            <h3>Modal Title</h3>
            <button class="closeBtn" data-modal-close>×</button>
        </header>
        <section>
            <p>Modal content goes here.</p>
        </section>
        <footer>
            <button class="btn primary">Confirm</button>
            <button class="btn secondary" data-modal-close>Cancel</button>
        </footer>
    </div>
</div>

Switching Themes

Themes can be changed programmatically or through the theme switcher:

// Change theme programmatically
document.documentElement.setAttribute('data-theme', 'dark');

// Or use the theme switcher component
<select id="theme-select">
    <option value="light">Light</option>
    <option value="dark">Dark</option>
    <option value="aesthetic">Aesthetic</option>
</select>

Customization

Axiom01 makes customization easy through CSS variables:

Color Variables

:root {
  /* Primary colors */
  --a-color-primary: hsl(210, 80%, 50%);
  --a-color-secondary: hsl(280, 60%, 60%);

  /* Semantic colors */
  --a-color-success: hsl(120, 50%, 45%);
  --a-color-error: hsl(0, 65%, 55%);
  --a-color-warning: hsl(45, 85%, 55%);
  --a-color-info: hsl(200, 80%, 55%);

  /* Surface colors */
  --a-color-surface: hsl(0, 0%, 100%);
  --a-color-background: hsl(0, 0%, 98%);
}

Spacing Variables

:root {
  /* Spacing scale */
  --a-space-tiny: 4px;
  --a-space-small: 8px;
  --a-space-base: 16px;
  --a-space-medium: 24px;
  --a-space-large: 32px;
  --a-space-xlarge: 48px;
  --a-space-xxlarge: 64px;
}

Typography Variables

:root {
  /* Font families */
  --a-font-family-sans: "Open Sans", system-ui, sans-serif;
  --a-font-family-mono: "Courier New", monospace;

  /* Font sizes */
  --a-font-size-base: 1rem;
  --a-font-size-small: 0.875rem;
  --a-font-size-large: 1.25rem;
  --a-font-size-xlarge: 1.5rem;

  /* Line heights */
  --a-line-height-base: 1.6;
  --a-line-height-heading: 1.3;
}

Accessibility

Axiom01 is built with accessibility as a core principle:

ARIA Attributes

All interactive components include appropriate ARIA attributes:

<button aria-haspopup="dialog" aria-expanded="false">Menu</button>
<div role="dialog" aria-modal="true" aria-labelledby="title">
    <h2 id="title">Dialog Title</h2>
</div>

Keyboard Navigation

All components support keyboard navigation:

  • Tab - Navigate between focusable elements
  • Enter / Space - Activate buttons and links
  • Escape - Close modals and dropdowns
  • Arrow keys - Navigate within components

Color Contrast

All themes meet WCAG 2.1 AA contrast requirements:

  • Normal text: 4.5:1 contrast ratio
  • Large text: 3:1 contrast ratio
  • Interactive elements: Clear focus indicators
