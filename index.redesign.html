The Axiom01 Philosophy

Simplicity, semantics, and maintainability. Axiom01 helps you build beautiful interfaces with less markup and more meaning.

Semantic-First

Use semantic HTML elements for structure, not extra divs. Style with .card header, not .card-header-wrapper-container.

<article class="card">
  <header>
    <h3>Card Title</h3>
  </header>
  <p>Content goes here</p>
</article>

Minimal Classes

Style with concise selectors like .card header, not long class names. One class per component, semantic variants for modifications.

<button class="primary large">Action</button>
<article class="card elevated">...</article>
<form class="compact">...</form>

Accessibility Built-In

Every component includes proper ARIA attributes, keyboard navigation, and screen reader support out of the box.

WCAG 2.1 AA Keyboard Navigation Screen Readers

Clean Markup

Fewer wrappers, logical order, and accessibility built-in.

Maintainable CSS

Use variables, element selectors, and combine classes for variants. Your codebase stays simple, flexible, and modern.

Minimal CSS Philosophy in Action

See the difference between utility-heavy frameworks and Axiom01's semantic approach

❌ Utility-Heavy Approach

<div class="bg-white rounded-lg shadow-lg p-6 max-w-sm mx-auto">
  <div class="flex items-center space-x-4">
    <div class="shrink-0">
      <img class="h-12 w-12 rounded-full" src="..." alt="">
    </div>
    <div>
      <div class="text-xl font-medium text-black">Chris Evans</div>
      <p class="text-slate-500">Product Manager</p>
    </div>
  </div>
</div>
Problems:
  • 15+ utility classes for simple card
  • No semantic meaning
  • Hard to maintain and customize
  • Verbose and repetitive

✅ Axiom01 Semantic Approach

<article class="card">
  <header class="profile">
    <img src="..." alt="Chris Evans">
    <div>
      <h3>Chris Evans</h3>
      <p>Product Manager</p>
    </div>
  </header>
</article>
Benefits:
  • Semantic HTML with meaning
  • Single class, clean markup
  • Easy to style and maintain
  • Accessible by default

Get Started with Axiom01

Add Axiom01 to your project in minutes, not hours

1

Include the CSS

Add Axiom01's CSS to your HTML head. That's it - no JavaScript required for basic components.

<link rel="stylesheet" href="css/axiom.css">
2

Write Semantic HTML

Use proper HTML elements with minimal, meaningful classes. Axiom01 enhances what you already know.

<article class="card">
  <header>
    <h3>Your Title</h3>
  </header>
  <p>Your content here</p>
  <footer>
    <button class="primary">Action</button>
  </footer>
</article>
3

Customize & Theme

Override CSS variables to match your brand, or choose from 20+ built-in themes.

:root {
  --a-color-primary: #your-brand-color;
  --a-font-family: 'Your-Font', sans-serif;
}
