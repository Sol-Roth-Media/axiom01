The Axiom01 Philosophy

Simplicity, semantics, and maintainability. Axiom01 helps you build beautiful interfaces with less markup and more meaning.

Semantic-First

Use semantic HTML elements for structure, not extra divs. Style with .card header, not .card-header-wrapper-container.

<article class="card">
  <header>
    <h3>Card Title</h3>
  </header>
  <p>Content goes here</p>
</article>

Minimal Classes

Style with concise selectors like .card header, not long class names. One class per component, semantic variants for modifications.

<button class="primary large">Action</button>
<article class="card elevated">...</article>
<form class="compact">...</form>

Accessibility Built-In

Every component includes proper ARIA attributes, keyboard navigation, and screen reader support out of the box.

WCAG 2.1 AA Keyboard Navigation Screen Readers

Clean Markup

Fewer wrappers, logical order, and accessibility built-in.

❌ Other Frameworks: 
<div class="btn btn-primary btn-lg btn-block">
✅ Axiom01: 
<button class="primary large">

Maintainable CSS

Use variables, element selectors, and combine classes for variants. Your codebase stays simple, flexible, and modern.

Minimal CSS Philosophy

Axiom01 is built on the belief that beautiful interfaces should be crafted with semantic HTML and minimal, meaningful CSS classes.

Markup Comparison

Before (Utility-heavy)
<div class="card card-header">
  <div class="card-header">Title</div>
  <div class="card-content">Content</div>
  <div class="card-footer">Actions</div>
</div>
After (Axiom01)
<article class="card">
  <header>Title</header>
  <section>Content</section>
  <footer>Actions</footer>
</article>

Result: Less markup, more meaning, and automatic styling for each section.

Real-World Example: Dashboard Layout

See how Axiom01 enables clean, semantic markup for a modern dashboard. This example uses proper HTML elements for structure and instant styling.

<header class="dashboard-header">Dashboard Title</header>
<nav class="dashboard-nav">Sidebar Navigation</nav>
<main class="dashboard-main">
  <section class="dashboard-widgets">
    <article class="card">Widget 1</article>
    <article class="card">Widget 2</article>
    <article class="card">Widget 3</article>
  </section>
</main>
<aside class="dashboard-info">Quick Info</aside>

Result: Instantly styled, accessible, and maintainable dashboard layout with minimal classes.

Tips & Tricks for Axiom01

  • Use semantic HTML tags (article, header, nav, etc.) for instant styling
  • Switch themes by setting data-theme on <body>
  • Use button.primary and button.secondary for consistent call-to-action styles
  • Check the style guide for all available classes and variables
  • Use aria-label and role attributes for accessibility
  • Preview components live before using them in your project
  • Customize colors and spacing via CSS variables
  • Read the Axiom01 Philosophy for design principles

Get Started with Axiom01

Add Axiom01 to your project in minutes, not hours

1

Include the CSS

Add Axiom01's CSS to your HTML head. That's it - no JavaScript required for basic components.

<link rel="stylesheet" href="css/axiom.css">
2

Write Semantic HTML

Use proper HTML elements with minimal, meaningful classes. Axiom01 enhances what you already know.

<article class="card">
  <header>
    <h3>Your Title</h3>
  </header>
  <p>Your content here</p>
  <footer>
    <button class="primary">Action</button>
  </footer>
</article>
3

Customize & Theme

Override CSS variables to match your brand, or choose from 20+ built-in themes.

:root {
  --a-color-primary: #your-brand-color;
  --a-font-family: 'Your-Font', sans-serif;
}

Style Guide & Theming

Axiom01 supports multiple themes and advanced styling. Try switching themes above, and see the style guide for more.

Light Theme

Clean, modern, and easy on the eyes. Ideal for most projects.

Dark Theme

Perfect for low-light environments and users who prefer dark UIs.

Forest Theme

Soft, creative, and visually unique for standout projects.

Color Palette

Axiom01 uses CSS variables for easy color customization:

:root {
  --a-color-primary: #3b82f6;
  --a-color-surface: #fff;
  --a-color-background: #f8f9fa;
}

Typography

Accessible, scalable, and themeable text styles. Use semantic tags (h1, p, small) for best results.

Spacing & Layout

Use CSS variables for consistent spacing:

padding: var(--a-padding-large);
margin: var(--a-margin-base);

Best Practices

  • Use semantic HTML for structure
  • Customize themes with CSS variables
  • Keep markup clean and accessible
  • Use utility classes for spacing and layout
