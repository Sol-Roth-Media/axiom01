Axiom01 UI Framework

Build beautiful, accessible interfaces with less markup. Semantic-first, themeable, and fast. Try switching themes above!

Get Started Component Demos GitHub

Why Axiom01?

Axiom01 is designed for developers who want semantic, maintainable, and accessible UI with minimal class clutter. Unlike utility-first frameworks, Axiom01 leverages CSS variables and semantic HTML for easier customization and better accessibility.

Comparison:

Axiom01

  • Semantic HTML
  • CSS variables for theming
  • Accessible by default
  • Minimal class usage
  • Easy theme switching

Utility Frameworks

  • Heavy class usage
  • Harder to customize themes
  • Accessibility requires extra effort
  • Higher learning curve
  • More markup bloat

How to Use Axiom01

Getting started with Axiom01 is simple:

Installation

Choose one of these installation methods:

Option 1: Download or Clone from GitHub

git clone https://github.com/Sol-Roth-Media/axiom01.git

Option 2: Using NPM

npm install axiom01

Basic Usage

  1. Include the necessary CSS files in your HTML: 
    <link rel="stylesheet" href="css/axiom.css">
  2. Include the JavaScript file at the end of your body: 
    <script src="js/scripts.js"></script>
  3. Structure your content using semantic HTML elements like <main>, <section>, <article>, and <aside>.
  4. Use Axiom01's components and grid system for layout and UI elements.
  5. Customize the design using Axiom01's CSS variables or by adding your own CSS rules.

Theming

Apply a theme by setting the data-theme attribute on the HTML element:

<html data-theme="dark">

For more detailed instructions and examples, refer to the documentation sections below.

Axiom01 Style Guide

The Axiom01 framework is built on semantic HTML, CSS variables, and accessible design tokens. Use the following guidelines to create consistent, beautiful interfaces:

  • Semantic-first: Use HTML elements for their intended purpose (e.g., <nav>, <section>, <button>).
  • Design tokens: Customize colors, spacing, and typography using CSS variables (see axiom_vars.css).
  • Accessibility: Use ARIA attributes and proper labeling for navigation, forms, and interactive elements.
  • Responsive layout: Use flexbox and grid utilities for adaptive design.
  • Themeable: Switch themes via data-theme or the theme switcher.

Design Tokens Example

:root {
  --a-color-primary: #007bff;
  --a-color-surface: #fff;
  --a-font-size-base: 1rem;
  --a-border-radius-base: 8px;
  --a-padding-base: 1rem;
}

Component API Reference

Button

<button class="btn">Primary</button>
<button class="btn">Default</button>
  • .btn: Base button style
  • .primary: Primary color button

Card

<div class="card">
  <header>Title</header>
  <section>Content</section>
  <footer>Footer</footer>
</div>
  • .card: Container for grouped content
  • Use <header>, <section>, <footer> for semantic structure

Alert

<div class="alert">Info message</div>
  • .alert: Highlight important messages
  • Style with inline background and color or use theme classes

Form

<form>
  <label for="input1">Label</label>
  <input id="input1" type="text">
  <button class="btn primary">Submit</button>
</form>
  • Label all fields for accessibility
  • Use .btn for form actions
